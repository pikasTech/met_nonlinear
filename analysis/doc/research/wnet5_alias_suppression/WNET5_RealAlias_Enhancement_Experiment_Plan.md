# WNET5_RealAlias 补偿效果增强实验计划

## 目标
通过系统性的超参数调优实验，提升 WNET5_RealAlias 项目的aliasing补偿效果，基于控制变量原则设计多个实验变体进行对比分析。

## 当前基线配置分析

### 现有配置 (WNET5_RealAlias - 基线)
```json
{
    "epoch_train": 30000,
    "learning_rate": 0.02,
    "use_model": "WaveNet5",
    "dataset_type": "Alias",
    "kernal_units": 6,
    "auto_lr_decay_steps": 1000,
    "model_subcfg": {
        "init_center_freqs": [8, 25, 50, 85, 120, 180],
        "init_quality_factors": [1.5, 2.0, 2.5, 3.0, 4.0, 5.0],
        "post_dense": true,
        "post_dense_activation": "relu",
        "post_dense_units": 8,
        "post_dense_layers": 4,
        "use_dense_bias": true
    }
}
```

### 问题分析
当前配置的补偿效果不明显，可能的原因：
1. **频率覆盖不充分** - 低频段（0-8Hz）和高频段（180Hz+）缺乏覆盖
2. **非线性建模能力不足** - Dense层配置可能过于保守
3. **学习率策略** - 可能不够优化
4. **滤波器数量限制** - 6个滤波器可能不足以捕获复杂的aliasing模式

## 命名规则

### 项目命名格式
`WNET5_RealAlias_E{编号}`

### 实验编号说明
- **E01-E03**: 核心验证组
- **E04-E06**: 补充对比组

### 示例
- `WNET5_RealAlias_E01` - 扩展频率范围
- `WNET5_RealAlias_E02` - 深层Dense网络
- `WNET5_RealAlias_E06` - 激活函数对比(tanh)

### 技术约束
- **激活函数**: 仅支持 `relu` 和 `tanh`，不使用高级激活函数
- **频率范围**: 基于电化学系统特性，重点优化 0-250Hz 范围

## 高优先级实验设计 (6个)

### E01 - 扩展频率范围
**实验目的**: 验证低频覆盖对补偿效果的关键影响
**变更参数**:
- `init_center_freqs`: [3, 8, 25, 50, 85, 120, 180, 250]
- `kernal_units`: 8

**预期情况分析**:
- **情况A (显著改善)**: 低频补偿效果明显提升 → 结论：低频aliasing是主要问题，需要扩展频率范围
- **情况B (轻微改善)**: 整体性能略有提升 → 结论：频率范围有帮助但非关键因素
- **情况C (无显著变化)**: 补偿效果无明显改善 → 结论：频率覆盖不是瓶颈，问题在其他方面
- **情况D (性能下降)**: 补偿效果变差 → 结论：过多频率点引入噪声，需要优化频率选择策略

### E02 - 深层Dense网络
**实验目的**: 验证增加网络深度对非线性建模能力的影响
**变更参数**:
- `post_dense_layers`: 6 (从4增加)
- `post_dense_units`: 12 (从8增加)

**预期情况分析**:
- **情况A (非线性建模改善)**: 补偿效果显著提升 → 结论：aliasing补偿需要更强的非线性建模能力
- **情况B (过拟合)**: 训练损失下降但验证性能变差 → 结论：需要正则化或减少复杂度
- **情况C (训练困难)**: 收敛缓慢或不稳定 → 结论：需要调整学习率或初始化策略

### E03 - 保守学习率策略
**实验目的**: 验证精细优化对最终补偿效果的影响  
**变更参数**:
- `learning_rate`: 0.005 (从0.02降低)
- `auto_lr_decay_steps`: 2000 (从1000增加)
- `epoch_train`: 45000 (增加训练轮次)

**预期情况分析**:
- **情况A (精细优化)**: 最终性能明显超过基线 → 结论：aliasing补偿需要更精细的参数调优
- **情况B (收敛缓慢)**: 训练时间过长但性能提升有限 → 结论：需要找到效率和性能的平衡点
- **情况C (效果相当)**: 延长训练没有带来性能提升 → 结论：当前学习率已经合适

### E04 - 高质量因子配置
**实验目的**: 验证提高频率选择性对补偿精度的影响
**变更参数**:
- `init_quality_factors`: [3.0, 4.0, 5.0, 6.0, 7.0, 8.0]

**预期情况分析**:
- **情况A (频率精度提升)**: 补偿效果明显改善 → 结论：需要更窄的频带滤波
- **情况B (稳定性问题)**: 训练不稳定或过拟合 → 结论：过高Q值导致数值问题
- **情况C (性能相当)**: 与基线相近 → 结论：当前Q值已经合适

### E05 - 宽层Dense网络
**实验目的**: 验证增加网络宽度与深度的不同效果
**变更参数**:
- `post_dense_units`: 16 (从8增加)
- `post_dense_layers`: 3 (从4减少)

**预期情况分析**:
- **情况A (表达能力增强)**: 性能优于E02 → 结论：宽层比深层更适合aliasing补偿
- **情况B (性能相当)**: 与E02相近 → 结论：深度和宽度效果相当，可选择计算效率更高的
- **情况C (效果不如深层)**: 性能低于E02 → 结论：aliasing补偿更需要深层特征抽象

### E06 - 激活函数对比
**实验目的**: 验证不同激活函数对补偿效果的影响
**变更参数**:
- `post_dense_activation`: "tanh" (替代relu)

**预期情况分析**:
- **情况A (性能提升)**: tanh的饱和特性有助于避免梯度爆炸，补偿效果改善 → 结论：tanh更适合aliasing补偿任务
- **情况B (收敛稳定性)**: 训练过程更稳定，损失曲线更平滑 → 结论：tanh的有界输出有助于训练稳定性
- **情况C (无显著差异)**: 与relu效果相当 → 结论：激活函数对该任务不敏感

## 实验执行计划

### 第一阶段 (核心验证)
1. **E01** - 扩展频率范围
2. **E02** - 深层Dense网络  
3. **E03** - 保守学习率策略

### 第二阶段 (补充对比)
4. **E04** - 高质量因子
5. **E05** - 宽层Dense网络
6. **E06** - 激活函数对比(tanh)

## 决策树分析

### 基于第一阶段结果的决策路径

**如果E01效果显著** → 频率覆盖是关键瓶颈，后续重点优化频率策略
**如果E02效果显著** → 网络容量是关键瓶颈，E05验证宽层vs深层效果  
**如果E03效果显著** → 学习策略是关键瓶颈，可以在最佳配置基础上进一步微调
**如果E01+E02组合最佳** → 频率和容量都重要，需要综合优化
**如果三者都无显著改善** → 重新审视问题假设，考虑数据质量或模型架构问题

## 成功标准

### 量化指标
- **显著改善**: 补偿效果提升 > 30%
- **轻微改善**: 补偿效果提升 10-30%
- **无显著变化**: 补偿效果变化 < 10%
- **性能下降**: 补偿效果下降 > 5%

### 质量指标  
- **训练稳定性**: 损失曲线平滑度
- **收敛速度**: 达到稳定性能所需epoch数
- **泛化能力**: 验证集与训练集性能差距
