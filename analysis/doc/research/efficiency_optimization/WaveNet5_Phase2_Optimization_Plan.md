# WaveNet5效率优化Phase2实验计划 - 基于Phase1经验的保守优化策略

## 背景与目标调整

Phase1实验的**严峻现实**：所有6个优化实验均未达到80% ASR目标，最佳的EFF_B1仅实现60.8% ASR（相比E05基准90.3%损失29.5%）。这证明了E05配置（8,641参数）已接近当前架构的效率边界。

**Phase2目标重新定位**：
- **原目标**：在保持80% ASR前提下，参数量<5,000
- **调整后**：在保持85% ASR前提下，探索参数量7,000-8,000的优化空间
- **核心转变**：从"激进削减"转向"精细优化"，寻找性能与效率的真实平衡点

## Phase1关键教训总结

### 残酷的事实
1. **参数削减代价巨大**: 3.1%参数减少导致31.7%性能损失（EFF_A1）
2. **IIR滤波器不可或缺**: 6→4个IIR直接导致29.5%性能损失
3. **Dense层贡献被低估**: 减少层数或单元数都导致严重性能下降
4. **效率悖论**: 参数效率提升（如EFF_D1达17.19）但绝对性能崩溃（39.5%）

### 积极的发现
1. **EFF_B1相对最佳**: 4个IIR配置在所有削减方案中表现最好
2. **5个IIR未探索**: 可能是6个和4个之间的最优平衡点
3. **微调空间存在**: Phase1步长较大，精细调整可能有改善
4. **E05验证成功**: 证明了E05各组件设计的合理性

## Phase2精细优化策略

基于E05（90.3% ASR，8,641参数）作为基准，采用**保守的微调策略**，承认大幅参数削减的不可行性。

### 实验设计原则
1. **微调为主**: 单个参数变化控制在10-20%以内
2. **性能优先**: 宁可参数减少有限，也要保证ASR≥85%
3. **科学探索**: 即使无法达到原始目标，也要明确优化边界
4. **实事求是**: 接受E05可能已是最优配置的可能性

## Phase2实验设计（6个保守实验）

### 实验1: 保守Dense微调（最小改动方案）
- **名称**: EFF2_A1
- **目标**: 基于E05，仅微调Dense层单元数，寻找最小性能损失点
- **配置修改**:
  ```json
  "kernal_units": 6,
  "model_subcfg": {
      "init_center_freqs": [8, 25, 50, 85, 120, 180],
      "init_quality_factors": [1.5, 2.0, 2.5, 3.0, 4.0, 5.0],
      "post_dense_layers": 3,
      "post_dense_units": 14  // E05基准16→14（减少12.5%）
  }
  ```
- **预计参数量**: ~8,200（减少5%）
- **设计理由**: Phase1证明IIR不可削减，仅尝试最小幅度Dense调整

**预期情况分析**:
- **情况A (最优)**: ASR 87-89%，损失1-3%，证明轻微调整可行
- **情况B (可接受)**: ASR 85-87%，达到调整后目标
- **情况C (边界)**: ASR 82-85%，接近可接受边界
- **情况D (失败)**: ASR <82%，证明16是最小必需单元数

### 实验2: 中度Dense调整（探索边界）
- **名称**: EFF2_A2
- **目标**: 基于E05，探索Dense层的调整边界
- **配置修改**:
  ```json
  "kernal_units": 6,
  "model_subcfg": {
      "init_center_freqs": [8, 25, 50, 85, 120, 180],
      "init_quality_factors": [1.5, 2.0, 2.5, 3.0, 4.0, 5.0],
      "post_dense_layers": 3,
      "post_dense_units": 12  // E05基准16→12（减少25%）
  }
  ```
- **预计参数量**: ~7,700（减少11%）
- **设计理由**: Phase1的EFF_A2证明16→8损失巨大，尝试中间值

**预期情况分析**:
- **情况A (期望)**: ASR 82-85%，找到可接受的平衡点
- **情况B (边缘)**: ASR 78-82%，接近但未达目标
- **情况C (预期)**: ASR 75-78%，符合Phase1的性能下降趋势
- **情况D (失败)**: ASR <75%，证明Dense削减代价过大

### 实验3: 5个IIR探索（寻找甜点）
- **名称**: EFF2_B1
- **目标**: 验证5个IIR是否为6个和4个之间的最优平衡
- **配置修改**:
  ```json
  "kernal_units": 5,
  "model_subcfg": {
      "init_center_freqs": [10, 30, 60, 100, 160],
      "init_quality_factors": [1.5, 2.0, 2.5, 3.5, 4.5],
      "post_dense_layers": 3,
      "post_dense_units": 16  // 保持E05的Dense配置
  }
  ```
- **预计参数量**: ~7,200（减少17%）
- **设计理由**: Phase1显示6→4个IIR损失29.5%，5个可能是平衡点

**预期情况分析**:
- **情况A (理想)**: ASR 78-82%，找到性能-参数的甜点
- **情况B (可行)**: ASR 72-78%，比4个IIR有明显改善
- **情况C (线性)**: ASR 68-72%，性能介于4个和6个IIR之间
- **情况D (不足)**: ASR <68%，证明至少需要6个IIR

### 实验4: Dense层数微调（结构精简）
- **名称**: EFF2_B2
- **目标**: 基于E05，尝试减少Dense层数但保持总容量
- **配置修改**:
  ```json
  "kernal_units": 6,
  "model_subcfg": {
      "init_center_freqs": [8, 25, 50, 85, 120, 180],
      "init_quality_factors": [1.5, 2.0, 2.5, 3.0, 4.0, 5.0],
      "post_dense_layers": 2,  // E05基准3→2
      "post_dense_units": 20   // 补偿层数减少
  }
  ```
- **预计参数量**: ~8,100（减少6%）
- **设计理由**: Phase1的EFF_A1显示3→2层损失严重，尝试增加宽度补偿

**预期情况分析**:
- **情况A (补偿有效)**: ASR 84-87%，宽度补偿了深度损失
- **情况B (部分补偿)**: ASR 80-84%，有一定效果但未完全补偿
- **情况C (补偿不足)**: ASR 75-80%，深度对性能很重要
- **情况D (失败)**: ASR <75%，3层是必需的最小深度

### 实验5: 训练策略优化（补偿方案）
- **名称**: EFF2_C1
- **目标**: 通过优化训练策略补偿适度的参数削减
- **配置修改**:
  ```json
  "epoch_train": 40000,      // E05基准30000→40000
  "learning_rate": 0.015,     // E05基准0.02→0.015
  "auto_lr_decay_steps": 1200,
  "kernal_units": 6,
  "model_subcfg": {
      "init_center_freqs": [8, 25, 50, 85, 120, 180],
      "init_quality_factors": [1.5, 2.0, 2.5, 3.0, 4.0, 5.0],
      "post_dense_layers": 3,
      "post_dense_units": 13   // 轻微削减
  }
  ```
- **预计参数量**: ~7,950（减少8%）
- **设计理由**: 验证精细训练策略能否补偿轻微的容量不足

**预期情况分析**:
- **情况A (策略有效)**: ASR 85-88%，训练优化补偿了参数削减
- **情况B (部分有效)**: ASR 82-85%，有一定帮助
- **情况C (效果有限)**: ASR 78-82%，训练策略影响有限
- **情况D (过拟合)**: ASR <78%，长时间训练导致过拟合

### 实验6: EFF_B1改进版（频率优化）
- **名称**: EFF2_C2
- **目标**: 基于Phase1最佳的EFF_B1，优化频率分布和Q值
- **配置修改**:
  ```json
  "kernal_units": 4,
  "model_subcfg": {
      "init_center_freqs": [10, 35, 75, 150],  // 优化频率分布
      "init_quality_factors": [1.5, 2.0, 3.0, 4.0],  // 优化Q值
      "post_dense_layers": 3,
      "post_dense_units": 18   // 轻微增加Dense容量
  }
  ```
- **预计参数量**: ~4,600（减少47%）
- **设计理由**: 基于60.8%的EFF_B1基线，通过频率优化寻求突破

**预期情况分析**:
- **情况A (频率优化成功)**: ASR 65-68%，优于原EFF_B1
- **情况B (轻微改善)**: ASR 62-65%，略有提升
- **情况C (性能相当)**: ASR 58-62%，频率影响有限
- **情况D (性能下降)**: ASR <58%，原始频率更优

## 实验执行策略

### 优先级分层
#### 第一批次（最高优先级）
1. **EFF2_A1** - 保守Dense微调（验证E05的稳健性）
2. **EFF2_B1** - 5个IIR探索（寻找6和4之间的平衡点）

#### 第二批次（根据第一批结果决定）
3. **EFF2_A2** - 中度Dense调整（如果A1显示有空间）
4. **EFF2_B2** - Dense层数微调（探索结构简化）

#### 第三批次（精细优化）
5. **EFF2_C1** - 训练策略优化（验证训练补偿效果）
6. **EFF2_C2** - EFF_B1改进版（基于60.8%基线优化）

### 决策树逻辑

```
如果 EFF2_A1 ASR > 85%:
    → 达到调整后目标，证明E05有微调空间
    → 执行EFF2_A2探索更多削减可能

如果 EFF2_B1 ASR > 80%:
    → 找到了理想的平衡点（5个IIR）
    → 这将是重大突破

如果 EFF2_A1 ASR 82-85%:
    → 轻微削减可行，执行EFF2_C1验证训练优化
    → 探索训练策略的补偿潜力

如果两者都 < 82%:
    → 执行EFF2_C2优化Phase1最佳配置
    → 承认E05已接近最优
```

## 成功标准

### 理想成功（调整后目标）
- **任一实验ASR ≥ 85%** - 达到调整后的性能目标
- **参数量 < 8,000** - 保持适度的参数精简
- **参数效率 > 10.6 ASR/千参数** - 超越E05的效率

### 部分成功
- **最佳实验ASR > 82%** - 接近调整后目标
- **至少2个实验ASR > 78%** - 证明保守策略有效

### 验证性结果
- **所有实验ASR < 82%** - 证明E05确实已优化到极限
- **明确架构边界** - 为架构创新提供数据支撑

## 风险控制与应对

### 主要风险
1. **性能提升有限** - Phase2实验未能显著超越60.8%
2. **参数敏感性** - 微调导致性能大幅波动
3. **训练不稳定** - 小网络训练困难

### 应对策略
1. **保守策略** - 优先执行最有希望的A1、A2实验
2. **多次验证** - 对有希望的配置进行3次重复实验
3. **渐进式优化** - 避免同时改变多个参数

## 预期贡献

### 技术贡献
1. **优化边界确认** - 明确E05配置的稳健性和优化空间
2. **甜点探索** - 验证5个IIR是否为理想平衡点
3. **训练策略价值** - 评估训练优化对参数效率的贡献

### 实践指导
1. **配置建议** - 为不同需求场景提供经验证的配置
2. **取舍指南** - 明确性能与参数的权衡关系
3. **优化路径** - 为类似任务提供系统化优化方法论
