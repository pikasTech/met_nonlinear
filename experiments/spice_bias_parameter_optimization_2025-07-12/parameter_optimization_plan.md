## 1. 实验背景与调优策略

### 1.1 前期验证实验总结

**✅ 关键发现**:
- **系统功能性**: SPICE偏置补偿系统完全可用，按设计正常工作
- **调整能力**: 系统具备强大的偏置调整能力（第5层626倍响应）
- **层级特性**: SVF层偏置很少且不调整，Dense层("0"-"3")是调整重点
- **参数应用**: layer_bias_adjustments正确应用到各层

### 1.2 新的调优策略 - 基于技术发现的修正策略

**🚨 重要技术发现**:
- **SVF层("0")不支持偏置补偿**: 经技术调查确认，SVFLayer.to_spice()完全不支持偏置调整
- **只有Dense层支持偏置补偿**: 仅Dense层(1,2,3)和输出层(4)实际支持SPICE偏置调整
- **配置验证更严格**: 系统现在会拒绝包含层"0"的偏置配置

**🎯 修正后的核心原则**:
1. **先观察原始状态**: 关闭补偿，测量各层实际偏置误差
2. **针对性补偿**: 偏出去+x，就补偿-x
3. **跳过SVF层**: ❌ 严禁配置层"0"，技术上完全不支持
4. **只调Dense层**: 仅调整层"1","2","3","4" (Dense层1,2,3和输出层)
5. **逐层优化**: 先调"1","2"层 → 固定前面参数 → 再调"3"层 → 最后调"4"层
6. **手动精调**: 不急于自动化，先手动调出经验和规律

**🔍 修正后的调优重点**:
- **SVF层("0")**: ❌ **严禁调整** - 技术上完全不支持，配置会被拒绝
- **"1","2" (Dense层1,2)**: 优先调整 - 主要的偏置来源  
- **"3" (Dense层3)**: 在前两层固定后调整
- **"4" (输出层)**: 最后精调 - 最敏感但依赖前面层的结果

#### 2.4 调优策略细节

**微调方法**:
```python
# 如果某通道偏置误差为 +0.02，当前补偿为 -0.015
# 说明补偿不足，增加补偿到 -0.025 (增加50%)
# 如果过补偿导致 -0.03，则回调到 -0.020

def fine_tune_compensation(current_bias_error, current_compensation):
    if abs(current_bias_error) < 0.005:  # 已经很好
        return current_compensation
    elif current_bias_error > 0:  # 正偏置，需要增加负补偿
        return current_compensation - 0.005
    else:  # 负偏置，需要减少负补偿  
        return current_compensation + 0.005
```

**收敛判断**:
- Dense层"1"各通道偏置误差 < 0.01
- Dense层"2"各通道偏置误差 < 0.01
- 连续3次调整改善幅度 < 0.001
