## 4. 路径兼容实现方案

### 4.1 方案对比

| 方案 | 优点 | 缺点 | 代码修改量 |
|------|------|------|-----------|
| **方案A: 符号链接** | 无需修改任何代码，完全兼容 | 跨平台兼容性(Windows需要特殊处理) | **0行** |
| **方案B: PYTHONPATH修改** | 简单直接 | 需要修改启动脚本 | 1-2行 |
| **方案C: 修改所有导入** | 干净无链接 | 需要修改所有导入语句，约200+处 | **~500行** |
| **方案D: __init__.py重定向** | 纯Python方案 | 需要创建大量重定向文件 | ~15行 |

### 4.2 推荐方案：符号链接 + 兼容层 (Windows最佳实践)

由于项目运行在 Windows 环境，推荐使用**符号链接**实现路径兼容：

**执行步骤**：

1. **创建 src/ 目录结构**
   ```powershell
   mkdir src
   ```

2. **移动核心模块到 src/**
   ```powershell
   mv core src/
   mv models src/
   mv inference src/
   mv spice_simulator src/
   mv visualization src/
   mv analysis src/
   mv utils src/
   mv tfkan src/
   mv experimental src/
   mv logger src/
   mv data_utils src/
   mv tests src/
   ```

3. **创建符号链接 (Windows)**
   ```powershell
   # 创建指向 src/core 的符号链接 core
   mklink /D core src\core
   mklink /D models src\models
   mklink /D inference src\inference
   mklink /D spice_simulator src\spice_simulator
   mklink /D visualization src\visualization
   mklink /D analysis src\analysis
   mklink /D utils src\utils
   mklink /D tfkan src\tfkan
   mklink /D experimental src\experimental
   mklink /D logger src.logger
   mklink /D data_utils src\data_utils
   mklink /D tests src\tests
   ```

4. **验证迁移**
   ```bash
   python cli.py --help
   ```

**优势**：
- 零代码修改
- 完全向后兼容
- 原有导入路径无需任何改变

### 4.3 备选方案：PYTHONPATH 修改

如果符号链接在特定环境不可用，可采用 PYTHONPATH 方案：

**修改 cli.py** (仅需1行)：
```python
import sys
import os
sys.path.insert(0, os.path.join(os.path.dirname(__file__), 'src'))
```

## 5. 风险评估与应对

### 5.1 潜在风险

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|
| 符号链接权限问题 | 低 | 中 | 备选 PYTHONPATH 方案 |
| IDE 路径识别问题 | 中 | 低 | 重启 IDE 或刷新项目 |
| 文件锁定无法移动 | 低 | 中 | 关闭所有占用进程 |
| 路径大小写敏感性 | 低 | 中 | Windows 通常不敏感 |

### 5.2 回滚方案

如需回滚，执行反向操作：
```powershell
# 删除符号链接
rm core
rm models
# 从 src 移回原位
mv src/core .
mv src/models .
```
