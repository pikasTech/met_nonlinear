# R3.2 多 ML 后端支持调查报告

## 任务概述

**任务目标**: 支持多 ML 后端（TensorFlow 和 PyTorch），通过底层公共接口层实现，在 config.json 中指定后端。

**调查范围**:
- training 任务的执行流程
- 模型加载和实例化方式
- 数据预处理流程
- 评估指标计算
- 现有抽象层和接口设计

### 2.2 核心接口定义 (base.py)

```python
# src/backends/base.py

from abc import ABC, abstractmethod
from typing import Any, Dict, Optional
import numpy as np


class BaseDataLoader(ABC):
    """数据加载器抽象接口"""

    @abstractmethod
    def load_dataset(self, config) -> Any:
        """加载数据集"""
        pass

    @abstractmethod
    def prepare_training_data(self, config) -> tuple:
        """准备训练数据，返回 (x_train, y_train, x_test, y_test)"""
        pass

    @abstractmethod
    def get_scaler(self) -> Any:
        """获取数据缩放器"""
        pass


class BaseModel(ABC):
    """模型抽象接口"""

    @abstractmethod
    def build(self, config, **kwargs) -> None:
        """构建模型"""
        pass

    @abstractmethod
    def forward(self, x: np.ndarray) -> np.ndarray:
        """前向传播"""
        pass

    @abstractmethod
    def train_step(self, x: np.ndarray, y: np.ndarray) -> Dict:
        """单步训练"""
        pass

    @abstractmethod
    def evaluate(self, x: np.ndarray, y: np.ndarray) -> Dict:
        """评估模型"""
        pass

    @abstractmethod
    def save_weights(self, path: str) -> None:
        """保存权重"""
        pass

    @abstractmethod
    def load_weights(self, path: str) -> None:
        """加载权重"""
        pass

    @abstractmethod
    def summary(self) -> str:
        """获取模型摘要"""
        pass


class BaseOptimizer(ABC):
    """优化器抽象接口"""

    @abstractmethod
    def step(self, gradients: np.ndarray) -> None:
        """更新参数"""
        pass

    @abstractmethod
    def get_lr(self) -> float:
        """获取当前学习率"""
        pass

    @abstractmethod
    def set_lr(self, lr: float) -> None:
        """设置学习率"""
        pass


class BaseLoss(ABC):
    """损失函数抽象接口"""

    @abstractmethod
    def __call__(self, y_true: np.ndarray, y_pred: np.ndarray) -> float:
        """计算损失"""
        pass


class BaseLRScheduler(ABC):
    """学习率调度抽象接口"""

    @abstractmethod
    def step(self, epoch: int) -> float:
        """根据 epoch 返回学习率"""
        pass


class BaseCallback(ABC):
    """回调抽象接口"""

    @abstractmethod
    def on_epoch_begin(self, epoch: int, logs: Dict = None) -> None:
        pass

    @abstractmethod
    def on_epoch_end(self, epoch: int, logs: Dict = None) -> None:
        pass

    @abstractmethod
    def on_train_begin(self, logs: Dict = None) -> None:
        pass

    @abstractmethod
    def on_train_end(self, logs: Dict = None) -> None:
        pass
```

### 2.3 后端工厂函数 (\_\_init__.py)

```python
# src/backends/__init__.py

from typing import Type, Dict
from .base import (
    BaseDataLoader, BaseModel, BaseOptimizer,
    BaseLoss, BaseLRScheduler, BaseCallback
)
from .tensorflow import (
    TFDataLoader, TFModel, TFOptimizer,
    TFLoss, TFLRScheduler, TFCallback
)

# 后端注册表
BACKEND_REGISTRY: Dict[str, Dict] = {
    'tensorflow': {
        'data_loader': TFDataLoader,
        'model': TFModel,
        'optimizer': TFOptimizer,
        'loss': TFLoss,
        'lr_scheduler': TFLRScheduler,
        'callback': TFCallback,
    },
    'pytorch': {
        # 待实现
    },
}


def get_backend(name: str) -> Dict:
    """获取指定后端的组件类"""
    if name not in BACKEND_REGISTRY:
        raise ValueError(f"Unknown backend: {name}. Available: {list(BACKEND_REGISTRY.keys())}")
    return BACKEND_REGISTRY[name]


def register_backend(name: str, components: Dict) -> None:
    """注册新的后端"""
    BACKEND_REGISTRY[name] = components


def create_data_loader(backend: str, **kwargs) -> BaseDataLoader:
    """创建数据加载器"""
    cls = get_backend(backend)['data_loader']
    return cls(**kwargs)


def create_model(backend: str, **kwargs) -> BaseModel:
    """创建模型"""
    cls = get_backend(backend)['model']
    return cls(**kwargs)


def create_optimizer(backend: str, **kwargs) -> BaseOptimizer:
    """创建优化器"""
    cls = get_backend(backend)['optimizer']
    return cls(**kwargs)


def create_loss(backend: str, **kwargs) -> BaseLoss:
    """创建损失函数"""
    cls = get_backend(backend)['loss']
    return cls(**kwargs)
```

## 五、风险与注意事项

### 5.1 技术风险

| 风险 | 影响 | 缓解措施 |
|-----|------|---------|
| TF/PyTorch API 差异大 | 实现复杂度高 | 抽象层设计需全面 |
| 权重格式不兼容 | 无法迁移预训练模型 | 提供转换工具 |
| 性能差异 | 训练速度不同 | 保持各后端独立优化 |
| 现有项目兼容性 | 迁移成本 | 默认使用 TF，保持兼容 |

### 5.2 迁移策略

```
阶段 1: 接口抽象 (向后兼容)
├── 定义抽象接口
├── 创建 TensorFlow 适配层
├── ModelEngine 可选使用新接口
└── 现有项目无感知

阶段 2: PyTorch 后端实现
├── 实现 PyTorch 适配层
├── 测试端到端训练流程
└── 验证与 TF 结果一致性

阶段 3: 权重迁移
├── 实现 TF → PT 权重转换
├── 实现 PT → TF 权重转换
└── 提供迁移文档
```
