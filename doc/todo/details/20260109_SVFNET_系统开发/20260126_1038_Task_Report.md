## 2. 迁移方案：PYTHONPATH 修改

### 2.1 核心原理

通过在 `cli.py` 开头添加 `sys.path.insert()`，将 `src/` 目录加入 Python 路径搜索列表，实现所有模块的自动兼容。

### 2.2 cli.py 修改

在 `cli.py` 文件开头（约第 5-10 行）添加：

```python
import sys
import os

# 将 src 目录加入 Python 路径，实现模块兼容
_SCRIPT_DIR = os.path.dirname(os.path.abspath(__file__))
if os.path.join(_SCRIPT_DIR, 'src') not in sys.path:
    sys.path.insert(0, os.path.join(_SCRIPT_DIR, 'src'))
```

### 2.3 方案对比

| 方案 | 优点 | 缺点 | 代码修改量 |
|------|------|------|-----------|
| **PYTHONPATH (本方案)** | 纯Python方案，跨平台兼容 | 子进程需单独设置 | **5行** |
| 符号链接 | 无代码修改 | Windows 权限问题，跨平台差 | 0行 |
| 修改所有导入 | 干净无链接 | 约200+处修改 | ~500行 |

## 4. 子进程兼容性

### 4.1 问题说明

如果代码通过 `subprocess` 调用 Python，需要确保子进程也设置正确的 `PYTHONPATH`。

### 4.2 解决方案

在代码中设置环境变量：

```python
import os
import subprocess

env = os.environ.copy()
env['PYTHONPATH'] = os.path.join(os.path.dirname(__file__), 'src')
subprocess.run(['python', 'script.py'], env=env)
```

或在命令行启动时：

```bash
PYTHONPATH=src python scripts/script.py
```
