# WaveNet5 超参数调优指南 - 控制变量法实践

## 概述

本文档基于 `WNET5_RealAlias_E01-E12` 和 `WNET5_EFF/EFF2` 系列实验的配置分析，总结控制变量法在超参数调优中的应用方法。

## 一、超参数分类

### 1.1 训练参数
| 参数 | 说明 | 典型范围 |
|------|------|----------|
| `epoch_train` | 训练轮数 | 30000-45000 |
| `learning_rate` | 学习率 | 0.005-0.02 |
| `auto_lr_decay_steps` | LR衰减步数 | 1000-2000 |

### 1.2 模型架构参数
| 参数 | 说明 | 典型范围 |
|------|------|----------|
| `kernal_units` | 滤波器核数量 | 3-8 |
| `init_center_freqs` | 初始中心频率 | 根据kernal_units调整 |
| `init_quality_factors` | 初始品质因子 | 1.0-8.0 |

### 1.3 后处理Dense层参数
| 参数 | 说明 | 典型范围 |
|------|------|----------|
| `post_dense_units` | Dense层神经元数 | 8-32 |
| `post_dense_layers` | Dense层数量 | 1-6 |
| `post_dense_activation` | 激活函数 | relu/tanh |

## 二、控制变量法实验设计

### 2.1 基线配置（Baseline）

建议使用以下配置作为基线：

```json
{
    "epoch_train": 30000,
    "learning_rate": 0.02,
    "auto_lr_decay_steps": 1000,
    "kernal_units": 6,
    "model_subcfg": {
        "init_center_freqs": [8, 25, 50, 85, 120, 180],
        "init_quality_factors": [1.5, 2.0, 2.5, 3.0, 4.0, 5.0],
        "post_dense_units": 16,
        "post_dense_layers": 3,
        "post_dense_activation": "relu"
    }
}
```

## 四、实验设计最佳实践

### 4.1 单变量原则

每次实验只改变一个超参数：

```
EFF_A1 (units=16, layers=2) → EFF_A2 (units=8, layers=3)  ❌ 改了两个变量
EFF_C1 (units=8) → EFF_C2 (units=12)  ✓ 只改units
```

### 4.2 命名约定

```
WNET5_[系列]_[组][序号]
         │     │   │
         │     │   └── 序号：同组内的变体 (1,2,3...)
         │     └────── 组别：A/B/C/D表示不同维度
         └──────────── 系列：EFF/EFF2表示迭代轮次
```

### 4.3 配置继承

基于表现好的配置进行微调：

```
EFF_A1 (表现良好)
  └── EFF2_A1 (继承，微调units: 16→14)
      └── EFF2_A1_PS-5_360 (继承，添加inference_config)
```

### 4.4 参数协同调整

当调整`kernal_units`时，需同步调整：
- `init_center_freqs` 数组长度
- `init_quality_factors` 数组长度

```json
// kernal_units=4 时
"init_center_freqs": [15, 40, 80, 140],
"init_quality_factors": [2.0, 2.5, 3.0, 3.5]

// kernal_units=6 时
"init_center_freqs": [8, 25, 50, 85, 120, 180],
"init_quality_factors": [1.5, 2.0, 2.5, 3.0, 4.0, 5.0]
```

## 五、实验矩阵参考

### 5.1 推荐的实验顺序

1. **第一轮**: 固定架构，调整Dense层
   - 变化 `post_dense_units`: 8 → 12 → 16 → 20
   - 变化 `post_dense_layers`: 2 → 3 → 4

2. **第二轮**: 基于最佳Dense配置，调整核数
   - 变化 `kernal_units`: 4 → 5 → 6 → 8

3. **第三轮**: 微调训练参数
   - 变化 `learning_rate`: 0.01 → 0.015 → 0.02
   - 配合调整 `epoch_train` 和 `auto_lr_decay_steps`

4. **第四轮**: 精细调整（可选）
   - 激活函数对比
   - Q值范围调整
   - 中心频率分布调整

## 六、总结

### 核心原则

1. **单一变量**: 每次实验只改变一个超参数
2. **有序探索**: 从粗到细，先架构后训练
3. **继承优化**: 基于好的配置进行微调
4. **完整记录**: 保持config.json的完整性和可追溯性

### 关键超参数影响

| 参数 | 增大影响 | 减小影响 |
|------|----------|----------|
| kernal_units | 更强表征，更多参数 | 更轻量，可能欠拟合 |
| post_dense_units | 更强非线性变换 | 更快训练，可能欠拟合 |
| post_dense_layers | 更深特征提取 | 更简单，训练更稳定 |
| learning_rate | 更快收敛，可能不稳定 | 更稳定，需更多epochs |
