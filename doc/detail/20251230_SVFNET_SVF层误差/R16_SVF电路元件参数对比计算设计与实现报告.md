# R16 SVF电路元件参数对比计算设计与实现报告

## 1. 任务背景
在 SVF 实际制作的电路中，两个 SVF 电路的电容标称值分别是 1.5uF 和 200nF。需要对比以下三个值：
1. 理论计算值（根据 R15 调查得到的计算公式计算出来的值）
2. 标称值（1.5uF 和 200nF）
3. 实测值，从实测 SVF 频率响应的拟合结果中反推出来的电容值（电阻都视为标称值）

该功能需要集成到 `wnet5-circuit-validation` 任务中，并在生成的 `report.md` 中输出对比表格。

## 2. 设计方案

### 2.1 计算公式 (基于 R15)
根据 R15 的调查结果，SVF 电路的元件参数计算公式如下：
- 电容 $C = \frac{1}{2\pi \cdot f_0 \cdot R_{integrator}}$，其中 $R_{integrator} = 10k\Omega$。
- 反馈电阻 $R_6 = (3Q - 1) \cdot R_7$，其中 $R_7 = 10k\Omega$。

### 2.2 实现步骤
1.  **参数获取**:
    - 从项目配置中获取理论 $f_0$ 和 $Q$。
    - 从拟合结果中获取实测（拟合后）的 $f_0$ 和 $Q$。
2.  **反推计算**:
    - 使用上述公式，分别计算理论值和实测值对应的 $C$ 和 $R_6$。
    - 标称值 $C$ 分别设定为 $1.5\mu F$ (SVF1) 和 $200nF$ (SVF2)。
    - 标称值 $R_6$ 视为与理论值一致。
3.  **表格生成**:
    - 计算实测值相对于理论值的百分比误差。
    - 格式化为 Markdown 表格。
4.  **集成**:
    - 在 `WNET5CircuitValidator` 类中新增 `_generate_component_comparison_table` 方法。
    - 在 `execute_validation` 中调用该方法。
    - 修改 `_generate_svf_error_report` 以接收并展示该表格。

## 3. 代码实现

### 3.1 新增 `_generate_component_comparison_table` 方法
在 `visualization/wnet5_circuit_validator.py` 中实现了该方法，负责核心计算逻辑。

### 3.2 修改 `execute_validation`
在 SVF 误差仿真流程中，如果启用了拟合功能，则自动触发元件参数对比计算。

### 3.3 修改 `_generate_svf_error_report`
在报告的“误差因素分析”章节下新增了“3.2.4 SVF 电路元件参数对比 (R16)”小节，展示对比表格。

## 4. 运行结果验证

执行命令：
`python cli.py ep ex_projects/inference/wnet5-circuit-validation/WNET5q1h2u6l3_layer1`

生成的 `report.md` 中包含如下表格：

| 阶段 | 参数 | 理论计算值 | 标称值 | 实测值 (拟合反推) | 误差 (vs 理论) |
| :--- | :--- | :--- | :--- | :--- | :--- |
| SVF1 | 电容 C (uF) | 1.5915 | 1.5000 | 1.3456 | -15.45% |
| SVF1 | 电阻 R6 (kΩ) | 20.00 | 20.00 | 19.81 | -0.94% |
| SVF2 | 电容 C (uF) | 0.1989 | 0.2000 | 0.1892 | -4.89% |
| SVF2 | 电阻 R6 (kΩ) | 20.00 | 20.00 | 19.99 | -0.05% |

### 结果分析
- **SVF1 电容**: 理论值为 1.59uF，标称值为 1.5uF，实测反推值为 1.35uF。实测值相对于理论值偏小约 15.45%。
- **SVF2 电容**: 理论值为 199nF，标称值为 200nF，实测反推值为 189nF。实测值相对于理论值偏小约 4.89%。
- **电阻 R6**: 实测反推值与理论值非常接近，说明 $Q$ 值的偏差较小。

## 5. 结论
成功实现了 SVF 电路元件参数的对比计算与展示功能，为硬件调试提供了直观的数据支持。
