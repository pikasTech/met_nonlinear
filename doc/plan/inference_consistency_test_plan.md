# 推理功能一致性测试计划

## 测试背景
在实施统一电阻计算架构并修复推理功能后，需要验证重构后的代码与基准版本(commit ebef4581)在功能上保持一致。

## 测试目标
1. 验证推理输出数值的一致性
2. 确认信号处理流程的正确性
3. 验证层级处理的完整性
4. 确保性能没有显著退化

## 测试环境
- **基准版本**: commit ebef4581792bafaf0bad0f4ebb55bf3f9f7d10b0
- **当前版本**: master分支 (commit e81236ad及之后)
- **测试项目**: WNET5q1h2u6l3
- **测试参数**: --layers 2 (测试前2层)

## 测试方法

### 1. 环境准备
```bash
# 创建基准版本的git worktree
git worktree add ../baseline_ebef4581 ebef4581

# 复制必要的依赖模块(基准版本缺少calibration_analyzer)
cp -r calibration_analyzer ../baseline_ebef4581/
```

### 2. 推理执行
```bash
# 基准版本推理
cd ../baseline_ebef4581
python cli.py -p WNET5q1h2u6l3 -t i --layers 2 > baseline_inference.log 2>&1

# 当前版本推理
cd ../met_nonlinear_master
python cli.py -p WNET5q1h2u6l3 -t i --layers 2 > current_inference.log 2>&1
```
