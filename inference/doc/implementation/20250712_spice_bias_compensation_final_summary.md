## 三、配置示例

### 3.1 配置文件结构
```json
{
    "inference_config": {
        "bias_compensation": {
            "enabled": true,
            "bias_adjustment_matrix": [0.5, -0.8, 0.3, 0.7, -0.4, 0.6],
            "layer_bias_adjustments": {
                "0": [0.2, -0.3, 0.1],
                "1": [0.5, -0.7, 0.4]
            }
        }
    }
}
```

### 3.2 使用方法
1. 在项目的 `config.json` 中添加 `inference_config`
2. 设置 `enabled: true` 启用补偿
3. 配置 `bias_adjustment_matrix` 或 `layer_bias_adjustments`
4. 正常执行 SPICE 推理，补偿将自动应用

## 四、关键学习点

### 4.1 正确的理解
- **NN 是基准**：神经网络的输出是标准答案，不应修改
- **SPICE 需要校准**：通过调整电路参数来匹配 NN 输出
- **补偿的应用时机**：仅在 SPICE 电路生成时，不影响其他推理路径

### 4.2 实施要点
- 使用临时属性传递补偿值，避免污染模型状态
- 支持灵活的补偿格式，提高易用性
- 充分的日志记录，便于调试和验证
