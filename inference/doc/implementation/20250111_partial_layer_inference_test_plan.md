# 部分层推理功能测试计划

文档创建日期：2025-01-11  
功能名称：部分层推理（Partial Layer Inference）  
测试版本：v1.0

## 1. 测试目标

验证部分层推理功能的正确性、稳定性和性能，确保：
- 命令行参数正确解析
- 各推理后端正确限制层数
- 错误处理机制完善
- 功能向后兼容

## 2. 测试环境

- Python版本：3.9
- TensorFlow版本：2.6
- 测试项目：WNET5q1h2u6l3（5层WaveNet模型）
- 推理后端：NN、SPICE、NumPy

## 3. 测试用例

### 3.1 基本功能测试

#### TC-001：默认全层推理
- **目的**：验证不指定layers参数时，推理所有层
- **步骤**：
  ```bash
  python cli.py -i -f WNET5q1h2u6l3
  ```
- **预期结果**：
  - NN后端推理5层
  - SPICE后端推理5层
  - NumPy后端推理5层

#### TC-002：部分层推理（2层）
- **目的**：验证指定2层时，只推理前2层
- **步骤**：
  ```bash
  python cli.py -i -f --layers 2 WNET5q1h2u6l3
  ```
- **预期结果**：
  - NN后端推理2层
  - SPICE后端推理2层
  - NumPy后端推理2层

#### TC-003：部分层推理（3层）
- **目的**：验证指定3层时，只推理前3层
- **步骤**：
  ```bash
  python cli.py -i -f --layers 3 WNET5q1h2u6l3
  ```
- **预期结果**：
  - 所有后端推理3层

### 3.2 边界条件测试

#### TC-004：推理1层
- **目的**：验证最小层数限制
- **步骤**：
  ```bash
  python cli.py -i -f --layers 1 WNET5q1h2u6l3
  ```
- **预期结果**：只推理第1层

#### TC-005：推理全部层数（5层）
- **目的**：验证指定全部层数时的行为
- **步骤**：
  ```bash
  python cli.py -i -f --layers 5 WNET5q1h2u6l3
  ```
- **预期结果**：推理所有5层（等同于不指定layers）

#### TC-006：超出模型层数
- **目的**：验证请求层数超过模型总层数时的处理
- **步骤**：
  ```bash
  python cli.py -i -f --layers 10 WNET5q1h2u6l3
  ```
- **预期结果**：
  - 显示警告信息
  - 推理所有5层

### 3.3 错误处理测试

#### TC-007：无效层数（0）
- **目的**：验证非正整数输入的错误处理
- **步骤**：
  ```bash
  python cli.py -i --layers 0 WNET5q1h2u6l3
  ```
- **预期结果**：
  - 显示错误信息："--layers 参数必须是正整数"
  - 程序退出

#### TC-008：无效层数（负数）
- **目的**：验证负数输入的错误处理
- **步骤**：
  ```bash
  python cli.py -i --layers -1 WNET5q1h2u6l3
  ```
- **预期结果**：
  - 显示错误信息
  - 程序退出

#### TC-009：非整数输入
- **目的**：验证非整数输入的错误处理
- **步骤**：
  ```bash
  python cli.py -i --layers abc WNET5q1h2u6l3
  ```
- **预期结果**：
  - 显示错误信息："--layers 参数必须是整数"
  - 程序退出

#### TC-010：缺少参数值
- **目的**：验证缺少参数值的错误处理
- **步骤**：
  ```bash
  python cli.py -i --layers WNET5q1h2u6l3
  ```
- **预期结果**：
  - 解析错误，WNET5q1h2u6l3被当作layers的值
  - 显示错误信息

### 3.4 输出验证测试

#### TC-011：输出文件数量验证
- **目的**：验证生成的文件数量与指定层数一致
- **步骤**：
  1. 执行 `python cli.py -i -f --layers 2 WNET5q1h2u6l3`
  2. 检查输出目录文件数量
- **预期结果**：
  - nn_layers/目录下有2个文件
  - spice_layers/目录下有2个文件
  - numpy_layers/目录下有2个文件

#### TC-012：元数据验证
- **目的**：验证推理元数据记录正确
- **步骤**：
  1. 执行部分层推理
  2. 检查inference_metadata.json
- **预期结果**：
  - metadata中记录layers_limit=2
  - 各后端的层数信息正确

### 3.5 性能测试

#### TC-013：推理时间对比
- **目的**：验证部分层推理的性能提升
- **步骤**：
  1. 记录全层推理时间
  2. 记录2层推理时间
  3. 比较时间差异
- **预期结果**：2层推理时间显著少于全层推理

### 3.6 兼容性测试

#### TC-014：与其他参数组合使用
- **目的**：验证与-q（快速模式）组合使用
- **步骤**：
  ```bash
  python cli.py -i -f -q --layers 2 WNET5q1h2u6l3
  ```
- **预期结果**：快速模式和层数限制同时生效

#### TC-015：批量项目推理
- **目的**：验证批量项目时的层数限制
- **步骤**：
  ```bash
  python cli.py -i --layers 2 WNET*
  ```
- **预期结果**：所有匹配项目都应用层数限制

   - 执行TC-001到TC-003

3. **边界和错误测试**（20分钟）
   - 执行TC-004到TC-010

4. **验证测试**（10分钟）
   - 执行TC-011到TC-012

5. **性能测试**（10分钟）
   - 执行TC-013

6. **兼容性测试**（10分钟）
   - 执行TC-014到TC-015

## 5. 通过标准

- 所有测试用例执行结果符合预期
- 无程序崩溃或异常退出（除预期的错误处理）
- 性能提升明显（部分层推理时间 < 全层推理时间）
- 日志信息清晰准确

## 6. 风险评估

- **风险1**：SPICE仿真耗时较长，可能导致测试超时
  - 缓解措施：设置合理的超时时间，使用快速模式测试

- **风险2**：不同模型架构的兼容性
  - 缓解措施：本期仅支持LayeredModelSupport接口的模型
