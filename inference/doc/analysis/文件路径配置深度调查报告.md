## 🚨 核心问题分析

### 问题1: 工作目录路径配置

**实际情况**:
```
文件位置: inference/temp/dataset_input.wave (存在)
代码期望: temp/dataset_input.wave (相对路径)
工作目录: /mnt/f/Work/met_nonlinear/ (项目根目录)
```

**推理阶段**:
```
期望temp/dataset_input.wave → 工作目录不匹配 → 报错
实际inference/temp/dataset_input.wave → 存在但找不到
```

### 问题2: 相对路径配置不一致

系统中不同模块使用不同的路径约定：
- 主程序在项目根目录运行
- 推理模块的数据在`inference/temp/`目录
- 代码使用相对路径`temp/`，但应该是`inference/temp/`

### 问题3: 路径配置缺乏灵活性

推理模块硬编码了输入路径，但：
- 没有配置选项允许用户指定正确路径
- 没有检查多个可能的路径位置
- 缺乏清晰的文档说明文件位置规则
