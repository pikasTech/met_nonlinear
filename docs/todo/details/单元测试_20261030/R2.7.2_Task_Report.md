## 三、根本原因分析

### 3.1 Coverage 配置冲突（根本原因）

**问题位置**: `F:\Work\met_nonlinear_worktrees\met_nonlinear_master\conftest.py` (第149-153行)

```python
# conftest.py 中覆盖了 pytest.ini 的配置
config.addinivalue_line(
    "addopts",
    f"--cov=src.calibration_analyzer --cov-report=term-missing --cov-report=html "
    f"--cov-config={_SRC_DIR / 'tests' / 'calibration_analyzer' / '.coveragerc'}"
)
```

**问题**:
1. `conftest.py` 中的 `--cov=src.calibration_analyzer` **仅覆盖 calibration_analyzer 子模块**
2. 但 `.coveragerc` 中的配置又被 `pytest.ini` 的全局配置覆盖
3. 导致实际统计范围不明确

**对比配置**:

| 配置文件 | source 配置 | 影响 |
|----------|-------------|------|
| `pytest.ini` (第41行) | `source = src` | 全局覆盖 |
| `conftest.py` (第151行) | `--cov=src.calibration_analyzer` | 仅子模块 |
| `.coveragerc` (第2行) | `source = src/calibration_analyzer` | 仅子模块 |

### 3.2 测试文件导入问题

检查发现多个测试文件存在导入问题：

| 测试文件 | 问题 | 影响 |
|----------|------|------|
| `test_project_manager.py` | 24个 ERROR | 集成测试无法运行 |
| `test_cli_parser.py` | 依赖 cli 模块 | 5个跳过测试 |
| `test_wave_generator.py` | 不可用 | 4个跳过测试 |

### 3.3 测试覆盖深度不足

**问题**: 虽然测试文件覆盖了 66.6% 的文件（221/332），但只执行了 27.1% 的代码行。

**原因**:
1. 测试主要集中在简单函数，复杂逻辑未覆盖
2. `project_manager.py` 等集成模块测试不足（38%）
3. `training.py` 覆盖率仅 11%
4. `visualization` 模块未建立有效测试

### 3.4 环境依赖问题

| 模块 | 依赖问题 | 解决方案 |
|------|----------|----------|
| `simulation.py` | 需要 NGspice 环境 | 使用 mock |
| `visualization` | 交互式绘图 | 需要 headless 测试 |
| `analysis` | 无测试用例 | 需要新建 |

## 四、与 R2.7.1 报告的差异分析

### 4.1 覆盖率计算方式差异

| 计算方式 | 覆盖率 | 说明 |
|----------|--------|------|
| R2.7.1 报告 | 37.77% | coverage.py 默认计算（加权平均） |
| 本次分析（行） | 27.1% | 已执行行数/总行数 |
| 本次分析（文件） | 66.6% | 至少执行1行的文件数/总文件数 |

### 4.2 数据可信度验证

通过独立分析 `.coverage` 文件验证：

```python
# 执行 28,829 行代码
# 总代码 106,349 行
# 行覆盖率 = 28829/106349 = 27.1%
```

**结论**: R2.7.1 报告的 37.77% 是 **coverage.py 的加权平均计算**，实际行覆盖率约 **27.1%**。
