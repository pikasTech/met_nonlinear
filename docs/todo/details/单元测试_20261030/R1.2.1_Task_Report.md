## 测试质量指标

### 测试原则
1. **单元隔离**: 每个测试用例只测试一个功能点
2. **Mock 外部依赖**: 使用 Mock 隔离外部依赖（文件系统、网络等）
3. **边界条件覆盖**: 测试正常情况和异常情况
4. **参数验证**: 测试有效参数和无效参数
5. **返回值验证**: 验证函数返回值正确性

### 测试技术
- **Fixture**: 使用 pytest fixture 管理测试数据和状态
- **Mock**: 使用 unittest.mock Mock 和 patch 模拟外部依赖
- **Parametrize**: 使用 pytest.mark.parametrize 进行参数化测试
- **Exception Testing**: 测试异常情况

## 经验总结

1. **Fixture 作用域**: 模块级 fixture 可在所有测试类中共享
2. **Mock 策略**: 需要完整 mock 整个调用链以避免副作用
3. **异常测试**: 使用 pytest.raises 测试异常情况
4. **路径处理**: Windows 路径和 POSIX 路径需要考虑兼容性问题
5. **测试隔离**: 每个测试应该独立运行，不依赖于其他测试的状态
