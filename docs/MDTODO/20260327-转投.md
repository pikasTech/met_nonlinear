## R10

Compute Cost (静态分析) 里面有关键的参数是 add、mul 和 map 的比例，按照 add 为 1 进行归一化。则有 2 个参数需要整定，之前的 6 是一个基础猜测数，需要整定，新的整定方法是：

1. 用 On board 的耗时作为整定的目标，然后在 add、mul、map 的比例空间里面进行搜索，找到一个新的比例，使得 Compute Cost 预测的耗时和 On board 的耗时最接近。

2. 将 add 的耗时系数归一化，则有总耗时为

A * (add + mul * M + map * P)，M 和 P 为比例系数，A 为整体的耗时缩放系数。

3. 要创建一个新的 ex_project 来进行这个参数搜索，config.json 里面配置好参与整定的 `on_board_inference` 的 ep 的路径和 `static_compute_cost` 的 project 的路径，一一配对，然后用一个搜索算法进行搜索并可视化。

完成之后撰写一个新的计算参数静态分析的中间高，参考 R7.x 的写法。然后整合到 R8 的 draft 里面。

完成任务后将详细报告写入[R10](./details/20260327-转投/20260422_211418_Task_Report.md)。

## R11

Freq Drift (Hz) 容易计算出来爆炸的结果，我觉得应该提高这个指标的计算稳定度，之前好像是先 fit 再提取中心频率，我觉得这个方法的 fit 稳定性有问题，才导致 `No positive (stress)` 出现 `3.26e+05` 的数值不稳定结果，我觉得应该把 fit 改为 `Spline` 插值，然后再计算中心频率（频带内的频率最高点），这样计算出的结果会稳定在 (10~128Hz)，而不是搞出来 `3.26e+05` 这种极端不稳定值。
