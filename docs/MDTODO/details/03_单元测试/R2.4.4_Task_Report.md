**低覆盖率原因分析**:

1. **模块架构问题**: `figure_paper.py` 设计为主要通过命令行运行 (`if __name__ == '__main__':`)，大部分代码在主程序块中

2. **外部依赖**: 该模块依赖 `paper.fig_process` 包，该包不属于本项目，需要完整的外部环境

3. **文件 I/O**: `ProjectResult` 类的方法涉及文件读取操作，测试需要模拟或临时文件

4. **测试可及代码**: 测试已覆盖所有可独立测试的 utility 函数:
   - `convert_numpy()`
   - `get_complementary_color()`
   - `my_arraw()`
   - `expand_project_patterns()`
   - `combine_images_with_labels()`
   - `concatenate_images()`
   - `get_model_color()`
   - `ProjectResult._calculate_parameters()`

5. **无法测试的代码**: 以下代码需要完整环境或外部依赖:
   - `ProjectResult.load_data()` - 需要真实 JSON 文件
   - `ProjectResult.process_data()` - 需要完整数据结构
   - `plot_*` 函数 - 需要 `paper.fig_process` 包
   - 主程序块中的代码 - 面向脚本执行
