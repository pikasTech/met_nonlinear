## 测试技术要点

### 1. Mock 外部依赖

由于 `figure_paper` 依赖 `paper.fig_process` 等外部模块，测试开始时创建了完整的Mock对象：

```python
# Mock external dependencies before importing figure_paper
mock_paper = MagicMock()
mock_paper.fig_process = MagicMock()
mock_paper.fig_process.plot_config.MODEL_COLOR_MAP = MOCK_MODEL_COLOR_MAP
sys.modules['paper'] = mock_paper
sys.modules['paper.fig_process'] = mock_paper.fig_process
```

### 2. 动态模块加载

对于 `report_generator` 模块，由于模块路径复杂，使用了 `importlib.util` 直接从文件加载模块：

```python
_MODULE_PATH = os.path.join(_project_root, 'inference', 'tools', 'visualization', 'utils', 'report_generator.py')
spec = importlib.util.spec_from_file_location('report_generator', str(_MODULE_PATH))
_REPORT_GENERATOR_MODULE = importlib.util.module_from_spec(spec)
spec.loader.exec_module(_REPORT_GENERATOR_MODULE)
```

### 3. 临时文件处理

所有涉及文件I/O的测试使用 `tempfile.mkdtemp()` 创建临时目录，并在测试结束后自动清理。
