## 问题根因分析

### 1. 导入路径配置错误

测试文件 `test_mimoiir.py` 原始导入配置:
```python
sys.path.insert(0, os.path.abspath(os.path.join(os.path.dirname(__file__), '..')))
```
这添加的是 `src/tests/` 目录，导致无法正确导入 `experimental.mimoiir` 模块。

**修复**: 需修改为 `.., '..'` 以指向 `src/` 目录。

### 2. 测试用例设计缺陷

- 断言使用错误的默认值 (`trainable` 实际为 `False`)
- 使用了类不支持的参数 (`debug`)
- DIAGIIR 的 `units` 参数与输入通道维度的关系理解有误
- IIR 标准方程测试的精度预期不合理

### 3. 未进行实际验证

R1.7.1 报告仅描述了测试设计，未实际运行验证测试通过率和覆盖率。
