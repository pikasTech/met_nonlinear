## 发现的代码问题

### 1. circuit_dense.py 第 94-96 行

**问题**：标量偏置处理存在边界情况

```python
biases_array = np.array(biases) if not isinstance(biases, np.ndarray) else biases
if np.isscalar(biases_array):
    # 如果是标量，扩展为数组
    self.biases = np.full(self.n_outputs, biases_array)
elif len(biases_array) != self.n_outputs:
    raise ValueError(...)
```

当传入 Python 标量（如 `0.5`）时，`np.array(0.5)` 创建的是 0 维数组，`np.isscalar()` 返回 `False`，导致 `len()` 调用失败。

**影响**：使用标量偏置时会抛出 `TypeError: len() of unsized object`

**建议修复**：
```python
if np.isscalar(biases_array) or (isinstance(biases_array, np.ndarray) and biases_array.ndim == 0):
    # 如果是标量或0维数组，扩展为数组
    self.biases = np.full(self.n_outputs, float(biases_array))
```

### 2. simulation.py 仿真测试依赖 NGspice

**问题**：`CircuitSimulation` 类需要 NGspice 仿真器才能进行实际测试

**影响**：无法在无 NGspice 环境中测试 SPICE 仿真功能

**建议**：创建独立的 mock 测试类，或在 CI 环境中安装 NGspice
