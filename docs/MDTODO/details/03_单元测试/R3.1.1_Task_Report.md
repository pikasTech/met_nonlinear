# R3.1.1 Coverage 配置冲突修复任务报告

## 任务目标

修复 Coverage 配置冲突，统一测量标准，为后续覆盖率提升奠定基础。

## 问题分析

### 1. 初始问题诊断

通过运行测试发现以下问题：
- 运行 `src/tests/core/test_core_modules.py` 时覆盖率仅显示 **7%**
- 大量模块显示 0% 覆盖率
- 子任务报告声称的高覆盖率与实际测量存在巨大差异

### 2. 根本原因分析

**配置冲突**：
- `conftest.py` 中硬编码 `--cov=src` 命令行参数
- `.coveragerc` 中定义了 `source = src`
- 两者可能产生冲突导致覆盖率数据收集不完整

**并行测试问题**：
- `.coveragerc` 中 `parallel = true` 可能导致覆盖率数据丢失

## 修复措施

### 1. 修改 `.coveragerc`

```ini
[run]
source = src
parallel = false  # 原来是 true，禁用并行以确保数据完整
branch = false
concurrency = multiprocessing
dynamic_context = test_function
```

### 2. 修改 `conftest.py`

```python
# 移除硬编码的 --cov=src，让 .coveragerc 的 source 配置生效
config.addinivalue_line(
    "addopts",
    f"--cov --cov-report=term-missing --cov-report=html "
    f"--cov-config={_PROJECT_ROOT / '.coveragerc'}"
)
```
