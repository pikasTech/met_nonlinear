# R5 Task Report

## 任务目标

在 `projects/07_RNN` 中继续执行 RNN 消融调参，约束如下：

- `epoch_train = 1000`
- `loss = MAE`
- 只允许修改 `learning_rate` 与 `model_subcfg`
- 目标区间与 R3 一致：
  - `Freq Drift (Hz)` in `(2.34, 5.0)`
  - `Sens Drift (%)` in `(9.09, 15.0)`
  - `Linearity (%)` in `(0.89, 2.0)`

## 执行说明

本轮没有改动任何训练主流程代码，只在 `projects/07_RNN` 下按规则复制已有 `config.json` 后，逐轮调整 `learning_rate` / `model_subcfg` 并前台执行 `python cli.py -t projects/...`。

已补评估的历史 RNN 结果表明，RNN 家族当前最接近目标的官方 1000 epoch 基线是：

- `projects/07_RNN/RNNu16_e1k_puremae_r15`

因此后续调参都围绕这个方向展开，优先验证：

1. 提高 `lr` 是否继续改善 `Freq Drift` / `Sens Drift`
2. 增减 Dense head 是否有助于压低 `Sens Drift`
3. 调整激活函数是否能改善 100 Hz 灵敏度扩散
4. 去掉 Dense head 是否能回到更稳定的线性读出

## 调参结论

### 1. `lr=1.5e-3` 明显优于 `7e-4`

从 `RNNu16_e1k_puremae_r7` 到 `RNNu16_e1k_puremae_r15`：

- `Val MAE`: `0.012409 -> 0.010006`
- `Freq Drift`: `10.6091 -> 6.6752`
- `Sens Drift`: `26.6126 -> 23.7510`
- `Linearity`: `1.7617 -> 1.0580`

说明在当前 RNN 实现下，`7e-4` 偏小，`1.5e-3` 是更好的固定学习率。

### 2. 加深 / 加宽 Dense head 不能解决 `Sens Drift`

`RNNu16d24l2_e1k_puremae_r15` 虽然保持了还可以的 `Val MAE` 和 `Linearity`，但：

- `Freq Drift` 从 `6.6752` 恶化到 `8.3581`
- `Sens Drift` 仍有 `24.0332%`

说明更强的 Dense head 不是当前 RNN 失配的主要瓶颈。

### 3. 改成 `relu` Dense head 会进一步恶化灵敏度漂移

`RNNu16_e1k_puremae_r15_relu` 的 `Sens Drift = 28.2631%`，明显差于原始 `silu` 头。

因此当前实现下，`relu` 并不能复用 LSTM 的优势。

### 4. 改成线性递归主干也没有救回来

`RNNu16_e1k_puremae_r15_rlin` 的结果更差：

- `Freq Drift = 9.1044`
- `Sens Drift = 32.6675`

说明单纯把 `SimpleRNN` 改成线性动态核，并不能在当前训练框架里逼近目标区间。

### 5. 去掉 Dense head 也不能压住 `Sens Drift`

`RNNr24d0_e1k_puremae_r15` 虽然 `Linearity = 0.9679` 尚可，但：

- `Freq Drift = 9.3000`
- `Sens Drift = 29.1662`

说明“直接线性读出”同样无法解决问题。

## 当前最佳结果

在所有完整 1000 epoch 且符合本任务约束的 RNN 变体中，当前最佳仍然是：

- `projects/07_RNN/RNNu16_e1k_puremae_r15`

对应指标：

- `Val MAE = 0.010006`
- `Val AFMAE = 0.094414`
- `Freq Drift = 6.6752 Hz`
- `Sens Drift = 23.7510 %`
- `Linearity = 1.0580 %`

它满足：

- `Linearity (%)` 已进入目标区间

但仍不满足：

- `Freq Drift (Hz)` 仍高于 `5.0`
- `Sens Drift (%)` 远高于 `15.0`

## 结论

截至本报告写入时，已经在 R5 允许的调参空间内验证了以下方向：

- 学习率：`7e-4 -> 1.5e-3`
- Dense head 加深加宽
- Dense activation 改为 `relu`
- RNN activation 改为 `linear`
- 去掉 Dense head

结论是：

- 在当前 `src/models/base_models.py` 中的 `RNN(SimpleRNN + Dense)` 实现下，
- 仅靠 `learning_rate` 与 `model_subcfg` 调整，
- 尚未把 RNN 指标推进到 R3 目标区间。

因此，`R5` 当前应保持 `in_progress`，不能按“已达标”口径关闭。
