# R2 任务报告：补齐真实 1DCNN / TCN 模型支持

**任务**: `projects/05_1DCNN` 与 `projects/06_TCN` 当前底层仍在调用 WaveNet 变体，需要补齐真正的 1DCNN / TCN 模型支持。  
**日期**: 2026-04-13  
**状态**: 已完成

---

## 1. 问题确认

我先检查了两组项目配置：

- `projects/05_1DCNN/*/config.json` 的 `use_model` 实际都写成了 `WaveNet3`
- `projects/06_TCN/*/config.json` 的 `use_model` 实际都写成了 `WaveNet2`
- `src/core/model_engine.py` 里也没有真实的 `1DCNN` / `TCN` 分支
- 结果是这两组“消融”项目虽然目录名叫 1DCNN / TCN，但训练时走的仍然是 WaveNet 路径，语义不成立

结论：问题不是项目命名，而是模型工厂层和项目配置层都没有真实接入对应架构。

---

## 2. 实现内容

### 2.1 新增真实 1DCNN

在 `src/models/conv_models.py` 中新增 `OneDCNN`：

- 使用真正的 `Conv1D` 因果卷积栈，而不是 WaveNet3 的简化卷积路径
- 支持：
  - `init_conv_units`
  - `kernel_size`
  - `conv_layers`
  - `dropout_rate`
  - `post_dense`
  - `post_dense_units`
  - `post_dense_layers`
  - `post_dense_activation`
  - `final_activation`
- 模型对外的 `model_name` 设为 `1DCNN`

当前结构语义：

1. 输入 `(batch, time, 1)`
2. 1 个初始因果卷积层（通道数 `init_conv_units`）
3. 若 `conv_layers > 1`，后续使用 `kernel_units` 通道的因果卷积层
4. 可选 1x1 `post_dense` 栈
5. 1x1 输出卷积到单通道

### 2.2 新增真实 TCN

在 `src/models/conv_models.py` 中新增 `TCN`：

- 使用真正的膨胀卷积残差块，而不是 WaveNet2 的 gated residual block
- 每个 temporal block 采用两层 `Conv1D(dilation_rate=...)` + dropout + residual add
- 支持：
  - `dilations`
  - `kernel_size`
  - `init_conv_units`
  - `use_residual`
  - `dropout_rate`
  - `block_dense`
  - `block_dense_units`
  - `post_dense`
  - `post_dense_units`
  - `post_dense_layers`
  - `post_dense_activation`
  - `final_activation`
  - `skip_output_conv`
- 对 WaveNet 专属语义显式拒绝：
  - `use_gating=True`
  - `use_parallel_blocks=True`
  - `combine_blocks_by_add=True`

这样做的目的，是避免用户误把 WaveNet 参数继续带入 TCN 而不自知。

### 2.3 接入模型工厂

更新了：

- `src/models/__init__.py`
- `src/core/model_engine.py`

新增支持：

- `use_model = "1DCNN"` 或 `use_model = "OneDCNN"` -> 构建 `OneDCNN`
- `use_model = "TCN"` -> 构建 `TCN`

这样项目配置终于可以直接表达真实架构，而不是继续借道 WaveNet。

## 3. 项目配置处理

为了避免把历史 WaveNet 训练产物和新模型语义混在一起，我**没有直接改旧目录里已经产生产物的历史项目**，而是基于现有同类配置复制出干净变体：

- `projects/05_1DCNN/1DCNNc4u8k20l8_e1k_lr18e3_pd8l8_true/config.json`
- `projects/06_TCN/TCNc4d1248k3_nopd_true_e1k_lr2e3/config.json`

处理方式遵守项目规则：

- 先复制同类型已有 `config.json`
- 再只替换 `use_model`
- 其他超参数保持不变

替换结果：

- 1DCNN 变体：`WaveNet3` -> `1DCNN`
- TCN 变体：`WaveNet2` -> `TCN`

这样后续如果要继续训练或评估真实架构，可以直接从这两个新变体开始，不会污染旧产物。
