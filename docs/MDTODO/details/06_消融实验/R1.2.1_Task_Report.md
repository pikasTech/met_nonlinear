## 2. 实现内容

### 2.1 新增文件

#### `src/visualization/ablation_study.py`

新增消融实验分析器 `AblationStudyAnalyzer` 类，核心功能：

- `SimpleProjectResult`: 简化版项目数据加载器（避免 broken imports）
- `AblationStudyAnalyzer`: 消融实验分析器
  - `load_all_projects()`: 加载所有配置的项目
  - `calculate_natural_frequency_drift()`: 计算固有频率漂移
  - `calculate_sensitivity_drift()`: 计算灵敏度漂移
  - `calculate_linearity()`: 计算线性度
  - `calculate_suppression_rate()`: 计算抑制率
  - `run_analysis()`: 执行完整分析
  - `generate_markdown_report()`: 生成 Markdown 报告
  - `save_results()`: 保存结果到 JSON 和 Markdown

**复用组件**:
- `calculate_drift()` from `visualization.image_data_process` - 漂移计算
- `linearity_by_frequency.json` - R² 线性度数据

### 2.2 修改文件

#### `src/core/external_cli_handler.py`

1. 在模板创建函数中添加 `_create_ablation_study_template()` 函数
2. 在任务执行函数中添加 `_execute_ablation_study_task()` 函数
3. 在 `create_external_template()` 中添加 `ablation-study` 类型处理
4. 在 `_execute_task()` 中添加 `ablation-study` 和 `compare` 类型处理
5. 在 `_validate_config()` 中跳过 `ablation-study` 和 `compare` 类型的验证

#### `src/core/external_path_parser.py`

1. 在 `SUPPORTED_TASK_TYPES` 中添加 `'ablation-study'` 和 `'compare'`
2. 在 `_parse_universal_path()` 中添加 `ex_projects/{task_type}/{task_name}` 格式处理
3. 添加 `compare/{task_name}` 短格式自动转换支持

## 5. 使用说明

### 5.1 创建新的消融实验

1. 在 `ex_projects/compare/` 下创建新目录
2. 创建 `config.json` 配置文件
3. 运行 `python cli.py ep "compare/新目录名"`

### 5.2 配置文件格式

```json
{
  "task_info": {"task_type": "ablation-study", "description": "描述"},
  "projects": [
    {"name": "项目名", "label": "显示标签", "path": "projects/下的路径"}
  ],
  "metrics": {
    "natural_frequency_drift": {"enabled": true, "reference": "参考项目"},
    "sensitivity_drift": {"enabled": true, "reference": "参考项目", "frequency_hz": 100},
    "linearity": {"enabled": true}
  }
}
```
