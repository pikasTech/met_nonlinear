# R5.1 Task Report

## 任务目标

在 `projects/07_RNN` 中继续对 Plain RNN 做调参，继续遵守 `R5` 约束：

- `epoch_train = 1000`
- `loss = MAE`
- 只允许调整 `learning_rate` 和 `model_subcfg`
- 目标区间与 `R3` 一致：
  - `Freq Drift (Hz)` in `(2.34, 5.0)`
  - `Sens Drift (%)` in `(9.09, 15.0)`
  - `Linearity (%)` in `(0.89, 2.0)`
- 额外约束：`Compute Cost` 控制在 `2000 +- 500`，即优先落在 `[1500, 2500]`

## 本轮执行策略

上一轮 `R5` 已经说明：

- `u16/d16` 基线 (`RNNu16_e1k_puremae_r15`) 是当前 Plain RNN 最好的完整 1000 epoch 结果；
- 但它的 `Compute Cost = 1280`，低于 `R5.1` 要求的计算量窗口；
- 因此本轮不再继续试 `compute < 1500` 的项目，而是只验证落在目标计算量窗口内的候选；
- 每次只训练一个项目，训完后再看指标决定下一步。

基于已有趋势，本轮最终验证了三个方向：

1. **只加大 Dense head，不改 recurrent 宽度**：`u16/d32`
2. **保持 RNN / Dense 等宽，同时整体加宽**：`u20/d20`
3. **比 `u20/d20` 更温和的等宽放大**：`u18/d18`

其中：

- `u16/d32` 与 `u20/d20` 的 `Compute Cost` 都是 `1920`
- `u18/d18` 的 `Compute Cost` 是 `1584`

三者都位于 `R5.1` 要求的 `[1500, 2500]` 窗口内。

- **把窗口内候选从 `1920` 再往下收缩到 `1584`，并没有出现“更稳”的迹象；**
- Plain RNN 在这个任务上不是单纯的“算力不够”问题，而更像是结构能力本身不足。

## 计算量窗口内的结论

截至本轮执行结束，可以确认：

1. **`compute` 落在 `[1500, 2500]` 的 Plain RNN 候选，已经验证过三种代表性方向：**
   - `u16/d32`：只放大 Dense head
   - `u20/d20`：更激进的等宽放大
   - `u18/d18`：更温和的等宽放大
2. **这三个方向都没有把指标推进到目标区间**；
3. **在 `[1500, 2500]` 计算量窗口内，当前没有一个完整 1000 epoch Plain RNN 结果优于 `u16/d16` 基线。**

这意味着：

- 仅在当前 `SimpleRNN + Dense` 结构里，通过 `learning_rate` 与 `model_subcfg` 做轻量宽度调整，
- 还不能把 `Freq Drift` 和 `Sens Drift` 推到 `R3/R5` 的目标区间；
- Plain RNN 在这个任务上的能力上限，仍明显落后于已经纳入横评的 LSTM / 1DCNN / TCN 等方向。

## 本轮最重要结论

`R5.1` 的额外要求是把 `Compute Cost` 推到 `2000 +- 500` 再继续调参。本轮已经满足这个要求并完成了三次完整试验，结论是：

- **Plain RNN 在目标计算量窗口内仍然无法达标；**
- **当前最优完整 Plain RNN 仍是 `RNNu16_e1k_puremae_r15`，但它的 `Compute Cost = 1280`，低于 `R5.1` 窗口；**
- **窗口内不存在一个对 `Freq Drift / Sens Drift / Linearity` 三项同时更优的明确候选。**

因此，本轮可以继续维持 `R5.1` 结束结论，但结论不是“已达标”，而是：

- **已完成窗口内继续调参；**
- **结论为 Plain RNN 在当前约束下仍未达标。**

## 追加结论

连续多轮自主调参后，目前 Plain RNN 在窗口内已经形成很清楚的趋势：

1. **等宽放大（`u18/d18`, `u20/d20`）**
   - 能进入窗口；
   - 但 `Freq Drift` 和 `Sens Drift` 仍远离目标。

2. **只放大 recurrent（`r19/d16`）**
   - 能把 `Freq Drift` 压到窗口内目前最低；
   - 但 `Sens Drift` 会明显爆掉。

3. **只放大 Dense head（`u16/d24`, `u16/d32`）**
   - 对三项物理指标几乎都是系统性恶化；
   - 这条分支已经没有继续挖的价值。

   - 反而把 `Val MAE / Val AFMAE / Linearity` 一起拉坏。

## 新增阶段性结论

结合本轮新增试验，可以把当前 Plain RNN 的窗口内候选分成几类：

1. **单层基线族 (`u16/u18/u20`)**
   - 相对更稳；

2. **大 recurrent 主干 (`r24` 系列)**
   - `Freq Drift` 往往更低；

3. **双层小宽度 (`u12r2`)**

   - 但物理指标没有优于单层家族。

4. **Dense head 扩大或缩小 (`d4/d8/d24/d32`)**
   - 无论往大加还是往小减，都没有带来预期的 `Sens Drift` 改善；
   - Plain RNN 的问题并不是“head 尺寸没调对”这么简单。
