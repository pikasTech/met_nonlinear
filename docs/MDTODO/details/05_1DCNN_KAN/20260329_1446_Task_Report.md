# R3.2 调查报告：CNNKAN 中 use_fast_model 是否真正生效

## 调查结论

**CNNKAN 的 use_fast_model 存储但从未使用，不生效。**

---

## 详细分析

### 1. use_fast_model 的存储（第 934 行）

```python
self.use_fast_model = use_fast_model
```

CNNKAN 在 `__init__` 中确实存储了 `use_fast_model` 参数值。

### 2. fast_model 的构建（第 976-979 行）

```python
if self.use_fast_model:
    self.fast_model = tf.keras.Model(
        inputs=fast_input, outputs=fast_output, name='fast_CNNKAN')
    self.fast_model.build(input_shape=(None, None, self.cnn_filters))
```

fast_model 被创建，但后续从未被调用。

### 3. 被重写的方法全部直接使用 self.model

| 方法 | CNNKAN 实现 | 说明 |
|------|-------------|------|
| `fit()` (1008-1015行) | `return self.model.fit(*args, **kwargs)` | 直接使用 model，不检查 use_fast_model |
| `evaluate()` (999-1006行) | `return self.model.evaluate(*args, **kwargs)` | 直接使用 model |
| `predict()` (1026-1034行) | `return self.model.predict(...)` | 直接使用 model |
| `save_weights()` (1017-1024行) | `return self.model.save_weights(...)` | 直接使用 model |

### 4. 与 BaseModel 的对比

BaseModel 中 use_fast_model 的正确用法：

```python
# fit 方法 (base_models.py 388-402行)
if self.use_fast_model:
    iir_features = self.fast_iir(args[0])  # 用 fast_iir 预处理
    # ...
    history = self.fast_model.fit(*args, **kwargs)  # 用 fast_model 训练
else:
    history = self.model.fit(*args, **kwargs)

# evaluate 方法 (base_models.py 406-414行)
if 'fast_model' in dir(self):
    print('Using fast model to evaluate')
    iir_features = self.fast_iir(args[0])
    return self.fast_model.evaluate(*args, **kwargs)
```

CNNKAN 重写这些方法时，**完全忽略了 use_fast_model 切换逻辑**。

### 5. 原始注释说明（开发者意图）

代码注释已解释原因（第 1003-1004, 1012-1013, 1021-1022, 1030-1031 行）：

> 对于 CNNKAN，直接使用完整模型评估/训练/保存/预测，因为 fast_model 期望的输入（CNN输出，8通道）与原始输入（1通道）不同

---

## 开启与不开启的区别

**没有任何区别。** 无论 use_fast_model=True 还是 False，CNNKAN 的行为完全一致：

- 模型构建时都会创建 fast_model（当 use_fast_model=True 时）
- 所有训练/评估/预测都使用 self.model
- use_fast_model 值仅被存储，从未被读取使用

---

## 根本原因

CNNKAN 的架构与 RNN/LSTM/IIR 不同：
- **RNN 类**：IIR 预处理后的特征作为 fast_model 的输入
- **CNNKAN**：Conv1D 本身就是模型的一部分，无法像 IIR 那样"分离"

开发者可能意图保留 fast_model 接口以保持代码一致性，但实际未实现切换逻辑。

---

## 建议

如果确实需要 use_fast_model 切换功能，可以考虑：
1. 将 Conv1D 层作为预处理步骤分离
2. 实现特征缓存机制
3. 或者完全移除 use_fast_model 相关代码以避免混淆
