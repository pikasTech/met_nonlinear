# R2.6.1 任务报告

## 任务概述

验证 CNNKANh8u6l6 项目的 CLI 训练功能（`python cli.py -t CNNKANh8u6l6`）

## 发现的问题

### 问题 1：CNNKAN 架构与 BaseModel 不兼容

**现象**：CNNKAN 的 `fast_model`（名为 `fast_CNNKAN`）期望的输入是 CNN 输出（8 通道），而 `model`（名为 `CNNKAN`）的输入是原始数据（1 通道）。父类 `BaseModel` 的多个方法假设 `fast_iir` 处理后的输出可以直接传给 `fast_model`。

**影响的方法**：
- `fit()`: 直接使用 `fast_iir` 预处理数据
- `evaluate()`: 使用 `fast_iir` 预处理数据
- `save_weights()`: 尝试从 `fast_model` 复制权重到 `model`
- `predict()`: 使用 `fast_iir` 预处理后传给 `fast_model`

**错误信息**：
```
ValueError: Layer weight shape (3, 1, 8) not compatible with provided weight shape (3, 8, 8)
```

### 问题 2：缺少 CNN 配置字段

**现象**：`CNNKAN` 需要 `CNN_FILTERS`, `CNN_KERNEL_SIZE`, `CNN_DROPOUT_RATE` 配置，但 `src/config.py` 和 `projects/CNNKANh8u6l6/config.json` 中均缺失。

## 解决方案

### 1. 添加 CNN 配置字段

**文件**: `src/config.py`
- 添加 `CNN_FILTERS = 8`
- 添加 `CNN_KERNEL_SIZE = 3`
- 添加 `CNN_DROPOUT_RATE = 0.2`

**文件**: `projects/CNNKANh8u6l6/config.json`
- 添加相同三个配置字段

### 2. 为 CNNKAN 添加兼容性方法

**文件**: `src/models/frikan_models.py`（CNNKAN 类）

| 方法 | 解决方案 |
|------|----------|
| `__init__()` | 添加 `self.fast_iir = self.fast_cnn` |
| `evaluate()` | 直接调用 `self.model.evaluate()` |
| `fit()` | 直接调用 `self.model.fit()` |
| `save_weights()` | 直接调用 `self.model.save_weights()` |
| `predict()` | 直接调用 `self.model.predict()` |
