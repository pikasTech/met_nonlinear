# R3.1.3 任务报告：修复配置项，将 CNN_STRIDES 改为 CNN_KERNEL_SIZE

## 任务目标

将 Conv1D 卷积核的时间步数控制从 `CNN_STRIDES`（控制 strides 参数）改为 `CNN_KERNEL_SIZE`（控制 kernel_size 参数）。原始设计中 `CNN_STRIDES` 用于控制卷积的步长，但实际上 Conv1D 的 `strides` 参数只影响输出的时间步数下采样，而 `kernel_size` 才是控制卷积核覆盖的时间步数（即"感受野"大小）的正确参数。

## 技术说明

### kernel_size vs strides 在 Conv1D 中的作用

- **`kernel_size`**：卷积核的时间步数长度，决定了卷积操作每次考虑多少个时间点的信息。例如 `kernel_size=3` 表示卷积核每次查看 3 个连续时间步。
- **`strides`**：卷积步长，决定了卷积核每次移动多少个时间点。如果 `strides=1`，时间步数不变；如果 `strides=2`，时间步数会减半（下采样）。

原始设计使用 `CNN_STRIDES` 来控制"时间步数"，但这是错误的语义。更改 `strides` 主要影响下采样率，而不是卷积核的感受野大小。正确的控制卷积核覆盖时间步数的参数是 `kernel_size`。
