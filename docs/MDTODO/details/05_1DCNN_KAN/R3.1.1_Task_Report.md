# R3.1.1 任务报告：config.json 增加 1DCNN 时间步配置项

## 任务目标

在 `config.json` 中增加配置项来调整 1DCNN 的卷积时间步数。

## 实施方案

通过添加 `CNN_STRIDES` 配置项来控制 Conv1D 层的 strides 参数，从而调整卷积的时间步数。

## 修改文件

### 1. `projects/CNNKANh8u6l6/config.json`

添加 `CNN_STRIDES` 配置项，默认值为 1：

```json
"CNN_STRIDES": 1,
```

位置：在 `CNN_KERNEL_SIZE` 之后，`CNN_DROPOUT_RATE` 之前。

### 2. `src/models/frikan_models.py`

在 `CNNKAN` 类中添加 `cnn_strides` 参数：

- **__init__ 方法** (line 848)：新增参数 `cnn_strides=1`
- **属性赋值** (line 883)：`self.cnn_strides = cnn_strides`
- **Conv1D 层初始化** (line 884-900)：
  - `self.cnn` 的 `strides` 参数从固定值 `1` 改为 `cnn_strides`
  - `self.fast_cnn` 的 `strides` 参数从固定值 `1` 改为 `cnn_strides`

### 3. `src/core/model_engine.py`

在初始化 CNNKAN 时传递 `cnn_strides` 参数 (line 416)：

```python
cnn_strides=self.config.CNN_STRIDES,
```

## 配置说明

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `CNN_STRIDES` | int | 1 | Conv1D 卷积步长，控制时间步数变化 |

### CNN_STRIDES 的作用

- `CNN_STRIDES = 1`：时间步数保持不变（标准卷积）
- `CNN_STRIDES > 1`：时间步数会按比例减少（下采样效果）

当 `padding='same'` 时，输出时间步数 ≈ ceil(输入时间步数 / strides)。

## 验证

1. config.json 语法正确，包含新增的 `CNN_STRIDES` 配置项

3. model_engine.py 正确从 config 读取并传递 `cnn_strides` 参数
