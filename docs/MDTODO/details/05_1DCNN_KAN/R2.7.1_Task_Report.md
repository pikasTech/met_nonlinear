## 发现的问题及修复

### 问题1：Unicode 编码错误
**现象**：Windows 环境下 GBK 编码无法处理 `R²` 字符，导致 `UnicodeEncodeError`

**解决方案**：
- 执行命令前设置环境变量 `PYTHONIOENCODING=utf-8`
- 使用 cmd.exe 执行：`cmd /c 'set PYTHONIOENCODING=utf-8 && ...'`

### 问题2：use_best_val_weights 权重文件缺失
**现象**：`fast_best_val.weights.h5` 文件不存在，导致加载失败

**原因**：`use_fast_model=True` 时会尝试加载 `fast_best_val.weights.h5`，但该文件不存在

**临时解决方案**：将 `config.json` 中 `use_best_val_weights` 设为 `false`

### 问题3：CNNKAN.predict() 不支持 use_scaler 参数
**现象**：`predict() got an unexpected keyword argument 'use_scaler'`

**原因**：`model_engine.predict_features()` 调用时传递了 `use_scaler=False`，但 `CNNKAN` 模型的 `predict` 方法不识别该参数

**修复**：在 `frikan_models.py` 的 `CNNKAN.predict()` 方法中添加：
```python
kwargs.pop('use_scaler', None)
```

## 结论
CLI 评估功能基本可用，但存在以下待改进点：
1. Windows 环境下需要设置 UTF-8 编码
2. 缺少 `fast_best_val.weights.h5` 文件（训练不完整或权重保存逻辑问题）
3. CNNKAN 模型的 predict 方法需要处理 use_scaler 参数（已修复）
