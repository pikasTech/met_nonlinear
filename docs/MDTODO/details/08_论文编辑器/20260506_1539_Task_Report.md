# R1 任务报告

## 任务目标

在现有 WebUI 中实现一个基于 TypeScript + React 的 LaTeX 论文快速预览编辑器，覆盖以下能力：

- main.tex 的源码编辑与实时预览
- LaTeX 到 Markdown 的快速转换预览
- 数学公式渲染与简单宏替换
- 解析 values.tex 等导入文件
- 图片展示与缺图占位
- 左侧大纲导航
- 左右分屏与滚动同步
- 性能分析面板
- Playwright 端到端测试
- 前后端运行时统一使用 bun

## 实现结果

新增服务端模块：

- src/webui/server/src/paperEditor.ts

- 读取 docs/paper/latex/main.tex
- 递归解析 \input、\include、\InputIfFileExists

## 关键问题与修复

### 1. 前端 api.ts 与 /api 代理冲突

问题：

- 前端模块命名为 src/webui/src/api.ts
- Vite dev proxy 使用 /api 前缀
- 浏览器请求 /api.ts 时被错误代理到后端，返回 HTML，导致模块加载失败

修复：

- 新增 src/webui/src/clientApi.ts
- 所有前端引用切换到 clientApi.ts
- 保留后端 /api 代理不变

### 2. main.tex 当前工作区缺少部分 png 产物

问题：

- 论文中多个 \includegraphics 路径对应的 png 目前不存在
- 当前工作区中只有 raw.json、pixel_regression.json、task_metadata.json 等文件

修复：

- 编辑器对缺图输出明确诊断
- 预览区用可视化缺图占位卡片代替空白失败
- 保持一旦 png 存在即可直接显示真实图片

## 当前限制

- LaTeX 解析仍是“快速预览”级别，不追求完整 TeX 语义兼容
- 复杂表格、复杂环境与高级宏目前采用降级展示或省略提示
- 当前 main.tex 中多数图片资源在工作区中尚未生成 png，因此预览里主要看到缺图占位卡片
