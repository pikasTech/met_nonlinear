# R1 任务报告：6000 词限制下的 Supplement 转移计划

## 1. 任务范围

本任务执行 `docs/MDTODO/20260531-MN最后冻结投稿.md` 中的 R1：在不继续改动论文实质内容的前提下，制定一个面向 MN Article 约 6000 词限制的主文压缩与 Supplement 转移计划。

本轮计划的核心边界如下：

- 不新增实验。
- 不替换当前 canonical 结果。
- 不把降学习率重训探索结果纳入本轮投稿。
- 不改变指标定义、数值结论、核心贡献和主模型叙事。
- 只做主文与 Supplement 之间的证据分流、轻量压缩和投稿格式收口。

特别强调：当前论文源文件仍以 `\CNEN{中文}{英文}` 作为中文中间稿与英文稿的双语载体。后续任何“转移到 Supplement”的动作，都必须以完整 `\CNEN{中文}{英文}` 宏为迁移单位，不能只迁移英文渲染文本，也不能丢弃中文槽位。

## 3. 总体策略

最快、风险最低的路线不是重写论文，而是“主文保留证据骨架，Supplement 承接推导和细节”。

建议第一轮冻结目标：

- 主文压缩到约 6500--7000 词。
- 主文 display items 控制到约 8 个。
- Supplement 保留所有被移走的公式、图表、参数说明和长解释。
- 所有移动内容保留完整 `\CNEN{中文}{英文}` 结构。

这一路线的好处是：格式风险显著下降，同时不重新打开实验、指标和结论。

## 4. `\CNEN` 迁移硬规则

后续执行正文压缩和 Supplement 分流时，必须遵守以下规则：

- 迁移正文段落时，必须整体移动完整的 `\CNEN{中文}{英文}` 宏。
- 如果一个段落的中文槽和英文槽跨多行，移动时必须包含完整的大括号范围，不得只复制当前英文 PDF 中看到的文字。
- 迁移 figure/table 周边说明时，必须同时移动相关的 `\CNEN` 正文引导语、caption、label、引用说明和必要公式。
- 如果主文中需要保留一句桥接语，也应继续写成 `\CNEN{中文桥接语}{English bridge sentence}`。
- Supplement 文件也应沿用同一套 `\CNEN` 宏定义或从主稿继承宏定义，确保中文中间稿和英文投稿稿可以同步渲染。
- 严禁把 Supplement 做成纯英文摘录，因为这样会破坏当前中文中间稿的长期维护能力。
- 严禁只移动 `\includegraphics` 或公式环境，而把解释它的双语 `\CNEN` 段落留在主文或丢失。

建议实现方式：

- 先创建 `docs/paper/latex/supplement.tex`，复用主稿的 `\CNEN` 宏。
- 主文和 Supplement 均通过 `\UseCNENEnglish` 渲染投稿英文版本。
- 若后续需要中文内部审阅，只切换为 `\UseCNENChinese`，主文和 Supplement 都能得到中文稿。
