## 本轮静态迁移与修复

### 1. legacy fallback 已从 entrypoint 主路径移除

本轮将 `src/core/board_inference/entrypoints.py` 从“native 优先，未覆盖则 fallback 到 legacy”改为“仅允许 native executor”，不再从主入口回退到 `legacy_adapter -> src/core/lstm_qemu_ep_task.py`。

- `src/core/board_inference/entrypoints.py`
  - 移除了 `run_legacy_qemu_task` / `run_legacy_keil_bench_task` 主路径 fallback
  - 对未注册 native executor 的模型直接报错并返回失败

### 2. 新增非 FRIKAN native 模块

新增：
- `src/core/board_inference/models/sequence.py`

该模块将旧单体 `src/core/lstm_qemu_ep_task.py` 中非 FRIKAN 的模型规格加载、TF 参考推理调试输出采集、QEMU C 工程生成、Keil benchmark 执行逻辑迁移到了新架构，覆盖：

- `lstm`
- `grn`
- `lstm_transformer`
- `onedcnn`
- `tcn`
- `wavenet2`
- `wavenet3`

迁移方式：
- 平台/比较/串口/Keil/QEMU 共用逻辑继续复用 `src/core/board_inference/platforms/benchmark_common.py`
- 模型相关逻辑在 `sequence.py` 内原生化，避免运行时导入旧单体模块
- `sequence.py` 内自带本地 `_detect_model_type()` / `_load_project_config()`，避免与 `registry.py` 形成循环依赖

### 3. native registry 已覆盖全部旧模型类型

更新：
- `src/core/board_inference/models/__init__.py`

结果：
- `frikan` 继续走既有 `frikan.py`
- 其余 sequence / conv / wavenet 模型统一注册到 `sequence.py`
- `NATIVE_QEMU_EXECUTORS` / `NATIVE_KEIL_BENCH_EXECUTORS` 现在覆盖：
  - `frikan`
  - `lstm`
  - `grn`
  - `lstm_transformer`
  - `onedcnn`
  - `tcn`
  - `wavenet2`
  - `wavenet3`

### 4. 静态迁移后发现并修复的新增问题

#### 4.1 native Keil 路径缺少 `KEIL_BENCHMARK_BASE_UVPROJX`

现象：
- `sequence.py` 初版在独立 Keil native 验证时，非 FRIKAN 模型会因 `KEIL_BENCHMARK_BASE_UVPROJX` 未定义而在生成 `keil_project_file` 时失败。

修复：
- `src/core/board_inference/models/sequence.py`
  - 显式复用 `benchmark_common.KEIL_BENCHMARK_BASE_UVPROJX`
