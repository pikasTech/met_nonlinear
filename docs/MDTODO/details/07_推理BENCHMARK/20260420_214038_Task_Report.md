2. 真机成功跑通并输出与历史 QEMU 完全一致的最终输出值；
3. 真机与 QEMU 的性能计时源不同（`dwt` vs `host_elapsed`），当前主机环境下无法做同量纲直接比较；
4. 历史 QEMU 的 MAE 已有完整离线产物，但真机侧完整 validation 波形尚未通过当前串口抓取链路稳定回收。

换言之：**功能实现与板端运行已经完成，结果一致性的主链路已经建立，但“完全一致”的最终证据仍受当前主机 QEMU 工具链缺失和串口超长 validation 输出截断的限制。**
