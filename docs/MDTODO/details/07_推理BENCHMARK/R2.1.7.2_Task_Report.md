### 2. 静态对比发现并修复的问题

#### 2.1 非训练任务误改 `training_state.json.timestamp`

现象：只要构建模型或执行离线任务，`projects/*/data/training_state.json` 的 `timestamp` 就会被刷新，即使根本没有真实训练发生。

根因：
- `src/core/model_engine.py` 在 `build_model()` 后总会执行 `self.state_manager['model_name'] = self.model_comp.model_name`
- 旧 `TrainingStateManager.update_state()/save_state()` 在“同值写入”时也会落盘并更新时间戳

修复：
- `src/core/training_state.py`
  - `update_state()` 变更为 no-op 不写盘
  - 仅真实状态变化时才刷新 `timestamp`
  - 初始化新 `training_state.json` 时不再二次改写时间戳
  - 读写统一显式 UTF-8

#### 2.2 非训练 `load_weights()` 重写 canonical `*.weights.json`

现象：离线任务/对比任务在只加载 `.h5` 权重时，会把已有 `best_val.weights.json` 重新导出，导致权重名从 canonical 形式被改写为当前 Keras 进程层自动编号，例如：

- `lstm/lstm_cell/kernel:0` -> `lstm/lstm_cell_1/kernel:0`
- `gru/gru_cell/kernel:0` -> `gru_1/gru_cell_1/kernel:0`
- `dense/kernel:0` -> `dense_2/kernel:0`

根因：
- `src/models/base_models.py` / `src/models/wavenet_models.py` 的 `load_weights()` 会在加载后立即 `save_weights_json()`，把运行时 `weight.name` 写回仓库权重 JSON

修复：
- `src/models/base_models.py`
- `src/models/wavenet_models.py`
  - 新增“仅在 JSON 缺失时才补导出”的 `_ensure_weights_json()`
  - `load_weights()` 不再覆盖已有 canonical `*.weights.json`
  - JSON 写入统一显式 UTF-8

#### 2.3 `debug_cli compare` 把调试副产物误判为行为差异

现象：旧侧为保留 stdout/stderr 新增的：
- `legacy_cli_stdout.log`
- `legacy_cli_stderr.log`
- `legacy_cli_process.json`

会被 compare 当成 left-only artifact，导致本来行为一致的模型被误报 `artifact_mismatch`。

修复：
- `src/core/board_inference/comparison.py` 将这 3 个调试副产物加入忽略列表

#### 2.4 Keil compare 中 summary / serial 噪声导致误报

现象：`keil_benchmark_summary.json` 中混入大量运行时噪声，导致 `GRN/LSTMTransformer/LSTM/TCN` 在首次 keil compare 时均被误判为 `artifact_mismatch`。噪声主要包括：
- Keil job `created_at/started_at/finished_at`
- `log_file/state_file/_log_file`
- `log_excerpt` 中 `Flash Load finished at HH:MM:SS`
- serial capture 的 `count/totalCount/stream_length/record_count/timeRange.end`
- 串口前导噪声里重复出现 `*_BENCHMARK_VALIDATION`，导致 `TCN` 的 stream compare 抓到了错误起点

修复：
- `src/core/board_inference/comparison.py`
  - alias 替换按路径长度倒序，避免 `C:\work\...\__EP_ROOT__` 这种半替换状态
  - `keil_benchmark_summary.json` 归一化新增 Keil job 元数据与 serial capture 计数/时间字段的归一化
  - benchmark stream 起点改为匹配 `BENCHMARK_VALIDATION` 后跟 `iterations=` 的真正 benchmark 起点，规避前导噪声假阳性
