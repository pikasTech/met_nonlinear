- 任务：支持横评 preset 中的 project 在 `config.json` 挂载 `board_inference` 的 EP 相对路径，并在 `metrics.json` / compare preset 报告中汇总 `QEMU-MAE`、`KEIL-MAE`、`KEIL-SPEED (ms/point)`。
- 任务文件：`docs/MDTODO/07_推理BENCHMARK.md`

## 1. 静态分析结论

本轮先沿着“project config -> metrics 汇总 -> compare preset 读取/导出”链路做静态梳理，再做实跑验证：

1. `src/config.py`
   - `Config.load_from_json()` 采用严格字段加载，未知 key 会直接失败；因此要支持 project 级挂载 `board_inference_ep_path`，必须先在默认配置里注册字段。
   - 本轮顺带统一 `load_from_json()` / `save_to_json()` 显式使用 UTF-8。
2. `src/core/project_manager.py` -> `src/core/metrics_summary.py`
   - `python cli.py --metrics PROJECT_NAME` 最终走到 `MetricsSummaryGenerator` 生成 `data/metrics.json`。
   - 板端指标最合适的汇总点就在 `metrics_summary.py`，这样 compare preset / WebUI / 后续分析都只读 `metrics.json` 即可。
3. `src/visualization/ablation_study.py`
   - compare preset 报告完全从各 project 的 `metrics.json` 读取指标。
   - 因此 compare preset 只需新增对 board inference 字段的消费，不需要直接再读 EP 目录。
4. compare preset 配置
   - `ex_projects/compare/horizontal_1k_epoch/config.json`
   - `ex_projects/compare/horizontal_1k_epoch_all/config.json`
   - 两者都需要显式打开 `metrics.board_inference.enabled`，否则综合表虽然有列，但不会读取板端指标。
