## 1. 概述

### 1.1 任务背景

在 R1-R3 阶段，项目完成了 SVF 层误差仿真功能的实现和验证。通过实测 SVF 频率响应的拟合分析，发现实际制作的 SVF 电路与理想模型存在显著偏差：
- 中心频率 (f0) 平均偏差约 2.8%
- 品质因数 (Q) 平均偏差约 3.2%
- 增益误差约 0.5dB~2dB

直接修正 SVF 电路的误差存在困难，因此本任务（R4）探索另一种技术路线：**硬件协同训练（Hardware-in-the-Loop Training / Hardware-Aware Training）**。其核心思路是：先测量实际电路的 SVF 频率响应（带误差），然后重新做神经网络训练，训练时使用实际测量的带有误差的 SVF 层模型替代理想 SVF 层，这样训练后的 Dense 层就能够具备适应 SVF 层实际电路误差的能力。

### 1.2 R4 任务目标

### 1.3 核心技术路线选择

经过 R4.1 的深入调研，综合考虑技术可行性、实现成本和项目实际需求，推荐采用**实测响应约束的硬件感知训练**方案：

**方案概述**: 测量实际电路的 SVF 频率响应，拟合得到实际传递函数参数，在训练过程中使用这些实测参数替代理想参数，使得 Dense 层在训练时就能适应硬件的实际特性。

**选择理由**:
1. **技术成熟度高**: 该方案复用了大量现有基础设施（拟合代码、IIR 滤波器层、配置管理系统）
2. **实现成本低**: 只需在配置层面增加硬件感知训练开关，无需大规模代码重构
3. **可验证性强**: 训练结果可以直接与实测数据对比验证
4. **向后兼容**: 默认行为保持不变，新增功能通过配置控制

## 2. 调研结果总结 (R4.1)

### 2.1 硬件协同训练技术概述

硬件协同训练是一种将硬件特性纳入神经网络训练过程的技术，旨在解决软件模型与硬件实现之间的差距问题。根据 R4.1 的调研，主要有以下几种技术路线：

#### 2.1.1 硬件在环训练 (Hardware-in-the-Loop, HIL)

**原理**: 将实际硬件纳入训练循环，每次前向传播使用真实硬件，反向传播使用模型近似。

**特点**:
- 前向传播使用真实硬件输出
- 反向传播使用伴随模型或数值梯度近似
- 需要硬件支持远程控制和状态同步

**项目适用性评估**: 对本项目可行性较低，原因包括：
- SVF 电路是模拟电路，难以实现实时状态同步
- 训练需要数万次迭代，硬件在环速度太慢
- 需要额外硬件接口电路

#### 2.1.2 硬件感知训练 (Hardware-Aware Training)

**原理**: 在训练损失函数中加入硬件特性约束，或使用参数化的硬件模型进行模拟。

**特点**:
- 使用参数化的硬件模型模拟硬件行为
- 损失函数中加入硬件误差项
- 训练后的模型直接适配硬件特性

**项目适用性评估**: 推荐此方案为本项目的主要技术路线。

#### 2.1.3 可微分硬件仿真 (Differentiable Hardware Simulation)

**原理**: 使用可微分的电路仿真器，使得梯度可以通过电路模型回传。

**特点**:
- 电路模型支持自动微分
- 可以端到端优化硬件和软件参数
- 需要开发或引入可微分电路仿真器

**项目适用性评估**: 作为未来的技术演进方向，当前阶段优先采用方案 2。

#### 2.1.4 硅后调优 (Post-Silicon Tuning)

**原理**: 训练后使用实际测量结果进行微调。

**特点**:
- 先用理想模型训练
- 部署后根据实测结果微调
- 实现简单但效果有限

**项目适用性评估**: 可作为baseline方案与硬件感知训练对比。

### 2.2 推荐技术方案详细设计

#### 2.2.1 实测响应约束的硬件感知训练

**核心思路**: 使用实测 SVF 频率响应的拟合参数替代理想参数进行训练。

**实现步骤**:

1. **测量阶段**: 使用信号发生器和示波器测量实际 SVF 电路的频率响应
2. **拟合阶段**: 使用曲线拟合算法从实测数据提取 f0、Q、gain 参数
3. **配置阶段**: 将拟合参数写入配置文件
4. **训练阶段**: 训练时使用实测参数初始化 IIR 滤波器层
5. **验证阶段**: 对比训练结果与实测响应的匹配程度

**损失函数设计**:

```python
def hardware_aware_loss(output, target, svf_layer, config):
    nominal_loss = F.mse_loss(output, target)
    if config.get("error_weight", 0) > 0:
        ideal_output = svf_layer.forward_ideal(x)
        measured_output = svf_layer.forward_measured(x)
        hardware_loss = F.mse_loss(ideal_output, measured_output)
        return nominal_loss + config["error_weight"] * hardware_loss
    return nominal_loss
```

### 2.3 调研结论

| 技术路线 | 实施难度 | 预期效果 | 推荐程度 |
|---------|---------|---------|---------|
| 硬件在环训练 | 高 | 好 | 不推荐 |
| 硬件感知训练（实测参数） | 低 | 好 | **强烈推荐** |
| 可微分硬件仿真 | 高 | 非常好 | 未来方向 |
| 硅后调优 | 低 | 一般 | 作为baseline |

def _init_filters(self):
    """
    初始化带通滤波器IIR参数，支持理想模式和硬件感知模式
    """
    hw_aware = self.subcfg.get('hardware_aware_training', {})

    if hw_aware.get('enabled', False):
        # 硬件感知模式：使用实测参数
        measured_params = self.subcfg.get('_measured_svf_params')
        if measured_params:
            self._init_filters_hardware_aware(measured_params)
            return
        else:
            print("警告: 硬件感知训练已启用，但未找到实测参数，回退到理想模式")

    # 现有的理想模式初始化
    self._init_filters_ideal()

#### 5.2.3 新增 _init_filters_hardware_aware() 方法

```python
def _init_filters_hardware_aware(self, measured_params: dict):
    """
    使用实测参数初始化滤波器
    """
    svf_channels = measured_params.get('svf_channels', [])

    if not svf_channels:
        raise ValueError("实测参数中 svf_channels 为空")

    # 提取IIR系数
    a1_list = [ch['iir_coefficients']['a1'] for ch in svf_channels]
    a2_list = [ch['iir_coefficients']['a2'] for ch in svf_channels]
    b0_list = [ch['iir_coefficients']['b0'] for ch in svf_channels]
    b1_list = [ch['iir_coefficients']['b1'] for ch in svf_channels]
    b2_list = [ch['iir_coefficients']['b2'] for ch in svf_channels]

    # 获取训练模式配置
    hw_aware = self.subcfg.get('hardware_aware_training', {})
    training_mode = hw_aware.get('training_mode', {})
    svf_trainable = training_mode.get('trainable_svf', False)

    # 创建使用实测参数的DIAGIIR层
    self.init_iir = DIAGIIR(
        units=len(svf_channels),
        a1_list=a1_list,
        a2_list=a2_list,
        b0_list=b0_list,
        b1_list=b1_list,
        b2_list=b2_list,
        fs=self.fs,
        trainable=svf_trainable,
        init_by_system=True
    )

    self.fast_iir = self.init_iir
    self._hardware_aware_mode = True
```

#### 5.4.1 配置验证

```python
def validate_hardware_aware_config(config: dict) -> tuple[bool, list]:
    """
    验证硬件感知训练配置的完整性和正确性
    """
    errors = []
    warnings = []

    hw_config = config.get('model_subcfg', {}).get('hardware_aware_training', {})

    if not hw_config.get('enabled', False):
        return True, []

    # 检查必要字段
    if 'measured_svf_source' not in hw_config:
        errors.append("缺少 measured_svf_source 配置")

    # 检查训练模式
    training_mode = hw_config.get('training_mode', {})
    if training_mode.get('trainable_svf', False):
        warnings.append("SVF层设置为可训练，这可能导致过拟合")

    is_valid = len(errors) == 0
    return is_valid, errors + warnings
```

#### 5.4.2 参数一致性验证

```python
def validate_measured_params_consistency(
    measured_params: dict,
    model_subcfg: dict
) -> tuple[bool, list]:
    """
    验证实测参数与模型配置的一致性
    """
    warnings = []

    # 获取理想配置
    ideal_freqs = model_subcfg.get('init_center_freqs', [])
    ideal_Qs = model_subcfg.get('init_quality_factors', [])

    # 计算预期通道数
    expected_channels = len(ideal_freqs) * 3

    svf_channels = measured_params.get('svf_channels', [])
    actual_channels = len(svf_channels)

    if actual_channels != expected_channels:
        warnings.append(f"通道数不匹配: 预期 {expected_channels}, 实际 {actual_channels}")

    # 检查频率偏差
    for i, channel in enumerate(svf_channels):
        fitted = channel.get('fitted_params', {})
        ideal = channel.get('ideal_params', {})

        if fitted and ideal:
            f0_deviation = abs(fitted['f0'] - ideal['f0']) / ideal['f0'] * 100
            if f0_deviation > 10:
                warnings.append(f"通道 {i} 的f0偏差较大: {f0_deviation:.1f}%")

    is_consistent = len(warnings) == 0
    return is_consistent, warnings
```
