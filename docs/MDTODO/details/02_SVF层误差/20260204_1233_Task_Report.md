# R1.15.2 任务报告：frequency_response.png 绘图方式优化

## 任务目标

优化 `frequency_response.png` 的绘图方式，使其与 `FR_for_comp_real_data()` 中 `system_origin.plot()` 的绘图风格保持一致。

## 问题分析

### System.plot() 的绘图方式

参考 `src/calibration_analyzer/exam_class.py` 中 `System.plot()` 方法的实现（行1701-1751）：

```python
def plot(self, marker='o', markersize=7, time_domin=False, label=None,
         use_parallel=False, legend=True, gain_range=None, phase_range=None,
         freq_range=None, linestyle='-', disable_phase=False):
    # ...
    System.ax_gain.loglog(
        self.f, self.toabs(), marker=marker, markersize=markersize,
        label=system_name, color=instance_color, linestyle=linestyle
    )
    System.ax_gain.set_xlabel("Frequency (Hz)")
    System.ax_gain.set_ylabel("Amplitude")
    System.ax_gain.grid(True, which="both", ls="--")
    if legend:
        System.ax_gain.legend()
```

**关键特征**：
- 使用 `loglog` 双对数坐标系（x和y都用对数刻度）
- 无标记或小标记（`markersize=0` 或较小值）
- 简洁的标题和英文标签
- 颜色自动分配
- 网格线使用 `--`

### 原 R1.15.1 实现的问题

原代码使用 `semilogx` + `yscale('log')` 的组合：

```python
# 问题代码
ax_r1.semilogx(
    theo_freq,
    np.power(10, mag_db / 20),
    ...
)
ax_r1.set_yscale('log')
```

这种实现方式与 `System.plot()` 不一致，可能导致：
1. 坐标系显示风格不一致
2. 坐标轴标签不统一
3. 与 `-e` 模式下生成的图片风格不匹配

## 修改内容

修改 `src/visualization/wnet5_circuit_validator.py` 的 `_generate_plots_multi_file` 方法（行2760-2820）：

### 1. 使用纯 `loglog` 坐标系

```python
# 修改前
ax_r1.semilogx(...)
ax_r1.set_yscale('log')

# 修改后
ax_r1.loglog(...)  # 直接使用 loglog
```

### 2. 简化标题和标签

```python
# 修改后
ax_r1.set_title(
    f'{self.model_project_name} Dense#{analysis_layer} 频率响应',
    fontsize=12,
    fontweight='bold'
)
ax_r1.set_xlabel('Frequency (Hz)')
ax_r1.set_ylabel('Amplitude')
ax_r1.grid(True, which="both", ls="--")
```

### 3. 移除冗余的图例列数计算

```python
# 简化图例
ax_r1.legend(fontsize=8, ncol=min(4, len(output_labels) * 2), loc='best')
```
