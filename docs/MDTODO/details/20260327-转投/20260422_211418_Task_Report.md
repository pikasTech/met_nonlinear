## 标定结果

当前 compare EP 给出的关键结果如下：

### 1. 搜索结果

- pure fit best: `1 : 841.4109 : 4492.5951`
- regularized fit best: `1 : 3.5636 : 23.4254`
- adopted rounded model: `1 : 4.0 : 23.0`

可以看出，如果完全追求数值最优，拟合会把 add 的相对权重挤压得几乎可以忽略，不利于解释和长期维护。因此最终采用的是经过轻正则和四舍五入后的 `1:4:23`。

### 2. 与旧口径相比的改善

- 旧默认 `1:1:6` 的 `log-RMSE = 0.2334`
- 新默认 `1:4:23` 的 `log-RMSE = 0.1168`
- 最大相对误差由 `42.16%` 降到 `20.22%`

也就是说，在保持 add-normalized、便于解释的前提下，新的默认模型把静态复杂度对 STM32F405 实测时延的拟合误差大约压缩到原来的一半量级。

### 3. 参与标定的模型覆盖

当前参与配对的代表模型包括：

- Wiener-KAN
- GRU
- LSTM
- LSTMTransformer
- 1DCNN(board-ready)
- TCN

这组样本同时覆盖了：

- Wiener/KAN 类结构；
- 传统循环网络；
- CNN/TCN 卷积序列模型；
- 不同的 add/mul/map 组成比例。

因此，`1:4:23` 更适合作为当前仓库统一默认值，而不是某个单一模型的特例参数。

## 对 draft 的意义

R10 完成后，draft 中的 `Compute Cost` 不再是一个“启发式复杂度分数”，而是一个经过 STM32F405 实测时延反标、可以解释板端速度趋势的 deployment-aware 指标。这样一来：

1. 主横评中的 `Compute Cost` 与 `KEIL speed` 形成了更可信的对应关系；
2. Wiener-KAN 的“低复杂度 + 快板端速度”主张有了更扎实的定量支撑；
3. 后续如果比较不同部署实现或不同模型结构，`Compute Cost` 的讨论不再容易被质疑为任意加权。
