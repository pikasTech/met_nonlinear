## 0. 总体结论

当前 `main.tex` 已经比 TIM 拒稿版本明显加强：补入了更多对比模型、KAN/约束/频率先验/损失函数消融、数据集说明、嵌入式部署和 limitations。它已经具备“转投 MN 的材料基础”，但还没有达到“可直接提交 MN”的状态。

最核心的问题不是缺少某一个结果，而是全文仍像多个修订任务的拼接稿：数据、图表、指标、叙事口径和自动生成痕迹尚未被统一成一篇完整英文论文。若直接投出，风险最高的审稿点会集中在：

1. 正文语言和内部生成痕迹不符合投稿稿要求；
2. 摘要、结果表和正文中的关键指标存在口径冲突；
3. 数据集划分与泛化验证不足，可能被认为存在同条件窗口泄漏；
4. 消融和横评虽已补齐数量，但统计严谨性、基线配置和重复实验说明不足；
5. MN 标杆论文强调的“实验设计 + 数据集 + 多维指标 + 工程部署”叙事链尚未完全前置和打通。

建议先进行一次“全文统一重构”，再进入英文润色和 LaTeX 终稿阶段。

## 3. 对照标杆论文后的结构建议

### 3.1 借鉴 VPI / AI-Based Sensor 的写法

文献笔记中 VPI 标杆最值得借鉴的是：自建数据集直接在 Method 前半部分集中交代，参数范围依据初步实验和领域知识确定，实验条件数量、划分比例、训练时间、推理时间和内存使用量都显式量化。

当前稿件应采用如下结构：

1. `Introduction`：MET 非线性问题、反馈方法代价、前馈补偿机会；贡献列表中明确三维频响数据集、Wiener-KAN、AFMAE、嵌入式 LUT。
2. `Experimental Design and Dataset`：振动台、MET 样品、采集链、频率-幅值矩阵；数据规模、划分、目标曲线来源、泛化测试设置；未补偿频响漂移作为 problem characterization。
3. `Wiener-KAN Method`：先给整体框架图，再讲 Wiener prior、frequency injection、constrained KAN、AFMAE。当前 IIR-to-LRNN 推导很长，可压缩到核心公式，把完整矩阵推导移到 Supplementary。
4. `Results`：主补偿结果；与 baselines 比较；消融：activation / constraints / frequency prior / loss；部署：LUT、MCU latency、memory、accuracy trade-off。
5. `Discussion`：环境/温度、多传感器、公开数据和真实地震信号边界。

这样能更贴近 MN 标杆中“自建实验设计 + 数据集 + AI 模型 + 工程指标”的组合，而不是先给大量网络公式再回头补实验。

### 3.2 借鉴 Schaller Measurement 的写法

Schaller 标杆强调：公开/现实数据、未见漂移水平泛化、重复运行、均值标准差、数据泄漏风险、limitations。

当前稿件最需要补的是：

- 以 “unseen magnitude levels” 或 “unseen frequency-magnitude operating points” 作为泛化定义；
- 所有主模型和 baseline 做至少 3 次随机种子重复，给均值 ± 标准差；
- 横评表中保留 3 个核心指标即可，但每个指标要可解释：IONS / SDS / NFDS 的定义和单位必须在 Results 前单独说明；
- Limitations 不要只写后续计划，还要明确当前结论的适用边界：single MET sample, 25 °C, sine-sweep excitation, laboratory shaker table。

### 3.3 借鉴 MN MEMS 线性化标杆的写法

MN MEMS 线性化标杆强调多物理耦合非线性的来源、反馈方法复杂性，以及 AI 与器件设计的结合。

当前稿件已有 MET 大信号非线性机理和并联 Wiener 等效模型，这是一项强点。建议继续强化：

- 在 Introduction 早期保留非线性来源，但缩短为一段清晰物理图景；
- 将并联 Wiener 等效模型定位为 “problem characterization / mechanistic interpretation”，不要让它与主方法 Wiener-KAN 命名混淆；
- 用一张总览图把 “MET nonlinear response -> magnitude-conditioned local transfer functions -> Wiener-KAN compensation -> embedded LUT deployment” 串起来。
