## 二、重构方案

### 2.1 设计原则

1. **主题对齐**: 所有工作流必须围绕转投后论文的主旨设计
2. **废弃明确**: 不调研 PIKAN、FRIRNN 等已废弃内容
3. **三阶段分工**: 调研→分析→综合，职责分明，不可跨权代行
4. **审稿意见映射**: 每个阶段都要考虑对审稿意见的回应支撑
5. **可追溯性**: 所有结论必须能回溯到原始文献或分析报告

### 2.2 新的工作流架构

```
.loop/
├── PRINCIPLE.md      # 总目标：文献调研的总方向和重点（已废弃PIKAN相关）
├── STEP1.md          # 调研阶段：文献搜索与收集
├── STEP2.md          # 分析阶段：深度阅读与理论提取
├── STEP3.md          # 综合阶段：论文融合与素材整理
└── REPORT_TEMPLATE.md # 报告模板：统一各阶段报告结构
```

## 三、各文件设计详解

### 3.1 PRINCIPLE.md - 总目标与方向

**核心目标**: 系统性扩充文献调研，为论文修订提供理论支撑和对比依据。

**重点方向（P0 必须覆盖）**:
1. **Wiener 模型与非线性系统识别** - 支撑建模模拟 + Wiener-KAN 架构联系
2. **KAN 网络理论** - 支撑计算效率优势（LUT 效率）
3. **AFMAE 损失函数** - 支撑训练方法声称

**重点方向（P1 必须覆盖）**:
4. 深度学习频率漂移补偿相关工作
5. 神经网络架构的计算效率对比（RNN vs 1D-CNN, KAN vs LSTM/GRU/Transformer）
6. 数据集构建参考

**重点方向（P2 可选扩展）**:
7. 计算效率评估方法
8. 公开数据集

**明确废弃（不调研）**:
- PIKAN 相关内容
- FRIRNN 频响注入相关内容

### 3.2 STEP1.md - 文献搜索与收集

**核心职责**: 发现文献线索并结构化落表

**检索方向调整**:
- 新增：KAN vs LSTM/GRU/Transformer 计算效率对比
- 新增：RNN vs 1D-CNN 计算效率对比
- 移除：PIKAN 物理约束相关

**输出**:
- `docs/research/literature/literature_catalog.md` - 文献总目
- `docs/research/literature/raw_literature.md` - 原始文献线索
- `docs/research/literature/{YYYYMMDD}/` 下的调研报告

### 3.3 STEP2.md - 文献分析与理论提取

**核心职责**: 深度阅读，提取理论支撑

**关键理论提取方向**:
- Wiener 模型：定义、辨识方法、与 Hammerstein 的区别
- KAN 网络：Kolmogorov-Arnold 表示定理、B-spline 基函数、LUT 实现
- AFMAE 损失：频域分析基础、与时域 MAE 的关系
- 计算效率：参数量、MACs、推理时间的评估方法

**输出**:
- `docs/research/literature/verified_literature.md` - 已核实文献
- `docs/research/literature/excluded_literature.md` - 排除文献
- `docs/research/literature/{YYYYMMDD}/` 下的分析报告

### 3.4 STEP3.md - 综合整理与论文融合

**核心职责**: 支撑论文修订声称，回应审稿意见

**审稿意见回应映射（第二稿重点）**:

| 审稿意见 | 需要支撑的内容 | 对应文献类型 |
|---------|--------------|-------------|
| R3-4 对比模型有限 | CNN、Transformer、RNN 在时序中的应用 | 架构对比 |
| R3-5 RVTDCNN 是否最佳 | 功率放大器线性化方法总结 | 补偿方法 |
| R3-6 数据集细节不足 | 数据集构建参考论文 | 数据集 |
| R4-1 激活函数比较 | ReLU、tanh、B-spline 对比研究 | 激活函数 |
| R4-8 计算成本分析 | 效率评估标准方法 | 效率评估 |

**已废弃的审稿意见回应（不需要专项支撑）**:
- R3-1: PIKAN vs 原始KAN → PIKAN 已废弃
- R3-2: 物理约束效果 → 物理约束已废弃
- R4-2: 物理约束消融实验 → 物理约束已废弃
- R4-3: 频率注入效果 → FRIRNN 已废弃
- R4-4: 损失函数消融（AFMAE vs MAE）→ 可作为实验设计参考，但不需要为 AFMAE 本身提供文献证明

**输出**:
- `docs/research/literature/key_references.md` - 核心文献清单
- `docs/research/literature/theory_framework.md` - 理论框架
- `docs/research/literature/paper_draft_segments.md` - 论文素材草稿

### 3.5 REPORT_TEMPLATE.md - 统一报告结构

定义了三种报告模板：
1. **调研报告（STEP1）**: 检索路径、发现结果、待核实事项
2. **分析报告（STEP2）**: 理论提取、文献质量评估、审稿意见支撑
3. **综合报告（STEP3）**: 理论框架、审稿意见回应映射、论文素材
