## 关键结果与结论变化

### 1. Wiener-KAN 不再适合写成“当前最快板端实现”

R10.3 最重要的变化不是补了两行数字，而是它改变了最稳妥的论文叙事。

在主横评当前这组 canonical projects 中：

- `RNN` 现在是**最快**的板端实现：`0.274 ms/point`
- canonical `1DCNN` 也**略快于** Wiener-KAN：`0.498 ms/point` vs `0.537 ms/point`
- Wiener-KAN 不再是速度第一名

因此，后续 draft 和中间稿里都不应再把 Wiener-KAN 写成“最快部署实现”。

### 2. Wiener-KAN 仍然保有最强的综合平衡叙事

虽然 Wiener-KAN 不再是板端最快，但它仍然是当前最容易成立的“综合平衡最优”对象：

- 三项正式物理指标仍是最低：
  - `Freq Drift = 2.34 Hz`
  - `Sens Drift = 9.09%`
  - `Linearity (in-band) = 0.511%`
- 静态 `Compute Cost = 5808` 仍明显低于 TCN / LSTM / LSTMTransformer / GRU
- `KEIL-SPEED = 0.537 ms/point` 仍处于 `sub-ms` 区间

更关键的是，新补齐的两个对象刚好给出了两种“不能只看速度”的反例：

1. **RNN：速度极快，但综合质量明显不成立**
   - `KEIL-SPEED = 0.274 ms/point`
   - `KEIL-MAE = 8.74e-02`
   - 三项物理指标全面落后

2. **1DCNN：板端一致性更强、速度略快，但正式物理指标仍整体弱于 Wiener-KAN**
   - `KEIL-MAE = 5.51e-04`
   - `KEIL-SPEED = 0.498 ms/point`
   - `Freq Drift`、`Sens Drift`、`Linearity` 都高于 Wiener-KAN

所以，Wiener-KAN 当前最稳的表述应从“最快”收口为：

> 在当前 canonical 对比集合内，Wiener-KAN 提供了最强的物理指标表现，并在静态复杂度与板端速度上保持了可部署的中低成本平衡。

### 3. 主横评表现在终于具备完整板端辅助列

R10.3 完成后，表 2 所对应的 7 个 canonical models 已全部具备：

- 正式物理指标三列；
- `Compute Cost`；
- `KEIL-MAE`；
- `KEIL speed (ms/point)`。

这意味着主横评现在终于可以在**统一 project、统一指标定义、统一部署挂载方式**下做完整比较，而不是继续把 1DCNN / RNN 排除在部署辅助列之外。

## 对 draft 的具体影响

本轮更新后，draft 的相关叙事已经同步收口为以下口径：

1. Wiener-KAN 的核心卖点是**三项正式物理指标 + 部署成本的综合平衡**，而不是单纯的最快速度；
2. `RNN` 应作为“最低复杂度 / 最快速度但综合效果失真”的反例；
3. canonical `1DCNN` 应作为“更强部署导向基线”，即它在 `KEIL-MAE` 与 `KEIL-SPEED` 上更漂亮，但正式物理指标仍落后；
4. 主横评的部署辅助列现在是完整的，因此相关结论比 R10 前后的旧稿更可信。

这一步让论文里的部署比较从“局部有缺列的趋势判断”升级成了“7 个 canonical models 全部落在同一规则下的完整横评”。
