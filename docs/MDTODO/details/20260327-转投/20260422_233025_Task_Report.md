## 完成内容

### 1. 将统一指标口径切换为 bounded fit

更新了：

- `src/core/metrics_summary.py`

本次修改后，`Freq Drift (Hz)` 的正式口径变为：

1. 从 `linear_response.json.fit_params_comped` / `fit_params_origin` 读取二阶拟合参数；
2. 由 `B = \omega_n^2` 恢复中心频率 `f = sqrt(B) / (2*pi)`；
3. 将恢复出的中心频率限制在 `10-128 Hz` band 内；
4. 再按中位数中心漂移定义 `Freq Drift`。

因此，正式口径不再依赖 spline 插值峰值，而是回到更符合原始物理解释的 `fit_params` 路径，同时避免异常拟合把中心频率抛出频带。

### 2. 将 band limit 接回上游拟合生成路径

更新了：

- `src/calibration_analyzer/exam_process.py`
- `src/visualization/model_analysis.py`

这一步不仅改 summary 重算，还把 `linear_response.json` 的 `fit_params_origin/comped` 生成过程本身纳入 band limit 约束。也就是说，`ws_system_fit(...)` 在当前项目默认频带下会把拟合中心频率约束在 `10-128 Hz` 内，避免上游就产生带外失真参数。

## 新结果

### 1. 主横评结果

新的正式口径下，主横评关键结果为：

| Model | Freq Drift (Hz) | Sens Drift (%) | Linearity (in-band, %) |
| --- | --- | --- | --- |
| Wiener-KAN | 2.34 | 9.09 | 0.511 |
| TCN | 2.65 | 16.08 | 0.517 |
| 1DCNN | 3.72 | 16.24 | 0.920 |
| LSTM | 4.50 | 13.80 | 0.512 |
| LSTMTransformer | 4.72 | 18.38 | 0.567 |
| RNN | 6.68 | 23.75 | 0.726 |
| GRU | 7.40 | 27.15 | 0.726 |

在这一口径下，Wiener-KAN 重新回到三项正式物理指标全部第一：

- `Freq Drift = 2.34 Hz`
- `Sens Drift = 9.09%`
- `Linearity = 0.511%`

### 2. 相对原始响应的抑制率

相对未补偿原始响应：

- `Freq Drift` 压低 `93.23%`
- `Sens Drift` 压低 `89.25%`
- `Linearity` 压低 `74.50%`

### 3. 相对 TCN 的对照

相对 deployment-ready 强基线 TCN：

- `Freq Drift` 再降低 `11.72%`
- `Sens Drift` 再降低 `43.46%`
- `Linearity` 再降低 `1.13%`
- `Compute Cost` 再降低 `56.20%`
- `KEIL-SPEED` 提速 `49.74%`

但 TCN 仍然保留更低的 `KEIL-MAE`，因此板端一致性仍然是 Wiener-KAN 的边界项，而不是已经完全解决的问题。

### 4. 损失消融重新变得更清晰

- `MAE+AFMAE` vs `MAE`：`Freq Drift` 再降低 `82.03%`
- `MAE+AFMAE` vs `AFMAE`：`Freq Drift` 再降低 `41.64%`

因此，在新的正式口径下，组合损失的优势不再只是 Sens/Linearity 两项，而是在三项正式物理指标上都能直接观察到。

### 5. 结构消融仍保持稳定

- `No positive (stress)` 不再爆炸，而是稳定在 `58.03 Hz`
- `No symmetry` 恶化到 `10.91 Hz`
- `FRIMLP` 为 `5.82 Hz`
- `CNNKAN` 为 `2.43 Hz`

这说明 band-limited fit 口径同时达成了两项目标：

1. 保住原始 `fit` 路径的物理解释；
2. 压住异常带外爆炸值，同时不再扭曲 Wiener-KAN 的结构排序。

## 结论

R11.1 已完成，最终正式策略为：

- 不采用 spline 峰频作为论文正式口径；
- 保留原始 `fit_params` 路径；
- 将中心频率限制在当前正式 band（`10-128 Hz`）内；
- 将该 bounded fit 口径集成进 CLI 默认 summary 流程与 draft 数据生成流程。

在这一新口径下，Wiener-KAN 重新具备了更强、更自然的结果表达：

- 三项正式物理指标全部第一；
- 同时保持较低 `Compute Cost` 与当前最快 `KEIL-SPEED`；
- 但仍需明确承认 `KEIL-MAE` 与 Flash 占用是当前部署边界。

因此，R11.1 的最终结果比 R11 的 spline 试验口径更适合作为后续中文 draft 与正式论文的默认指标标准。
