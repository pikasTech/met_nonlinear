# R15 Task Report

## 任务目标

围绕 `docs/MDTODO/20260327-转投.md` 中的 `R15`，对 FRIKAN 基线 `FRIKANh8u6l6g8s2_e1k_lr7e4_base` 做主超参数敏感性分析，覆盖：

- `H_UNITS`
- `INNER_KAN_UNITS`
- `INNER_KAN_LAYERS`
- `GRID_SIZE`
- `SPLINE_ORDER`

目标不是盲目寻找单点最优，而是回答：

1. 当前基线是否只是偶然命中的脆弱超参数点；
2. 可训练区域内是否存在显著更优的替代配置；
3. 哪些轴已经触碰当前训练环境的可行域边界。

## 结果结论

### 1. 现有成功点已经足够支撑论文中的敏感性分析

理由如下：

- `H_UNITS` 和 `GRID_SIZE` 两条主轴已经完整扫描；
- `INNER_KAN_UNITS`、`INNER_KAN_LAYERS` 和 `SPLINE_ORDER` 虽然高侧被 OOM 截断，但低侧 / 中侧已经覆盖；
- 所有 OOM 点都被显式记录，能够界定当前训练环境中的可行域边界；
- 成功点中没有任何一个项目同时严格支配基线的 `Freq Drift / Sens Drift / Linearity` 三项核心指标。

因此，论文可以成立地写成：

- 基线不是一个脆弱的偶然点；
- 在可训练区域内，不同超参数会改变性能折中，但没有出现一个全面优于基线的替代点；
- 更大的 `U / L / S` 在当前环境下首先暴露的是训练可行域问题，而不是稳定性能收益。

### 2. 当前最重要的论文边界条件

论文必须明确写出：

- 结果结论限定在 `feasible training region`
- `u7/u8/l7/l8/s3` 为 `OOM / train-time infeasible under current environment`
- 不能把这些 OOM 点写成“已经证明更差”

### 3. 当前数据中的可写结论

- `H_UNITS`：较低或中等容量（如 `h5/h7/h9`）会改变个别指标，但未出现全面优于基线的稳定点。
- `INNER_KAN_UNITS`：从 `u3 -> u5` 可改善部分灵敏度漂移，但高侧 `u7/u8` 已触碰显存边界。
- `INNER_KAN_LAYERS`：`l3` 与 `l5` 在部分指标上接近甚至略优于基线，但线性度没有同时占优，高侧 `l7/l8` 不可训练。
- `GRID_SIZE`：可训练范围内存在一定波动，`g5` 是较好的可训练折中点之一；同时 `Compute Cost` 不变，说明该轴主要改变拟合能力而不改变当前部署成本估算。
- `SPLINE_ORDER`：`s1` 明显恶化线性度，`s3` 直接 OOM，说明 `s2` 是当前更合理的工程折中点。
