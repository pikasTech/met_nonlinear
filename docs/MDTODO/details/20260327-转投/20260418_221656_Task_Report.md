# R6 任务报告：MN 转投还应补哪些消融实验

## 任务目标

结合此前的转投分析、当前论文主旨和仓库里的最新项目状态，判断：

**如果现在还要继续补消融实验，最值得补的是哪些，优先顺序应该怎么排。**

本报告重点不是“还能补多少实验”，而是“在当前 MN 转投叙事下，哪些实验最有边际价值，哪些实验已经够用或暂时不值得继续扩”。

---

## 一、结论先行

当前最值得补的，不是继续扩 `model zoo`，也不是去做一轮新的盲扫超参数，而是：

1. **优先补一组干净的 `Wiener/IIR 前端先验` 消融矩阵。**
   这是当前证据链里最薄弱、但对 `Wiener-KAN` 命名和主叙事最关键的一环。
2. **其次补齐约束消融中“错误对称先验”这一格。**
   当前已经有 `nosym` 和 `nopositive`，但还缺一个能证明“不是随便加个对称性都行”的版本。
3. **再把 LUT / 部署 trade-off 做成正式消融，而不是只保留一张加速结果图。**
   这是 MN 叙事下少数真正值得保留的工程主张之一。
4. **损失函数消融和 `KAN vs MLP` 消融已经基本够用，不应再优先继续扩。**
   这两条线当前更需要的是“收口正式矩阵”，不是继续新开很多项目。

一句话概括：

**如果只再补 2~3 组最有价值的消融，我建议优先押注“前端先验 -> 约束闭环 -> 部署权衡”，而不是再去补激活函数、更多 baseline 或超参数敏感性。**

---

## 二、当前仓库已经能回答什么

在决定“还补什么”之前，先看当前仓库里的现有资产已经回答了哪些问题。

### 2.1 loss 消融其实已经够用了

当前仓库已经有：

- `projects/01_LR_STUDY/FRIKANh8u6l6_e1k_lr7e4`
- `projects/01_LR_STUDY/FRIKANh8u6l6_e1k_lr7e4_puremae`
- `projects/03_FRIKAN_PUREAFME/FRIKANh8u6l6_e1k_lr5e4_pureafmae`

关键指标大致为：

| 项目 | Freq Drift (Hz) | Sens Drift (%) | Linearity (%) |
|---|---:|---:|---:|
| MAE+AFMAE | 2.3366 | 9.0926 | 0.8902 |
| pure MAE | 13.0025 | 20.2839 | 1.3689 |
| pure AFMAE（当前最优） | 4.0034 | 21.0583 | 1.7534 |

这说明：

- 组合损失明显优于纯 MAE；
- 组合损失也优于当前最优 pure AFMAE；
- 因此 **loss 这条线已经能支撑“这是一个有效设计选择”**。

所以，loss 现在不是“最该补的新实验”，而是：

- 把正式稿件里引用的 pure AFMAE 项目切到当前最优版本；
- 不要再把更多精力花在继续扩 loss 家族上。

### 2.2 `KAN vs MLP` 其实也已经有可用证据

当前最有代表性的 `FRIMLP` 项目是：

- `projects/04_FRIMLP/FRIMLPh8u6l6_e1k_lr7e4_mlp20l6_tanh_d00`

其指标为：

- Freq Drift = `5.8179`
- Sens Drift = `27.4803`
- Linearity = `1.0120`

相对基线 `FRIKANh8u6l6_e1k_lr7e4`：

- 频漂、灵敏度漂移都明显退化；
- 但并不是完全跑崩，说明这是一个**有效且有辨识度的结构消融**。

这意味着：

- `KAN vs MLP` 已经足够回答“为什么不是一个同前端的普通 MLP 就够了”；
- 当前更大的问题不是“没有 FRIMLP”，而是**正式结构消融矩阵还没把最好的 FRIMLP 用进去**。

### 2.3 约束消融已经很强，但还没完全收口

当前约束相关项目包括：

- 基线：`projects/01_LR_STUDY/FRIKANh8u6l6_e1k_lr7e4`
- 去对称：`projects/01_LR_STUDY/FRIKANh8u6l6_e1k_lr7e4_nosym_r5`
- 去正定：`projects/01_LR_STUDY/FRIKANh8u6l6_e1k_lr7e4_nopositive`

关键指标如下：

| 项目 | Freq Drift (Hz) | Sens Drift (%) | Linearity (%) |
|---|---:|---:|---:|
| 基线 | 2.3366 | 9.0926 | 0.8902 |
| `nosym_r5` | 10.9128 | 10.6135 | 2.1996 |
| `nopositive` | 325784.5030 | 50.8654 | 35.1479 |

这已经足以说明：

- 约束不是装饰；
- 尤其 `only_positive` 去掉后几乎是灾难性退化；
- `use_symmetry` 去掉后也明显变差。

所以当前不是“约束线完全没证据”，恰恰相反，**这条线已经有很强的正面证据**。

它还差的只是：

- 补一个“错误对称先验”版本；
- 让结论从“去掉会退化”升级为“默认先验优于其他合理但错误的结构先验”。

### 2.4 真正薄弱的是 `Wiener/IIR 前端先验` 这条线

当前仓库里与前端先验最相关的项目是：

- 基线：`projects/01_LR_STUDY/FRIKANh8u6l6_e1k_lr7e4`
- 随机前端：`projects/01_LR_STUDY/FRIKANh8u6l6_e1k_lr14e5_randfrirnn_r2`
- CNN 前端：`projects/01_LR_STUDY/CNNKANh8u6l6_e1k_lr28e5_c8k5d05`

关键问题在于：

1. `randfrirnn_r2` 不是干净消融。它同时改了：
   - `IIR_INIT_BY_SYSTEM: true -> false`
   - `IIR_TRAINABLE: false -> true`
   - `learning_rate: 7e-4 -> 1.4e-4`
2. 也就是说，它把“有没有物理先验”“线性前端是否可训练”“学习率变化”三个因素揉在了一起。
3. 与此同时，`CNNKAN` 的结果又相当接近基线：
   - `CNNKAN`: `2.4307 / 9.2237 / 0.9502`
   - Base: `2.3366 / 9.0926 / 0.8902`

这会导致一个非常现实的问题：

**当前证据更像是在证明“KAN 主体很重要”，而不是在干净地证明“Wiener/IIR 先验本身非常关键”。**

而论文当前又是按 `Wiener-KAN` 来命名和叙事的，所以这条线反而成了最该补强的地方。

---

## 三、最值得补的消融实验清单（按优先级排序）

## P0：先把已有但未纳入正式矩阵的关键项目收口（零新增训练优先）

这一步严格说不算“新增消融”，但它是当前 ROI 最高的动作，应该放在所有新训练之前。

### 建议先纳入正式矩阵的项目

1. `projects/01_LR_STUDY/FRIKANh8u6l6_e1k_lr7e4_nopositive`
   - 当前结构消融 preset 里没有它；
   - 但它是约束线里最强的一条反证。

2. `projects/04_FRIMLP/FRIMLPh8u6l6_e1k_lr7e4_mlp20l6_tanh_d00`
   - 当前结构消融 preset 里还是 `lr2e3_mlp18l7` 那个较弱版本；
   - 应切换到修复后的 stable FRIMLP。

3. `projects/03_FRIKAN_PUREAFME/FRIKANh8u6l6_e1k_lr5e4_pureafmae`
   - 当前 loss 对照若还引用 `lr1e4_pureafmae`，建议升级到当前更优的 `lr5e4` 版本。

### 为什么这一步优先级最高

因为如果正式矩阵本身还没把已有最强证据用进去，那么现在直接新增实验，会把“实验不足”和“矩阵没整理好”两个问题混在一起。

---

## P1：补一组干净的 `IIR 前端先验` 2×2 消融矩阵（最高优先级的新训练）

### 目标

把当前混在一起的三个问题拆开：

1. 物理系统初始化到底有没有价值？
2. 线性前端设为 trainable 到底有没有价值？
3. 两者有没有交互效应？

### 推荐矩阵

以当前基线 `projects/01_LR_STUDY/FRIKANh8u6l6_e1k_lr7e4` 为母本，只改这两个开关，其余保持完全一致：

| 版本 | `IIR_INIT_BY_SYSTEM` | `IIR_TRAINABLE` | 目的 |
|---|---|---|---|
| A（现有基线） | `true` | `false` | 当前 Wiener-KAN 正式版本 |
| B（新增） | `false` | `false` | 只测试“去掉系统先验”的代价 |
| C（新增） | `true` | `true` | 只测试“让系统先验可微调”的收益/风险 |
| D（新增或重跑） | `false` | `true` | 测试“无先验但可学习”的上限 |

### 推荐项目命名（示意）

- `projects/01_LR_STUDY/FRIKANh8u6l6_e1k_lr7e4_randiir_frozen`
- `projects/01_LR_STUDY/FRIKANh8u6l6_e1k_lr7e4_sysiir_trainable`
- `projects/01_LR_STUDY/FRIKANh8u6l6_e1k_lr7e4_randiir_trainable`

### 这一组实验能回答什么

它能把现在最关键但最模糊的问题彻底拆清：

- 如果 B 明显退化，说明**系统先验本身有价值**；
- 如果 C 不明显优于 A，说明**冻结系统先验比继续训练更稳**；
- 如果 D 仍明显不如 A，说明**不是“给个可训练线性层就行”，而是“正确的系统初始化更重要”**。

### 为什么它是最高优先级

因为当前稿件若继续使用 `Wiener-KAN` 这一主叙事，这条线就是最需要补硬证据的地方。

---

## P2：补齐约束矩阵中的“错误对称先验”版本（第二优先级）

### 当前已有的三格

根据 `src/models/frikan_models.py` 当前约束语义，已有：

- 默认：`use_symmetry=true, only_positive=true, use_even=false`（正定 + 奇对称）
- `nopositive`：`use_symmetry=true, only_positive=false, use_even=false`
- `nosym`：`use_symmetry=false`

### 当前缺的那一格

缺的是：

- `use_symmetry=true, only_positive=true, use_even=true`

也就是：

**保留对称处理和正定约束，但把“奇对称”改成“偶对称”。**

### 为什么这格最值得补

因为它能回答一个比“去掉约束会不会变差”更强的问题：

**默认约束之所以有效，是因为它编码了正确的物理先验，而不是因为“随便加点结构限制都能变好”。**

### 推荐项目命名（示意）

- `projects/01_LR_STUDY/FRIKANh8u6l6_e1k_lr7e4_poseven`

### 完成后可形成的正式约束矩阵

| 版本 | 语义 |
|---|---|
| base | 正定 + 奇对称（默认） |
| poseven | 正定 + 偶对称 |
| nopositive | 奇对称但不强制正定 |
| nosym | 完全关闭对称处理 |

这套矩阵已经足以支撑一段相当完整的约束故事，而且比继续盲目补更多组合更有解释力。

---

## P3：把 LUT / 部署 trade-off 做成正式消融（第三优先级）

### 为什么它值得做

在当前 MN 转投方案里，真正值得保留的强主张之一是：

- 不是单纯“精度比别人高一点”；
- 而是“在有效补偿的同时具备部署潜力”。

如果继续保留这条主张，那么现在只靠一张 `LUT 加速图` 还不够，最好把它升级成正式消融。

### 推荐做法

固定同一个训练完成的 `Wiener-KAN` 权重，只改 LUT 配置，例如：

- 100 点
- 200 点
- 400 点
- 800 点
- 1600 点

统一导出并记录：

- MAE / 频漂 / 灵敏度漂移 / 线性度退化
- STM32 latency
- Flash 占用
- 是否插值

### 这一组实验回答什么

它不回答“模型有没有效果”，而是回答：

**当部署资源受限时，Wiener-KAN 到底能以多大的精度代价换来多少速度和存储收益。**

这对 MN 叙事很重要，因为这比再补一组 activation sweep 更接近真正的工程结论。

---

## P4：如果还有余力，再做 `loss` 或 `前端简化` 的小补强，但不是首选

这一层不是当前最优先，但如果 P1 / P2 / P3 做完后还有精力，可以考虑。

### 方案 A：loss 清洁版补强

当前 loss 线已经够用，所以这里只建议做**很小的补强**，例如：

- 确认正式图表使用当前最优 `pureafmae` 版本；
- 若特别在意“同 lr 下是否成立”，再补一组严格同 lr 对照。

但这不是首选，因为现在这条线并不缺故事。

### 方案 B：更“简化前端”的版本

如果后续还想把 `Wiener` 前端价值写得更强，可以再考虑补一个“更弱前端”的版本，用来回答：

- 是不是任何时序前端都可以；
- 还是“显式对齐频响/系统参数”的前端更合理。

但在当前 repo 里，这一步的边际价值仍低于 P1 那组干净的 `IIR_INIT_BY_SYSTEM × IIR_TRAINABLE` 2×2 矩阵。

---

## 四、哪些实验当前不建议优先补

为了避免重新掉进“实验越做越散”的陷阱，我建议当前明确**不优先**做下面这些：

### 4.1 不建议优先做激活函数扫表

原因：

- 当前主稿不会靠“tanh 比 ReLU 更好”来立论；
- 这条线对 MN 工程主叙事贡献很有限；
- 它更像旧版方法稿会被追问的细节，不是当前最值钱的补强。

### 4.2 不建议继续扩更多 baseline 名字

比如再补更多 Transformer / RNN / CNN 变体。

原因：

- 现在真正缺的不是“模型数目不够多”；
- 而是 **机制性证据没有完全闭环**；
- 如果再去扩 baseline，很容易重新走回 `model zoo`。

### 4.3 不建议优先做超参数敏感性分析

原因：

- 这更像补充材料级内容；
- 对当前最核心的 claim budget 帮助不大；
- 训练成本高、叙事收益低。

### 4.4 不建议把多 seed 当成当前最重要的“消融实验”

原因：

- 它更接近 validation protocol strengthening，而不是机制消融；
- 在当前阶段，它的优先级低于 P1/P2/P3；
- 如果时间有限，应先把机制线补闭。

---

## 五、推荐执行顺序

如果现在只想走一条最务实的路线，我建议按这个顺序推进：

1. **先不新增训练，先把现有矩阵收口**
   - `nopositive`
   - stable `FRIMLP`
   - best `pureafmae`

2. **优先补 `IIR_INIT_BY_SYSTEM × IIR_TRAINABLE` 的 2×2 前端先验消融**
   - 这是当前最缺、也最关键的一组。

3. **再补 `poseven`，完成约束矩阵闭环**
   - 让约束故事从“去掉会变差”升级为“正确先验优于错误先验”。

4. **最后补 LUT trade-off 正式消融**
   - 服务部署和工程价值主张。

如果这四步都完成了，那么当前 MN 稿件最关键的消融短板就已经补到相当够用了。

---

## 六、最终建议

对当前这个 repo 和当前这篇 MN 转投稿来说，最准确的建议是：

**下一轮不要再平均用力地补各种消融，而要把新增消融集中砸在“前端先验是否真的必要”这条线。**

因为：

- `loss` 线已经基本够用；
- `KAN vs MLP` 已经有可用证据；
- `约束` 线已经很强，只差最后一格；
- **真正还没有被干净证明的，是 `Wiener/IIR 先验` 这条主叙事。**

所以，如果只允许再补少量消融，我的优先级排序是：

1. **前端先验 2×2 清洁矩阵**
2. **`poseven` 约束闭环**
3. **LUT 部署 trade-off**

而在这之前，先把已有但未正式纳入矩阵的 `nopositive`、stable `FRIMLP` 和最优 `pureafmae` 用起来。

这样补出来的实验，才最能服务当前的 Wiener-KAN / MN 转投主线，而不是重新把论文带回“方法点很多但证据分散”的旧路。
