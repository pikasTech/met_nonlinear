# R11 任务报告：将 Freq Drift 改为 spline in-band 峰频口径并验证稳定性

## 任务目标

根据 `docs/MDTODO/20260327-转投.md` 中 R11 的要求，本轮任务要解决 `Freq Drift (Hz)` 在旧 `fit_params` 路径下可能出现的数值爆炸问题，例如 `No positive (stress)` 项目曾出现 `3.26e+05` 这类明显不可信的结果。用户提出的新方案是：

1. 不再直接依赖 `fit_params_comped` 的峰频；
2. 先对带内频响曲线做 `Spline` 插值；
3. 再在当前 band 内提取最高点对应的中心频率；
4. 若验证效果合理，则把它升级为统一指标新标准，并同步更新长期参考文档与 draft。

---

## 完成内容

### 1. 修改统一指标实现

更新了：

- `src/core/metrics_summary.py`

本次修改将 `Freq Drift` 的正式提取逻辑改为：

1. 从 `linear_response.json.gains_comped` / `gains_origin` 读取频响曲线；
2. 仅保留 `<=128 Hz` 的 in-band 频点；
3. 对每条曲线做 cubic spline 插值；
4. 在 2048 点带内均匀网格上寻找峰值频率；
5. 再用中位数中心漂移定义 `Freq Drift`。

同时，`Linearity (%)` 也被固定为 `<=128 Hz` 的 in-band 平均非线性误差口径，并写入相应元数据字段。

### 2. 补齐测试

更新了：

- `src/tests/core/test_metrics_summary.py`

用于验证：

- `Freq Drift` 的新实现来自 spline 带内峰频，而不是旧的 `fit_params`；
- `Linearity (%)` 只统计 `<=128 Hz` 的频点；
- `metrics.json` 会写出新的 calculation standard 与带内元数据。

### 3. 刷新长期参考文档与 draft

更新了：

- `docs/reference/metrics.md`
- `docs/reference/paper_metric_calculation_method.md`
- `docs/paper/draft/20260422/paper_draft_zh.md`

并重新生成了 draft 相关图表与表格，使新口径能直接进入结果分析链。

## 验证结果

### 1. 极端爆炸值被消除

在新的 spline in-band 口径下：

- `No positive (stress)` 的 `Freq Drift` 从旧的异常爆炸值回落到 `60.35 Hz`

这表明新方法至少达成了本轮任务最核心的稳定性目标：把明显失真的拟合结果重新限制到可解释的物理带内范围。

### 2. Wiener-KAN 的相对优势发生了变化

新口径带来的副作用也非常明确：

- Wiener-KAN：`Freq Drift = 7.55 Hz`
- TCN：`4.44 Hz`
- 1DCNN：`4.09 Hz`
- LSTM：`5.19 Hz`
- LSTMTransformer：`5.02 Hz`
- RNN：`5.65 Hz`
- GRU：`6.11 Hz`

也就是说，Wiener-KAN 在新的 spline 峰频定义下，不再是 `Freq Drift` 最低的模型。它仍然在 `Sens Drift` 和 in-band `Linearity` 上表现最好，但“峰频稳定化第一”的叙事明显被削弱了。

---

## 对论文叙事的影响

R11 带来的直接影响是：

1. `Freq Drift` 指标本身变得更稳定、更物理可解释；
2. draft 中不能再把 Wiener-KAN 写成“三项主物理指标全部最优”；
3. 更合理的表述变成了：Wiener-KAN 的 strongest claim 主要来自
   - 最低 `Sens Drift`
   - 最低 in-band `Linearity`
   - 较低 `Compute Cost`
   - 最快 `KEIL speed`
4. `Freq Drift` 则需要承认 1DCNN / TCN 在新口径下更强。

因此，R11 虽然提高了指标稳定性，但也明显压缩了 Wiener-KAN 在主横评中的 claim budget。
