## 关键结果变化

### 1. Wiener-KAN 的论证重心从“最快”收口为“综合平衡最好”

主横评当前结果为：

- Wiener-KAN：`Freq Drift = 2.34 Hz`、`Sens Drift = 9.09%`、`Linearity = 0.511%`、`Compute Cost = 5066`、`KEIL speed = 1861.9 Points/s`
- TCN：`2.65 Hz / 16.08% / 0.517% / 8704 / 935.7 Points/s`
- 1DCNN：`3.72 Hz / 16.24% / 0.920% / 3984 / 2009.9 Points/s`
- RNN：`6.68 Hz / 23.75% / 0.726% / 2816 / 3652.9 Points/s`

这组结果意味着：

1. Wiener-KAN 仍然是三项正式物理指标的最佳模型；
2. 它不再是最快板端实现，最快的是 RNN，其次是 canonical 1DCNN；
3. 但 RNN 的 `KEIL-MAE = 8.74e-2` 且三项物理指标明显退化，1DCNN 虽有更强板端一致性和更高吞吐，三项正式物理指标仍显著落后于 Wiener-KAN；
4. 因此，Wiener-KAN 当前最稳的写法是“在当前 canonical 对比集合中，物理指标、复杂度、RAM 与板端速度的综合平衡最好”，而不是“绝对最快”。

### 2. 六指标雷达图已恢复，并与新图组逻辑一致

本轮恢复了两张关键雷达图：

- 主横评六指标雷达：不同模型对比；
- 损失消融六指标雷达：不同 Wiener-KAN loss 版本对比。

六指标统一为：

- `Freq Drift`
- `Sens Drift`
- `Linearity`
- `Compute Cost`
- `KEIL speed`
- `KEIL RAM Usage`

这使得“物理指标最优”“速度最快”“RAM 最省”与“综合平衡最好”可以在同一张图里被直观看到。

### 3. `Compute Cost` 的正式默认口径已改成 `1:3:20`

当前 adopted calibration result 为：

- `pure fit best`: `1 : 652.5750 : 3326.9858`
- `regularized fit best`: `1 : 3.1383 : 20.1587`
- `adopted rounded model`: `1 : 3 : 20`

与旧启发式 `1:1:6` 相比：

- 默认 `log-RMSE`: `0.2266 -> 0.1341`

因此，本轮 draft 与专题中间稿里所有 `Compute Cost` 相关叙事都已按 `1:3:20` 重写，不再沿用旧的 `1:4:23` 或更早的 `1:1:6`。
