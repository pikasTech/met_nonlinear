## 二、本次已新增补齐的项目

为了补齐此前最明确的卷积类横评缺口，本次新增并训练完成了两个项目：

### 2.1 标准 1D-CNN baseline

- 路径：`projects/05_1DCNN/1DCNNu8k8_e1k_lr2e3_pd8l2_relu`
- 模型：`WaveNet3` 风格的纯 1D 卷积基线，作为标准 1D-CNN 对照
- 结果文件：`projects/05_1DCNN/1DCNNu8k8_e1k_lr2e3_pd8l2_relu/data/metrics.json`

关键指标：

- `Freq Drift (Hz)`: `4.8069`
- `Sens Drift (%)`: `24.5885`
- `Linearity (%)`: `1.5048`
- `Compute Cost`: `400.0`
- `Total Params`: `225`

判断：

- 这是一个**很轻量、计算代价很低**的纯卷积基线；
- 但补偿效果明显弱于 FRIKAN，也弱于 LSTM / TCN；
- 作为“标准 1D-CNN baseline”非常有价值，因为它回答了“为什么不是普通卷积网络就够了”。

### 2.2 标准 TCN baseline

- 路径：`projects/06_TCN/TCNu8d1248k3_e1k_lr2e3_pd8l1_relu`
- 模型：`WaveNet2` 风格的 TCN，对应 dilated causal conv 基线
- 结果文件：`projects/06_TCN/TCNu8d1248k3_e1k_lr2e3_pd8l1_relu/data/metrics.json`

关键指标：

- `Freq Drift (Hz)`: `4.5868`
- `Sens Drift (%)`: `14.7550`
- `Linearity (%)`: `0.5913`
- `Compute Cost`: `2112.0`
- `Total Params`: `1105`

判断：

- 它比 1D-CNN 明显更强，尤其在线性度上更接近 LSTM；
- 但频漂和灵敏度漂移仍明显落后于 FRIKAN；
- 它补上了此前最关键的一个横评位：**TCN / dilation-based CNN**。

## 四、综合现有项目后，仍然存在的核心缺口

虽然横评主干现在已经明显加强，但距离最理想的 MN 稿件实验矩阵仍有剩余缺口。

### 4.1 缺口 A：物理约束消融仍然不完整

当前已有：

- `FRIKANh8u6l6_e1k_lr7e4_nosym_r5`

它能回答“去掉 symmetry 以后会怎样”，但还不能完整回答：

- 仅 odd symmetry 的作用
- 仅 positivity / 同向性 的作用
- 两者同时 vs 全部去掉

因此，如果论文仍然保留较强的“物理约束本身有效”主张，那么**目前只有 `nosym` 还不够完整**。

判断：

- 若后续决定弱化物理约束主张，则这一缺口可接受；
- 若后续仍要强写 constraint effectiveness，这一组还需要补完整四组合或至少再补 1~2 个关键版本。

### 4.2 缺口 B：FRIRNN/频响注入消融仍然不够“干净”

当前已有：

- `FRIKANh8u6l6_e1k_lr14e5_randfrirnn_r2`

这个项目能说明：

- 正确频响先验优于随机频响先验

但它不能完全回答：

- “有没有注入本身是否必要”
- “一个普通线性前端 / 无注入前端是否足够”

因此，若论文中还想把 FRIRNN / 频响注入作为关键机制之一，**目前的随机注入版本还不够构成最强证据链**。

更理想的补法是再有一个：

- 无注入 / 简化线性前端版本

### 4.3 缺口 C：force-feedback 仍未进入正式实验矩阵

当前所有 preset 都主要是神经网络与块结构模型之间的比较。

但从 IDEA 和转投主旨看，一个非常重要的工程对照其实是：

- force-feedback
- feedforward calibration

即便 force-feedback 不一定能重跑成同口径训练项目，论文中也仍建议至少形成一张统一对照表，明确：

- 动态范围
- 系统复杂度
- 稳定性风险
- 额外硬件需求
- 可部署性

这一缺口不一定意味着要新增 training project，但它是**论文证据组织上的明确缺口**。

### 4.4 缺口 D：验证 protocol 仍需加强，但不包括多 seed

用户已明确：

- 先不补多 seed，因为 `AI_Based_Sensor` 本身并不以多 seed 为主

这个判断成立。

但这不代表验证 protocol 已经够强。仍然需要优先考虑：

- 更严格的数据划分方式
- cross-validation 或等价稳健验证
- 重复测量 / 误差估计

因此，**当前剩余的验证缺口，不是 multi-seed，而是 protocol-level strengthening**。

### 4.5 缺口 E：计算成本表述还需要口径治理

新加的 `1DCNN` 和 `TCN` 都已经生成了 `compute_analysis.json` 和 `metrics.json`，但两者都带有 `compute_cost_status = partial` / `unsupported_layers` 提示：

- `1DCNN`: 激活层未完全计入
- `TCN`: Activation / Concatenate 等层未完全计入

这意味着：

- 当前 `Compute Cost` 可用于相对比较；
- 但论文里需要明确标注为“估算值”或“partial”；
- 并在正文或图表注释里说明 unsupported layers 的影响，避免把这些数字写成绝对精确成本。

这不是训练缺口，但属于论文写作时必须处理的口径缺口。

## 六、当前阶段的最小剩余动作

如果以“尽快把 MN 稿件实验矩阵补到可写论文”为目标，当前最小剩余动作建议是：

1. **决定是否继续强化物理约束主张**
   - 如果强化：继续补 1~2 个约束组合版本
   - 如果弱化：现有 `nosym` 可作为辅助证据即可

2. **决定是否继续强化 FRIRNN 注入主张**
   - 如果强化：再补一个“无注入 / 简化前端”版本
   - 如果弱化：现有 `randfrirnn` 可作为辅助说明

3. **补 protocol-level 验证，而不是多 seed**
   - 更严格划分 / cross-validation / 误差估计

4. **论文写作时加入工程对照表**
   - 把 force-feedback 纳入讨论框架

## 七、最终判断

基于当前 preset 与本次新增项目，当前状态可以压缩成四句话：

1. **横评主干已经明显补强，尤其是卷积类 baseline 的关键缺口已通过新训练的 `1DCNN` 和 `TCN` 项目补齐。**
2. **loss 消融已经基本够用，结构消融已经有骨架，但物理约束和频响注入两条机制线仍不算完全闭环。**
3. **当前最主要的剩余缺口，不再是“再加一个模型”，而是：机制闭环是否继续补、验证 protocol 是否加强、force-feedback 是否纳入统一证据框架。**
4. **因此，现阶段实验矩阵已经从“明显偏弱”提升到“可以写论文主实验”，剩余工作更偏向补关键缺口和重组叙事，而不是盲目扩张项目数量。**
