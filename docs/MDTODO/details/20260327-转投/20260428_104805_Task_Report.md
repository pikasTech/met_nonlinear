# R25 任务报告：Paper Figure Pipeline 配置化与 Figure Studio WEBUI

## 任务目标

R25 要求围绕论文出图流程完成一套“配置化 + 可视化 + 可回写”的 Figure Pipeline 改造，核心目标包括：

- 将 paper figure 的单图参数与拼图参数集中收敛到 `docs/paper/config.json`；
- 为 `docs/paper/gen_figures.py` 增加单图定向重绘能力，保证 WEBUI 能直接驱动真实出图；
- 在 `src/webui` 中新增一套前后端均为 TypeScript 的 Figure Studio，并与原 Project Visualizer 共用现有 WebUI 端口；
- 将原 home 页改为导航页，可进入旧的项目可视化页与新的绘图工作台；
- WEBUI 必须直接加载本地实际生成后的 figure 位图，支持自动重绘 / 手动重绘，并把配置写回 `config.json`；
- 完成后使用 Playwright 做 E2E 验证。

### 2. `gen_figures.py` 单图重绘能力

重写了 `docs/paper/gen_figures.py` 的入口，使其既能保留原批量流程，也能按图号做定向渲染：

- 新增 `--list-figures`，用于列出可渲染图目录；
- 新增重复参数 `--figure FIG_ID`，支持单图 / 多图定向出图；
- WEBUI 触发重绘时，直接调用同一个 `gen_figures.py` 入口，保证预览与 `main.tex` 实际引用图完全一致。

### 3. `paper_pipeline.py` 读取 figure config 并支持按图生成

在 `docs/paper/src/paper_pipeline.py` 中补齐了面向 Figure Studio 的能力：

- 增加 `FIGURE_PIPELINE_CONFIG` 与相关配置读取/合并辅助函数；
- 增加 `list_renderable_figures()` 与 `renderable_figure_catalog()`，供服务端建立图目录；
- 增加 `generate_selected_figures(...)`，支持按图号定向生成；
- 将多类单图/拼图生成逻辑接入 `figure_pipeline` 配置；
- `main()` 入口支持可选 `figure_ids`，使 WEBUI 可以复用同一套 pipeline。

### 4. Figure Studio 后端接入现有 WebUI 服务

在 `src/webui/server/src/paperFigures.ts` 与 `src/webui/server/src/index.ts` 中新增 figure studio API：

- `GET /api/paper-figures/catalog`
  - 读取 `docs/paper/config.json`，输出图目录、缩略图地址、配置对象与更新时间；
- `PUT /api/paper-figures/config/:id`
  - 将单图配置回写到 `config.json`；
- `POST /api/paper-figures/render`
  - 调用 `tf26` Python 运行 `docs/paper/gen_figures.py --figure ...` 发起单图重绘任务；
- `GET /api/paper-figures/render/:jobId`
  - 查询异步重绘任务状态与日志；
- `app.use('/paper-figures-assets', express.static(...))`
  - 直接暴露 `docs/paper/figures/` 下真实位图，保证前端显示的就是 LaTeX 使用的本地图像。

### 5. 前端路由、导航页与 Figure Studio 页面

在 `src/webui/src/` 下完成前端整合：

- 原 `App.tsx` 重构为轻量路由壳：

  - 右侧：配置检查器，支持直接编辑 `config.json` 对应字段；
  - 支持自动重绘与手动重绘两种模式；
- 新增 `src/webui/src/pages/ProjectVisualizerPage.tsx`
  - 复制并保留原项目可视化页逻辑，通过新路由继续访问；
- `src/webui/src/api.ts` 增加 paper figure catalog / config / render job 类型与请求函数；

- `src/webui/vite.config.ts` 改为 `base: '/'`，并为 `/paper-figures-assets` 增加 dev proxy，保证嵌套路由和真实位图预览正常工作。

### 6. 联调中发现并修复的关键问题

在 Figure Studio 联调过程中，发现一个会影响拼图选择的前端状态回退问题：

- 初版 `FigureStudioPage` 的首次加载 `useEffect` 依赖 `refreshCatalog`；
- 而 `refreshCatalog` 又依赖 `selectedFigureId / selectedKind / dirty`；
- 结果是在切换 montage 并选择其他图时，catalog effect 会再次执行并把选择强制重置回当前分类第一张图。

本次已将首次 catalog 加载改为仅在 mount 时执行，从而修复：

- montage 列表中的图可稳定切换；
- 配置检查器会跟随当前图正确切换；
- Playwright E2E 可以稳定覆盖 figure 选择、保存和重绘流程。
