## 结论先行

当前稿件已经完成了一个重要转向：**从旧版 FRIKAN 方法稿，转成了一个可继续向 MN 靠拢的 Wiener-KAN 内部工作稿**。但它离“可提交 MN 的成稿”还差一条完整证据链，主要短板不再是“没有任何实验”，而是：

## 一、当前状态清单

### 1. 期刊方向已经基本明确，但稿件落点仍未完全对齐

- 根据 R3 / R3.1 / R3.2 / R3.3 / R3.4，`MET + AI` 与 MN **并非 scope 不匹配**，但当前稿件还没有完全长成 MN 更偏好的 `physical sensor / calibration / linearization / dynamic-range` 论文形态。
- 现在的主要矛盾已经不是“能不能投 MN”，而是“稿件是否已经按 MN 的证据组织方式重构完成”。

### 2. 主叙事已经完成第一轮改名，但仍残留明显的方法稿惯性

- `docs/paper/latex/main.tex` 已将面向读者的主模型叙事从 `FRIKAN` 收敛到 `Wiener-KAN`。
- 但当前摘要、引言、结果段里，仍然保留了较强的“模型结构更优”“比 LSTM / GRU / RVTDCNN 更好”的方法稿表达。
- 这意味着：**主旨已经改了一半，但结果解释方式还没有完全跟上。**

### 3. LaTeX 基础设施已经具备，但还不是 submission-ready

- MN 模板已经套上，`docs/paper/latex/build/main.pdf` 已可生成。
- `docs/paper/latex/values.md` 已经把一批硬编码数值收敛成宏，说明稿件基础设施已进入可维护状态。
- 但 `docs/paper/latex/build/main.log` 仍显示大量 undefined citations / undefined references，说明**参考文献链和交叉引用链还没有真正闭合**。
- 当前 `main.tex` 也仍然是**内部中文工作稿**，不是直接提交版英文稿。

### 4. 实验资产已经从“偏弱”提升到“可以写主实验”，但还没完全闭环

- 根据 `20260413_MN_实验缺口清单.md`，横评主干已经不再只有旧版 RNN 对照，而是已经补进了 `1DCNN` 和 `TCN`，loss 消融与结构消融也都有骨架。
- 这意味着稿件已经具备“写出一个像样主实验章节”的条件，不再是空架子。
- 但仍存在几个关键缺口：
  - 物理约束消融不完整；
  - 频响注入 / 前端先验消融还不够干净；
  - force-feedback 尚未进入统一证据框架；
  - protocol 级验证仍偏弱；
  - compute cost 的表述还需要统一口径。

### 5. 稿件正文落后于现有实验资产

- 当前 `main.tex` 中的正式对比对象仍主要是 `Wiener / GRU / LSTM / RVTDCNN`。
- 但现有项目资产已经扩展到了 `1DCNN / TCN`，且已有专门的实验缺口清单。
- 这说明**实验先走了一步，正文没有及时跟上**；如果现在先润色摘要或结论，很容易继续引用一版过时的实验矩阵。

### 6. 数据与实验条件已基本可写，但还不够“期刊级规整”

- 当前稿件已经写入了采样率、频率范围、震级范围、测试样机、环境温度等关键信息。
- 但训练/验证/测试切分、重复性、误差估计、边界条件、失败模式等内容，仍未被系统组织成 MN 更偏好的“protocol first”表达。
- 同时，当前实验仍然主要建立在 **单器件、单环境（25°C）** 条件下，相关限制虽可讨论，但必须被更清晰地放进 manuscript 的限制与边界部分。

### 7. 一些主张是否保留，仍未最终冻结

- 如果继续强写“物理约束有效”“频响先验注入有效”“AFMAE 明显更优”等机制性主张，则还需要对应的实验闭环。
- 如果不准备再补这些关键实验，则应当主动下调对应表述，把它们从“核心创新结论”降成“设计选择 / 辅助机制”。
- 这一步尚未完全定稿，因此它是后续所有改稿动作的最上游依赖。

## 二、目标状态清单

MN 目标下，最终稿件应达到以下状态：

### 1. 主旨层

- 论文主线从“提出一个新模型”转为“解决 MET 在大震级下的非线性频响漂移，并证明前馈校准/线性化方案的工程价值”。
- AI 只是工具，不再是文章主角。

### 2. 证据层

- 横评矩阵覆盖主要方法范式；
- 关键保留主张都有对应证据；
- protocol 足够清楚，能支撑结果可信度；
- 讨论中能清楚交代边界条件和局限。

### 3. 结果表达层

- 结果不是“某模型比某模型好一点”，而是围绕：
  - 线性化效果；
  - 频漂/灵敏度漂移抑制；
  - 动态范围扩大；
  - 计算复杂度与部署可行性；
  - 与 force-feedback 的系统级取舍；
  来组织。

### 4. 文稿结构层

- 实验章节按“主横评 -> 机制消融 -> protocol/稳健性 -> 部署与系统讨论”组织；
- 引言按“工程问题 -> 现有方法缺口 -> 本文方案与贡献”组织；
- 摘要、标题、结论与正文主旨完全一致。

### 5. 成稿层

- 英文稿可提交；
- 引用与交叉引用闭合；
- 图表与数值来源统一；
- 编译状态可控；
- 不再保留与最终主旨冲突的旧叙事残留。

## 三、修改总原则

### 原则 1：先冻结 claim budget，再做正文改写

先决定哪些主张保留，哪些主张降级，哪些主张删除。  
**没有被冻结的主张，不要先写进摘要、引言、结论。**

### 原则 2：先固化证据，再解释证据

实验矩阵、表格口径、图表结构属于上游；结果分析和讨论属于下游。  
**不要在上游未稳定前反复改写讨论。**

### 原则 3：先写“事实层”章节，再写“宣传层”章节

实验设计、数据集、方法定义、表格口径是事实层；摘要、引言、结论是宣传层。  
**事实层先收口，宣传层最后写。**

### 原则 4：不为保留旧创新点而硬补叙事

如果某一条创新点已经没有足够证据，就应主动减弱，而不是靠措辞硬保。  
MN 更接受一个证据闭环、表达克制的稿件，而不是一个创新点很多但证据不闭环的稿件。

## 四、按依赖顺序组织的修改步骤

下面的顺序不是写作顺序，而是**依赖顺序**。越靠前越上游，越不宜跳过。

### 阶段 A：冻结目标版本与保留主张（最上游）

#### 目标

先把“这篇稿子最终到底要证明什么”定下来。

#### 具体动作

1. 明确本稿主标题方向：
   - 主问题必须是 `MET nonlinear frequency-response drift / calibration / linearization / dynamic-range extension`；
   - 不是“提出一个新命名网络”。
2. 明确哪些主张保留：
   - 建议保留：前馈补偿的工程价值、Wiener-KAN 结构化表达、频漂/灵敏度漂移抑制、部署潜力。
3. 明确哪些主张需要降级或删掉：
   - 若不补齐实验，不要强写完整的 constraint effectiveness；
   - 若不补“无注入 / 简化前端”版本，不要把频响注入写成已被严格证实的核心贡献；
   - 若 protocol 不补强，不要强写泛化/鲁棒性结论。

#### 输出物

- 一份简短的“保留主张/降级主张/删除主张”清单。

#### 为什么必须先做

因为后面所有实验补充、表格组织、摘要写法，都依赖这一层；这一步不定，后面会一直返工。

---

### 阶段 B：冻结实验矩阵与 protocol（证据框架层）

#### 目标

决定这篇稿子最终采用哪一组实验作为正式证据，不再让正文落后于资产。

#### 具体动作

1. 从现有项目中冻结正式横评名单：
   - 至少覆盖 Wiener-KAN、Wiener、GRU、LSTM、1DCNN、TCN；
   - 视最终叙事决定 LSTMTransformer、RVTDCNN 放正文还是补充材料。
2. 冻结正式消融名单：
   - loss 消融；
   - KAN/MLP 或等价结构消融；
   - 对称性/约束消融；
   - 频响先验相关消融（若保留该主张）。
3. 冻结 protocol 口径：
   - 数据划分怎么写；
   - 是否补更严格划分或等价稳健验证；
   - 误差估计和边界条件如何写；
   - compute cost 如何声明为估算值/partial。
4. 冻结系统级对照口径：
   - force-feedback 不一定要做成训练项目，但必须纳入统一工程对照框架。

#### 输出物

- 一份正式实验矩阵清单；
- 一份 protocol/metric/table 口径清单。

#### 为什么必须先做

因为一旦表格口径冻结，后面的图表、结果解释和摘要中的数字才能稳定。

---

### 阶段 C：补齐最关键的上游证据缺口

#### 目标

把“需要保留的主张”补到最低可成立强度，或者明确放弃对应强主张。

#### 具体动作

1. 对物理约束主张做决策：
   - 若保留强主张，则继续补关键约束组合；
   - 若不补，则把它降为辅助设计选择。
2. 对频响先验/注入主张做决策：
   - 若保留强主张，则补“无注入 / 简化前端”对照；
   - 若不补，则只保留为结构说明，不作为强结论展开。
3. protocol 级补强：
   - 至少补一轮更严格的数据划分或等价稳健验证；
   - 明确误差估计、限制条件和失败边界。
4. force-feedback 工程对照表：
   - 整理成统一表，而不是只在引言零散讨论。
5. compute cost 口径收口：
   - 统一说明哪些结果为 partial estimate，避免正文误写成精确计算成本。

#### 输出物

- 一组稳定的最终指标表源数据；
- 一组确定会进入正文的对照与消融证据。

#### 为什么必须先做

因为这些内容一旦定下来，后面的图表、结果段、讨论段就能稳定；如果先写结果解释，这一步一改又要重写。

---

### 阶段 D：先写“事实层”文本与图表

#### 目标

先完成那些直接陈述客观事实、改动依赖最强的章节。

#### 具体动作

1. 重写/整理实验设置与数据集部分：
   - 数据采集流程；
   - 样机与环境；
   - 扫频/扫幅范围；
   - 划分策略；
   - protocol 与限制。
2. 重写方法章节中真正保留的内容：
   - Wiener-KAN 结构；
   - 先验 Wiener 线性层；
   - KAN 非线性层；
   - AFMAE；
   - 仅保留已被证据支撑的机制表述。
3. 重组正式图表与表格：
   - 主横评表；
   - 主效果图；
   - 机制消融表；
   - 部署/复杂度表；
   - force-feedback 对照表（若正文纳入）。

#### 输出物

- 稳定版“实验设置 + 方法 + 图表”。

#### 为什么这一层早于引言/摘要

因为这些部分最依赖客观证据，一旦稳定，后面高层叙事才不会漂。

---

### 阶段 E：再写“解释层”文本

#### 目标

在事实层固定后，再去解释这些事实代表什么。

#### 具体动作

1. 重写结果章节：
   - 不再按“哪个模型更强”散点式描述；
   - 改按“主横评 -> 机制消融 -> 部署与系统价值”组织。
2. 重写讨论章节：
   - 重点写工程价值、使用边界、与 force-feedback 的取舍；
   - 明确单器件、单环境、未覆盖现场工况等限制。
3. 将已有结果重新解释为：
   - 线性化能力；
   - 频响稳定化能力；
   - 动态范围扩展的可行性；
   - 部署代价与收益。

#### 输出物

- 稳定版结果与讨论章节。

#### 为什么不能更早写

因为解释层最容易受上游数据和图表变化影响，属于典型下游内容。

---

### 阶段 F：最后重写摘要、引言、标题、结论

#### 目标

在所有事实和解释都稳定后，再统一高层表达。

#### 具体动作

1. 引言重写为：
   - 工程问题；
   - 现有方法缺口；
   - 本文方案；
   - 本文贡献。
2. 摘要重写为：
   - 问题背景；
   - 方法定位；
   - 数据/实验范围；
   - 关键结果；
   - 工程意义。
3. 结论重写为：
   - 只总结已被正文证据支撑的内容；
   - 不写过度外推。
4. 题目最终定稿：
   - 以 `MET calibration / linearization / frequency-response stabilization` 为核心；
   - `Wiener-KAN` 作为方案名辅助出现即可。

#### 输出物

- 可提交的高层叙事版本。

#### 为什么必须最后做

因为摘要、引言、结论是最典型的“下游包装层”，任何上游变化都会直接波及它们。

---

### 阶段 G：英文定稿、参考文献与编译收尾

#### 目标

把内部工作稿真正收口成 submission-ready 文档。

#### 具体动作

1. 从中文工作稿过渡到英文定稿；
2. 补齐 bibliography 流程，消除 undefined citations / references；
3. 全文统一术语、图表编号、宏变量与表述口径；
4. 清理与最终主旨冲突的旧文本残留；
5. 最终编译检查。

#### 输出物

- 编译稳定的英文提交稿。

#### 为什么这是最后一步

因为英文润色和编译收尾属于最下游动作；如果前面证据和叙事还在变，这一步会高频返工。

## 五、推荐的实际执行顺序

如果要把工作压缩成最务实的一条路线，我建议按下面顺序推进：

1. **先开一次“主张冻结”清单**：决定哪些创新点保留、哪些降级。
2. **再冻结正式实验矩阵**：把现有资产和正文对齐。
3. **只补最关键的上游缺口**：优先补能决定主张成立与否的实验，不盲目扩 baseline。
4. **先重写实验设置/方法/图表**：先写事实层。
5. **再重写结果和讨论**：把结果解释成 MN 需要的工程叙事。
6. **最后统一改摘要、引言、结论和标题**。
7. **收尾英文、引用与编译问题**。

## 六、当前阶段不建议优先做的事情

1. **不建议现在先润色摘要或标题。**  
   因为实验矩阵和主张还未完全冻结，改完还会返工。

2. **不建议现在先做大规模语言 polishing。**  
   当前更大的问题是证据链和叙事结构，不是句子优美度。

3. **不建议继续无目标扩 baseline。**  
   现在缺的不是“再多一个模型名”，而是关键主张闭环和 protocol 收口。

4. **不建议继续保留未经闭环的强结论。**  
   对应实验不补，就应主动下调表述强度。

## 七、最终建议

对 R5 这个“MN 修改计划”任务，最关键的一句话是：

**先把“要证明什么”和“用哪些证据证明”冻结，再去写“怎么讲这个故事”。**

换句话说，下一阶段最优先的不是动摘要，而是：

1. 冻结 claim budget；
2. 冻结实验矩阵和 protocol；
3. 用最少但最关键的补强把证据链补闭；
4. 再按 MN 的工程论文结构去重写全文。

只要按这个顺序推进，就能最大程度避免“上游一变、下游全返工”的问题，这也正是当前这篇稿子最需要的改法。
