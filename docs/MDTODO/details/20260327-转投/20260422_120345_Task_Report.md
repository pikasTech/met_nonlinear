# R7.3 任务报告：论文边缘设备推理评估方法中间稿

## 目标

基于真实 `compute_analysis`、`qemu-c-inference` 与 `keil-bench` 代码路径，撰写一份面向论文的长期边缘设备推理评估方法参考文档。文档只解释方法和产物来源，不记录 benchmark 数值。

## 产出

- `docs/reference/paper_edge_inference_evaluation_method.md`
- `AGENTS.md`

## 本次沉淀的内容

1. 将边缘设备评估拆分为静态 compute cost、QEMU 验证、Keil 验证、构建资源占用与时序测量五个主题。
2. 记录哪些字段已经进入 `metrics.json`，哪些字段目前仍停留在 EP 侧构建输出或 summary 文件中。
3. 固化当前从 Keil `Program Size` 行读取 Flash / RAM 的规则。
4. 明确当前部署边界：只有具备 native board-inference 支持的模型，才属于正式部署子章节。

## 为什么符合当前 MN 目标写法

文档把静态复杂度、动态时延、资源占用与数值一致性拆开描述，比把它们统称为一个笼统的“速度”指标更符合期刊方法章节的表达方式。
