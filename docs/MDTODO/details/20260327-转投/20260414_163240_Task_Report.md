# R4.1 任务报告：将 LaTeX 稿件中的 FRIKAN 叙事收敛为 Wiener-KAN

## 任务目标

根据 `docs/IDEA.md` 中“用 Wiener-KAN 统一建模与补偿叙事”的思路，对 `docs/paper/latex/main.tex` 做一次**不是简单关键词替换**的重写：

- 将论文主模型名称由 `FRIKAN` 改为 `Wiener-KAN`
- 将原先 `FRI` 对应的“频响注入新模块”改写为**先验 Wiener 线性层**
- 下调“提出全新命名模型”的攻击面，改为“沿用 Wiener 线性-非线性分解，并用 KAN 作为非线性映射”
- 同步修改摘要、引言、方法章节、结果章节、结论，以及会在成稿中实际显示的变体命名

## 本次完成的修改

### 1. 统一模型命名为 Wiener-KAN

已在以下位置把面向读者的主命名统一为 `Wiener-KAN`：

- 标题
- 摘要
- 引言总述
- 方法章节的框架描述
- 结果章节中的模型称呼
- 结论与未来工作

同时保留了部分内部 `label` / 变量名（如 `fig:FRIKAN_structure`、`eq:FRIKAN`、`valFrikan*`），仅作为 LaTeX 内部引用与历史宏名使用，不再在正文中作为面向读者的模型命名出现。

### 2. 把 FRI 叙事改写为“先验 Wiener 线性层”

这次改动的重点不是把 `FRIKAN -> Wiener-KAN` 做机械替换，而是把原来“提出一个 FRIRNN / FRIKAN 新模块”的叙述，改成：

- 采用 Wiener 结构“线性动态 + 静态非线性”的经典分解
- 前级是由相对频率响应初始化的**先验 Wiener 线性层**
- 后级是物理约束 KAN 非线性映射

对应地，方法章节中原先把线性前端写成独立新模块的表述，已改为：

- 这不是独立新模型
- 而是 Wiener 结构中的线性动态部分的可训练实现
- QLFRS 的作用是初始化 Wiener 线性层，而不是给一个全新命名模块“注入创新性”

### 3. 弱化过新命名，保留核心技术内容

本次没有删除稿件中的核心技术内容，而是把其叙事位置调整为更稳妥的表述：

- 物理先验仍然保留
- KAN 非线性映射仍然保留
- QLFRS -> IIR -> 线性 RNN 的实现路径仍然保留
- AFMAE 损失函数仍然保留

变化主要在于：

- 不再把这些内容包装成一个“命名创新过强”的 FRIKAN/FRIRNN 体系
- 而是强调它们服务于 `Wiener-KAN` 这一更容易与已有理论对齐的结构化叙事

### 4. 同步修改变体显示名称

为避免成稿表格和图注中继续出现 `FRIKANh...`，已同步修改：

- `docs/paper/latex/values.md`

将显示宏改为：

- `Wiener-KANh6u6l3`
- `Wiener-KANh6u6l4`
- `Wiener-KANh8u6l6`

内部宏名 `valFrikanVariant*` 仍保留，以避免额外大范围改动 `main.tex` 中的宏调用。
