# R7.4 任务报告：论文指标计算方法中间稿

## 目标

基于当前仓库真实的 summary 导出路径，撰写一份面向论文的长期指标计算方法参考文档。文档只解释稳定字段定义、公式与产物边界，不记录任何一次性数值。

## 产出

- `docs/reference/paper_metric_calculation_method.md`
- `AGENTS.md`

## 本次沉淀的内容

1. 将 `projects/.../data/metrics.json` 固定为当前各 preset 分组的唯一正式论文指标来源。
2. 记录从 `cli.py`、`task_dispatcher.py`、`project_manager.py` 到 `metrics_summary.py` 的完整上游调用链。
3. 写清当前 `Freq Drift (Hz)`、`Sens Drift (%)`、`Linearity (%)`、`Compute Cost`、`Total Params`、`QEMU-MAE`、`KEIL-MAE` 与 `KEIL-SPEED` 的真实计算规则。
4. 区分优化诊断字段、物理指标、复杂度指标与部署一致性指标的当前边界。
5. 记录 `partial` summary 的判定规则，避免论文在正式表格中无提示混入不完整项目 summary。

## 为什么符合当前 MN 目标写法

文档把指标生成表述为“代码路径 + 产物路径 + 公式定义”的可复现协议，适合直接改写为期刊方法章节，同时把真正的数值保留到结果表中。
