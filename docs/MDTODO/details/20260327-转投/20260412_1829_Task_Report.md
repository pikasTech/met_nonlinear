# R3.1 任务报告：从 AI+领域论文结构看当前主稿的真实差距

## 任务概述

R3 旧报告把重点放在了期刊领域匹配性上，但 R3.1 的要求更明确：**领域匹配本身不是当前最核心的问题，真正需要重做的是“AI+领域论文如何成稿”的结构分析**。本次分析基于以下材料：

- 当前主稿：docs/paper/main.tex
- 对标论文：docs/research/literature/markdown/[VPI]AI_Based_Sensor.md
- 领域匹配参考：docs/research/literature/markdown/[VIP]Electrochemical_vibration_sensor.md
- 既有转投主旨：docs/IDEA.md

这里先明确一个前提：

- `Microsystems & Nanoengineering` 已经发表了与 MET 完全同领域的综述论文，说明 “MET + AI” 在领域上并不天然失配。
- 因此，本次不再把“能不能投这个领域”作为主问题，而是聚焦：**为什么对标论文更像一篇成熟的 AI+工程领域论文，而当前主稿还没有形成同等级的结构性说服力。**

---

## 一、结论先行

本次 R3.1 的核心结论可以压缩为一句话：

**当前主稿的主要差距不是领域错位，而是论文结构仍然是“方法驱动型 AI 论文”，尚未升级为“工程问题驱动、证据链完整、实验矩阵自洽”的 AI+领域论文。**

更具体地说，当前稿件存在四个核心问题：

1. **问题定义仍然偏“模型先行”**：标题、摘要、引言都在优先解释 FRIKAN，而不是优先定义 MET 工程问题。
2. **对比实验矩阵还不够像工程论文**：当前有基线比较，但还没有把“经典方法、主流 AI 方法、物理启发方法、工程已有方案”组织成完整层次。
3. **消融实验几乎缺席**：现有结果更像“不同规模模型对比”，而不是“每个关键设计是否必要”的因果验证。
4. **系统级证据还不闭环**：虽然已有 MCU 部署结果，但缺少统计稳健性、泛化边界、数据划分合理性、多条件验证等 AI+工程论文常见支撑。

因此，R3.1 的重点不是再争论期刊是否接收 MET，而是明确：

- 当前主稿**该补什么实验**；
- 当前主稿**该怎么重写叙事顺序**；
- 当前主稿**该如何把“模型创新”降格为“解决工程问题的工具”**。

---

## 二、对标论文真正值得借鉴的，不是题材，而是成稿结构

R3 旧报告过度强调“DRIE/SEM 与 MET 不是同一问题域”，这个判断本身没有错，但对 R3.1 不够有用。R3.1 更重要的问题是：**一篇 AI+具体工程领域论文，是如何把方法论文包装成领域论文的。**

从 [VPI]AI_Based_Sensor 可以提炼出四条非常清晰的结构原则。

### 2.1 原则一：问题先行，模型后置

对标论文的出发点不是“我设计了 VLSet-AE”，而是：

- DRIE 是 MEMS 制造关键工艺；
- SEM 轮廓分析是工艺优化瓶颈；
- 人工测量误差大、吞吐低；
- 现有自动方法在噪声和复杂形貌下不稳。

也就是说，**模型只是在最后被引入为解决瓶颈的方案**。

而当前主稿从标题开始就是：

- Feedforward Nonlinearity Compensation...
- via Frequency Response Prior Injection and Physics-Constrained KAN

这说明主稿的第一印象仍然是“提出了一个新模型”，而不是“解决了一个工程系统中的关键痛点”。

对于 AI+领域论文，这种顺序差异很重要。因为领域审稿人首先要确认的是：

- 你解决的是不是本领域真正关心的问题；
- 你的方法是否只是众多可能方案之一；
- 你的新模型是否真的有必要。

当前主稿在第一步就把回答顺序放反了。

### 2.2 原则二：先给出完整实验工作流，再讲模型细节

对标论文在方法章节前，已经把实验来源、工艺变量、图像采集、正交设计、分层 segmentation 的工作流讲清楚了。模型只是整个工作流中的一个环节。

当前主稿虽然在实验部分给出了测试平台和数据集结构，但整体观感仍然是：

- Section II 讲背景和机理；
- Section III 长篇讲 FRIKAN 结构；
- Section IV 才给实验数据和对比。

这会导致读者在进入模型章节时，还不知道：

- 数据到底如何采集；
- 为什么这个任务需要这种模型；
- 现有工程方法究竟卡在什么地方。

对 AI+领域论文来说，更好的做法通常是先建立工作流：

1. 器件或系统问题是什么；
2. 数据怎么来的；
3. 评价指标如何定义；
4. 然后再解释模型如何嵌入这个流程。

### 2.3 原则三：结果必须是“多维证据”，而不是单一精度表

对标论文的结果不是只给一个准确率，而是同时给出：

- 关键尺寸误差；
- 多模型比较；
- 训练时间；
- 推理时间；
- 参数量；
- 内存；
- 交叉验证；
- 测量误差与工艺波动讨论。

这类结构会让论文看起来不像“模型跑赢了基线”，而像“这个方法已经具备工程采用的充分证据”。

当前主稿其实已经有一部分正确方向：

- 有 IONS / SDS / NFDS 三个任务指标；
- 有参数量比较；
- 有 STM32F405 上的实时部署；
- 有 LUT 精度与存储的权衡。

但这些证据还没有形成完整闭环，因为仍缺少：

- 统计显著性或重复实验；
- 数据划分合理性证明；
- 机制级 ablation；
- 工程已有方案的同口径比较；
- 泛化边界说明。

### 2.4 原则四：AI+领域论文的实验设计，本质上是在回答“为什么必须是你这个方案”

对标论文虽然也强调模型创新，但其所有实验最终都在回答：

- 为什么常规方法不够；
- 为什么物理约束有意义；
- 为什么它在真实噪声和复杂形貌下更稳；
- 为什么它不仅准，而且快，而且能用于制造流程。

这正是当前主稿最需要补的实验逻辑。

## 三、当前主稿的真实位置：已经有结果，但还不是成熟的 AI+工程论文证据链

### 3.1 已有优点：主稿并不是没有工程价值

当前主稿并非“结构很差”，相反，它已经具备一些很强的素材：

- 问题是真实工程问题：MET 在大振幅下存在显著幅频耦合非线性；
- 数据是真实测试数据：13 个频率、25 个振幅、单条序列 8000 点；
- 指标不是单一 MAE，而是包含 IONS / SDS / NFDS；
- 有真实硬件部署：STM32F405RG + LUT 加速；
- 有与 Wiener、GRU、LSTM、RVTDCNN 的对比。

这说明主稿不是“只有模型没有实验”，而是**实验结构还没有升级到足以支撑 AI+领域期刊说服力的程度**。

### 3.2 真正的不足：实验更像结果展示，而不是因果验证

当前 Section IV 的结果主要分成三块：

1. FRIKAN 的补偿效果；
2. 不同模型的性能比较；
3. MCU 上的实时性与 LUT 权衡。

这三块本身没有问题，但它们回答的是：

- “FRIKAN 效果不错”；
- “FRIKAN 比几个模型强”；
- “FRIKAN 还能部署”。

它们还没有充分回答：

- 为什么一定要频响注入；
- 为什么一定要 KAN 而不是普通 MLP；
- 为什么物理约束确实贡献了泛化；
- 为什么 AFMAE 比纯时域损失更合适；
- 为什么结果不是数据泄漏或划分方式带来的乐观偏差；
- 为什么这个方案对工程使用者是可靠的，而不仅是单次试验里的最优模型。

这就是“有结果”和“有证据链”之间的差距。

## 四、R3.1 最重要的部分：当前主稿的对比实验设计差在哪里

### 4.1 当前对比实验已经有雏形，但层次还不完整

主稿当前的基线包括：

- Wiener；
- GRU；
- LSTM；
- RVTDCNN；
- 多个 FRIKAN 规模变体。

这比很多单纯方法论文已经好一些，因为至少覆盖了：

- 传统非线性建模方法；
- RNN 类方法；
- 物理启发方法；
- 资源约束部署考量。

但如果目标是成熟的 AI+领域论文，这个矩阵还缺三类关键角色。

### 4.2 缺口一：缺少“工程已有方案”的同口径比较

主稿引言大量讨论力反馈方法，但正文主结果并没有把“工程既有方案”作为主对照之一纳入统一表格。

这会导致一个问题：

- 论文口头上说自己比 force feedback 更适合前馈部署；
- 但实验表里真正同口径的对照主要还是 AI 或建模方法。

如果目标是 AI+工程领域论文，审稿人更关心的是：

- 你相对工程现有方案到底提升了什么；
- 你是不是只比几个学术 baseline 好。

因此，最关键的对比不是再多加一个神经网络，而是把已有工程方案纳入统一口径比较。哪怕无法完全重现实验，也应给出：

- 统一指标下的历史结果表；
- 明确的适用范围差异；
- 为什么当前方案在稳定性、成本、动态范围上更可取。

### 4.3 缺口二：缺少“任务上最直接的强基线”

对当前任务而言，最直接的对照不只是 LSTM/GRU，而应该包括：

- Wiener 或 Wiener-Hammerstein 这类经典块结构模型；
- 1D-CNN / TCN 这类时序卷积模型；
- 纯 MLP / 普通 spline MLP 这类没有 KAN 先验的非线性映射模型；
- 简化的物理先验注入模型。

当前主稿已经有 Wiener 和 RNN，但缺少一个非常清晰的“卷积型时序模型”或“非 KAN 非线性映射模型”作为桥梁。RVTDCNN 来自功放场景，虽然能作为物理启发基线，但它并不是 MET 任务上最自然的中间参考点。

简言之，当前对比的不足不在“模型数量不够多”，而在“没有覆盖最自然的中间层基线”。

### 4.4 缺口三：多组 FRIKAN 变体不能代替机制性对比

表中有 FRIKANh6u6l3、FRIKANh6u6l4、FRIKANh8u6l6，这对参数效率分析有价值，但它们本质上是**规模扫描**，不是**机制对比**。

规模扫描只能说明：

- 更大或更深未必更好；
- 小模型也可有效。

它不能说明：

- FRIRNN 注入是否必要；
- PIKAN 的物理约束是否必要；
- KAN 相比普通 MLP 的优势是否真实存在；
- AFMAE 是否真的是关键组成。

而这些恰恰是 AI+领域论文里最核心的问题。

## 五、R3.1 的核心：当前主稿最缺的是消融实验，而不是再多一个 baseline

如果只允许指出一个最关键的实验短板，那就是：**主稿几乎没有真正意义上的消融实验。**

### 5.1 当前稿件里有哪些“看似像消融、其实不是消融”的内容

目前最接近消融的是：

- FRIKANh6u6l3 / h6u6l4 / h8u6l6 的比较；
- LUT 点数和精度 / 存储权衡；
- 训练与验证损失曲线比较。

这些都有效，但严格来说它们回答的是：

- 模型规模怎么选；
- 部署参数怎么选；
- 训练有没有过拟合。

它们没有回答“设计本身是否必要”。

### 5.2 当前主稿最应该补的 4 个机制性消融

如果保留现有 FRIKAN 叙事，那么最低限度需要下面四类消融。

#### A1. 频响注入消融

目的：验证 FRIRNN 的频响先验是否真的提供了有效 inductive bias。

建议设置：

- 完整 FRIKAN；
- 去掉频响注入，仅保留同结构线性 RNN；
- 用随机初始化或可学习初始化替代解析注入；
- 只保留固定 IIR / Wiener 线性前端。

应回答的问题：

- NFDS 的提升到底来自“频响注入”，还是来自“多几个状态量”；
- 频响注入是否主要改善收敛速度，还是最终精度；
- 注入在高振幅段是否更有价值。

#### A2. KAN 与普通非线性映射消融

目的：验证可训练样条激活到底是不是核心贡献。

建议设置：

- KAN 非线性层；
- 相同参数量的 MLP / spline-MLP；
- 单纯线性读出层；
- 可选的传统静态非线性函数近似。

应回答的问题：

- IONS 的优势是否主要来自 KAN；
- KAN 是否在小参数量下更有效；
- KAN 的优势是拟合能力还是更好的校准曲线形状约束。

#### A3. 物理约束消融

目的：验证 odd symmetry / positivity 等约束是有效 regularization，还是只是写法上的合理化。

建议设置：

- 完整物理约束；
- 去掉 odd symmetry；
- 去掉 positivity / monotonic-related 约束；
- 全部去掉，仅保留裸 KAN。

应回答的问题：

- 约束主要改善训练稳定性、外推性还是高幅段表现；
- 约束是否减少 boxplot 分布发散；
- 约束是否真的抑制过拟合，而不仅是偶然更优。

#### A4. 损失函数消融

目的：验证 AFMAE 的必要性。

建议设置：

- MAE；
- MSE；
- AFMAE；
- MAE + AFMAE；
- 可选：频率带加权版本。

应回答的问题：

- AFMAE 是否主要改善 SDS / NFDS，而不是 IONS；
- 频域损失是否牺牲了时域波形质量；
- AFMAE 是否让训练更稳，还是只是更偏向某几个指标。

### 5.3 若转向第二稿主旨，消融矩阵也应同步改写

如果后续不再坚持 FRIKAN，而转向 R2 已经规划的 Wiener-KAN 路线，那么以上消融仍然成立，但命名应收敛成更低风险的工程问题：

- Wiener 线性块是否必要；
- KAN 相比普通静态非线性块是否必要；
- 频域损失是否必要；
- LUT 加速的误差代价是否可接受。

也就是说，**真正成熟的 AI+领域论文不会执着于自创术语本身，而会把消融组织成“哪个工程设计选择更必要”。**

## 六、当前主稿还缺哪些“AI+工程论文标配”的证据

### 6.1 数据划分方式证据偏弱

主稿当前写法是：

- 先把数据归一化；
- 将时间序列切成长度为一半的子序列；
- 随机抽取训练集与测试集。

这类划分在纯深度学习论文中很常见，但在工程期刊里会带来一个直接疑问：

- 训练和测试是否来自同一条原始频率-振幅记录；
- 是否存在明显的数据泄漏或同条件重复采样带来的乐观偏差；
- 模型是否真的泛化到“未见条件”，还是仅仅泛化到“同条件下的新窗口”。

因此，后续至少应补一类更严格划分：

- 按频率条件留出；
- 按振幅条件留出；
- 按整条 sequence 留出；
- 最理想是按传感器个体留出。

这样才能把论文从“回归拟合”升级成“泛化验证”。

### 6.2 缺少重复实验和统计区间

当前论文给出了 boxplot、损失曲线和指标表，这已经比单点结果好很多，但仍然缺少：

- 多次训练随机种子结果；
- 指标均值 ± 标准差；
- 关键图中的 error bar；
- 对高振幅区域失败样本的误差分析。

AI+领域论文尤其忌讳“最优一次结果”直接当主结论，因为工程读者更在意稳定可复现，而不是 SOTA 式最优值。

### 6.3 缺少边界条件与失败模式分析

对标论文很像工程论文的一点，是它不仅展示成功，还讨论噪声、复杂形貌、测量误差等边界。

当前主稿则更偏“展示效果最好时模型有多好”。

后续应补两类分析：

- 哪些频率 / 振幅组合下补偿最困难；
- 失败样本是由物理饱和、数据稀疏还是模型偏差导致。

这会显著提高“工程可信度”。

### 6.4 单器件、单环境证据仍然偏窄

当前实验对象是单个 MTSS-1001 反应腔、25°C 环境、无力闭环条件下的测试。对方法论文来说已经足够，但对 AI+领域论文来说，这会自然引出两个质疑：

- 它是否适用于其他器件个体；
- 它是否对环境变化稳健。

主稿结尾已经把多传感器野外实验与温湿度影响放在 future work，这本身是诚实的，但也意味着**当前稿件的证据边界必须写得更克制**。

## 七、R3.1 需要给出的真正改稿方向：怎样把它写成一篇 AI+领域论文

### 7.1 标题和摘要要从“模型名中心”改成“工程问题中心”

当前标题的中心词是：

- Frequency Response Prior Injection
- Physics-Constrained KAN

这对方法稿是加分，对领域稿则有风险，因为领域编辑首先想看到的是：

- 你服务的是哪类器件；
- 你解决的是哪类性能问题；
- 你提出的是一套什么工程方案。

更稳妥的标题结构应是：

- 对象：electrochemical vibration sensor / electrochemical seismometer；
- 问题：magnitude-dependent frequency-response drift / nonlinear calibration；
- 手段：data-driven feedforward compensation / physics-guided calibration。

方法名可以后置，不能压过问题本身。

### 7.2 引言要改成“工程问题 -> 现有方案缺口 -> 本文方案”三段式

当前引言内容很丰富，但顺序偏向“我有一个新模型”。

更适合 AI+领域论文的顺序是：

1. MET 的工程价值与非线性问题；
2. 结构优化和力反馈各自解决到哪里、卡在哪里；
3. 数据驱动补偿为何有潜力；
4. 为什么现有黑盒模型还不够；
5. 本文提出一种工程化、可部署、带物理启发的数据驱动校准方法。

也就是要把“FRIKAN 是什么”延后，把“为什么这个问题值得用 AI 做”提前。

### 7.3 实验章节应改成“主对比 -> 机制消融 -> 部署验证 -> 失败分析”

这是 R3.1 最关键的结构建议。

建议实验章节顺序重组为：

1. **主任务结果**：与工程方案、传统模型、主流 AI 模型的统一比较；
2. **机制消融**：频响注入、KAN、物理约束、AFMAE；
3. **泛化与稳健性**：更严格划分、重复实验、误差区间；
4. **部署结果**：MCU latency、Flash、LUT trade-off；
5. **失败样本与边界讨论**。

现在的主稿已经有第 1 和第 4 部分，但第 2、3、5 部分都明显偏弱。

### 7.4 讨论段要从“模型更好”升级成“工程采用是否成立”

AI+领域论文最终不是要证明“模型优于 baseline”就结束，而是要回答：

- 这个方案是否真的适合进入工程链路；
- 需要多少标定成本；
- 是否受设备个体差异影响；
- 部署和维护代价是否可接受。

当前主稿的 discussion 还不够显式，后续需要把这些工程问题拉到台前。

## 八、最小可执行的实验补强方案

如果不考虑大规模新增器件实验，只从当前已有条件出发，最小可执行方案如下。

### 8.1 必做：四组机制消融

必须补：

- 频响注入消融；
- KAN vs 普通 MLP 消融；
- 物理约束消融；
- AFMAE 损失消融。

这是把“方法看起来合理”变成“方法被因果验证”的最低门槛。

### 8.2 必做：一组更严格的数据划分

至少增加一种不是随机子序列切分的 protocol，例如：

- 留出若干完整频率点；
- 留出若干完整振幅点；
- 或留出整条测量序列。

否则当前论文的泛化性证据会一直偏弱。

### 8.3 建议做：多随机种子统计

哪怕只补 3 到 5 个随机种子，也能显著提升工程论文可信度。

### 8.4 建议做：增加一个更自然的中间 baseline

优先级上，应该补的不是 Transformer，而是：

- 1D-CNN / TCN；
- 或普通 MLP / spline-MLP。

因为这类 baseline 更能回答“为什么非得要 KAN / prior injection”。

### 8.5 可选做：把工程已有方案纳入统一讨论表

即使无法完全重跑 force-feedback，也应在统一指标框架下给出：

- 适用振幅范围；
- 对线性度 / 灵敏度漂移的改善；
- 额外硬件代价；
- 稳定性风险。

这样主稿才更像在和工程现实对话，而不是只和学术 baseline 对话。

## 九、最终判断

R3.1 需要得出的最终判断不是“这个题目能不能投 MN”，而是：

**当前主稿要想成为一篇更成熟的 AI+领域论文，最优先补的不是更多领域背景，而是更完整的实验结构。**

具体来说：

- 领域匹配问题已经大幅缓解，因为 [VIP]Electrochemical_vibration_sensor 说明 MET 本身属于该领域可接受对象；
- 当前最大的短板是论文仍在以模型术语组织叙事，而没有以工程问题组织证据；
- 当前最大的实验短板不是 baseline 数量，而是缺少机制性消融；
- 当前最大的可信度短板不是结果不够好，而是数据划分、统计复现和边界分析偏弱。

因此，如果后续继续推进这条转投路线，正确顺序应当是：

1. 先重写论文叙事，把“问题”放到“模型”前面；
2. 再补四组机制消融；
3. 再补一组更严格的泛化验证；
4. 最后保留 MCU 部署结果作为工程闭环，而不是把部署结果当成唯一工程证据。

只有完成这一步，当前工作才会从“效果很好的方法稿”，提升为“结构更成熟的 AI+领域论文”。
