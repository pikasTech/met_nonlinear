# R7.1 任务报告：论文消融实验方法中间稿

## 目标

基于当前 preset 文件和仓库中的真实代码路径，撰写一份面向论文的长期消融实验方法参考文档。文档只解释方法设计，不记录任何实验结果数值。

## 产出

- `docs/reference/paper_ablation_method.md`
- `AGENTS.md`

## 本次沉淀的内容

1. 将当前损失消融与结构消融 preset 明确为正式实验分组清单。
2. 对每个被选中的 project 回溯其 `config.json`，确保文档解释的是代码级变体语义，而不只是 project 名称。
3. 将 `ProjectManager.prepare_dataset_and_model()`、`ModelEngine.build_model()` 到 `metrics.json` 的共享执行链沉淀为长期稳定说明。
4. 记录 `randfrirnn_r2` 这类混合变体的当前写作边界，避免论文把实现上并不干净的实验夸写成更强的消融结论。

## 为什么符合当前 MN 目标写法

文档按“分组定义 -> 变体定义 -> 共享指标路径”的顺序组织，适合直接转写为期刊论文中的方法章节，同时把一次性数值留在结果章节。
