## 五、A/B/C/D 总结论

### 5.1 当前矩阵最稳妥的最终结论

从当前所有可用结果看：

1. **A（`true`, `false`）仍然明显最好。**
   - 这说明当前 Wiener-KAN 的正式前端设计不是偶然碰巧，而是当前最合理的选择。

2. **B（`false`, `false`）可训练，但稳定落后于 A。**
   - 这条线证明：系统初始化先验本身是有价值的。

3. **C（`true`, `true`）没有体现出“微调系统先验”的收益，反而带来明显稳定性风险。**
   - 当前实现下不建议把 IIR 前端改成 trainable 正式方案。

4. **D（`false`, `true`）说明“只给一个可学习前端”并不能替代正确的物理先验。**
   - 它比 `lr14e5` 更能训练，但仍远不如 A。

### 5.3 是否还要继续调

在“当前代码路径 + 当前 GPU 会话”下，我认为 R6.2 可以先收口，理由是：

- B、D 已经各自拿到了可用的代表性完整结果；
- C 组已经有三条不同 LR 的中断 checkpoint 指标，并且本轮再次复现了中途 `CUDA_ERROR_ILLEGAL_ADDRESS`；
- 现阶段即使继续把 LR 再细扫，最核心结论也大概率不会变：
  - A 最优；
  - B 次之但明显退化；
  - D 远差于 A；
  - C 不值得作为正式版本。

如果后续还要继续深挖，最合理的顺序不是继续盲调 A/B/C/D 的 LR，而是：

1. 先查清 `C` 路径中 `trainable IIR` 的训练稳定性问题；
2. GPU 状态恢复后，再决定是否补 `B/lr3e4` 或 `B/lr4e4` 这一类近邻点做最后确认；
3. 论文正式表格里优先先把当前已经足够清楚的结论写进去。
