## 论文草稿的核心结论（本轮整合得到）

### 1. Wiener-KAN 的当前 strongest claim 已经更清楚

本轮整合后，正文不再把 Wiener-KAN 写成“全面碾压所有模型”的故事，而是更准确地收口为：

- 在当前 canonical project 集合中，Wiener-KAN 拿到最低的固有频率漂移和最低的灵敏度漂移；
- TCN 仍然有更低的平均非线性误差；
- RNN 仍然有更低的计算量；
- 部署端 Wiener-KAN 最快、RAM 最低，但 Flash 成本极高。

这比旧稿那种纯“方法最强”叙事更适合后续转向 MN 的工程型表达。

### 2. 组合损失是当前证据最扎实的一条机制线

通过统一整理后，MAE+AFMAE 相比纯 MAE / 纯 AFMAE 的优势已经非常清楚，适合作为后续稿件中保留的有效设计主张。

### 3. 结构消融真正说明的是“协同有效”，不是“单点绝对必要”

整理后的图表显示：

- `nosym` 和 `nopositive` 会明显退化，说明约束有效；
- `random trainable IIR` 会明显变差，说明“任意可训练前端”不能直接替代当前先验写法；
- 但 `CNNKAN` 仍与 Wiener-KAN 接近，因此不应把当前结果写成“Wiener 前端是唯一可行方案”。

这使正文更符合当前仓库事实，也更符合 R5/R5.1 要求的“克制 claim budget”。

### 4. 部署章节的 trade-off 被显式写出来了

旧稿阶段更容易把“快”写成单向优点；这次整合明确写出了：

- Wiener-KAN 在 Keil 上时延最低；
- 但 Flash 占用约 1022 KB，几乎耗尽 1 MB 级器件容量；
- TCN / 1DCNN 的导出数值一致性更好；
- 因此当前最准确的结论是“Wiener-KAN 快且 RAM 小，但 Flash 压力大且导出误差仍需继续优化”。

这会让后续 MN 方向的部署叙事更稳，不会过度宣传。
