# R22.2 任务报告

## 目标

修复论文拼图中 `(a)(b)(c)` 标签在 PDF 中显示大小不一致的问题，并确保标签位于子图内容外部，避免遮挡图像内容。

## 修改内容

1. 统一标签物理字号计算
   - 在 `src/visualization/subfigure_montage.py` 中为 `compose_subfigures()` 增加 `label_font_size_pt`、`label_reference_width_pt`、`latex_width_fraction` 参数。
   - 标签像素字号按 `label_font_size_pt * canvas_width_px / (label_reference_width_pt * latex_width_fraction)` 计算，使拼图缩放到 LaTeX 目标宽度后保持固定物理字号。
   - 保留旧的 `label_font_size` 作为 fallback，兼容未传入物理字号参数的调用。

2. 将标签放到子图内容外部
   - 增加 `outside-top-left`、`outside-top-right`、`outside-top-center` 标签位置。
   - 对每一行计算独立的 `row_label_heights` 标签带，并将子图内容下移到 `content_y`，从布局层面避免标签覆盖内容。
   - raw metadata 记录 `label_font_size_px`、`label_font_size_pt`、`latex_width_fraction`、`label_position`、`label_outside_top`、`row_label_heights`，便于后续追踪。

3. 论文绘图 pipeline 接入统一规则
   - 在 `docs/paper/src/paper_pipeline.py` 中设置默认目标字号 `SUBFIGURE_LABEL_TARGET_PT = 9.0`，参考宽度 `SN_A4_TEXT_WIDTH_PT = 372.0`。
   - `make_bitmap_montage()` 默认使用 `outside-top-left`、无标签框、物理字号 9 pt，并向 montage helper 传入 LaTeX 宽度比例。
   - 对使用非 100% 宽度插入的图回填比例，例如 0.98、0.86、0.85、0.78 等，保证最终 PDF 中标签大小一致。
   - `docs/paper/src/translate_legacy_figures.py` 中的 `local_transfer_slices.png` 也切换到统一 montage helper。

4. LaTeX 版面收尾
   - `docs/paper/latex/main.tex` 中将 loss ablation 段落前的 `\FloatBarrier` 移到 Fig. 18 后、Discussion 前，避免标签外置后浮动体集中输出造成的欠填页，同时仍保证实验章节浮动体在 Discussion 前完成。
