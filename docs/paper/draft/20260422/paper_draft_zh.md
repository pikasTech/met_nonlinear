# 基于 Wiener-KAN 的电化学地震检波器大震级非线性频响校准：方法、横向对比与嵌入式评估（中文草稿）

> 写作说明：本文为 2026-04-22 的内部中文整合稿，结构参照 `[VPI]AI_Based_Sensor.md` 的“Abstract → Introduction → System design and methodology → Results and discussion → Conclusion”组织方式。文中正式数字优先取自当前项目的统一汇总产物，而不是训练日志截图；因此，横评应理解为共享数据协议下的 canonical project comparison，而不是严格等超参的单因子结构对照。

## Abstract

电化学地震检波器（MET）在大震级工况下会出现显著的非线性频响漂移，具体表现为固有频率漂移、100 Hz 灵敏度漂移以及输入输出非线性误差同时放大。围绕这一问题，本文基于当前仓库中已经固化的实验资产，整合出一篇带有方法、数据、图表与结果分析的中文论文草稿。本文采用 Wiener-KAN 作为主方法：用系统初始化的 Wiener/IIR 线性前端描述震级相关局部频响，再用 KAN 静态非线性主体完成前馈补偿，并以 MAE+AFMAE 组合损失联合约束时域误差与幅频响应误差。实验评估频点覆盖 10-200 Hz，其中正式 `Linearity (%)` 指标按 `<=128 Hz` 的 in-band 频点统计；其余协议包括 25 个震级点（0.24-6.0 m/s^2）、4 s 序列窗口和 2000 Hz 采样率。对比 Wiener-KAN、TCN、1DCNN、LSTM、LSTMTransformer、RNN、GRU 共 7 个代表性模型后可以看到，在当前 canonical project 集合中，Wiener-KAN 取得最低的固有频率漂移（2.34 Hz）、最低的灵敏度漂移（9.09%）以及最低的带内平均非线性误差（0.511%）。相对未补偿原始响应，这三项指标分别改善 93.23%、89.25% 和 74.50%；与当前较强的卷积基线 TCN 相比，Wiener-KAN 的固有频率漂移进一步降低 11.72%，灵敏度漂移降低 43.46%，并把加权计算量压到 1710。损失消融表明，组合损失相对纯 MAE 将固有频率漂移再降低 82.03%，相对纯 AFMAE 再降低 41.64%，同时把带内平均非线性误差分别再降低 35.55% 和 52.14%。结构消融表明，去掉对称性约束、随机化且可训练的 IIR 前端，或将 KAN 替换为 MLP，都会带来明显退化；完全移除正定约束时，固有频率漂移甚至恶化到 3.26×10^5 Hz。嵌入式评估显示，Wiener-KAN 在 STM32F405 上实现了 0.537 ms/point 的最低板端时延和 21.2 KB 的最低 RAM 占用，但当前导出实现占用约 1022 KB Flash，几乎耗尽 1 MB 级器件的程序空间，说明后续工作需要继续压缩 LUT/参数存储。总体上，本文支持将 Wiener-KAN 定位为一种在当前统一物理指标口径下最优、同时兼顾结构化建模和板端部署潜力的 MET 前馈校准方案。

**Keywords**: electrochemical seismometer; Wiener-KAN; nonlinear frequency-response drift; AFMAE; ablation study; embedded deployment

## 1 引言

电化学地震检波器在弱振动测量中具有高灵敏度和低噪声优势，但随着输入震级增大，其频率响应会表现出明显的震级依赖性：固有频率向高频漂移，100 Hz 处灵敏度随震级变化而偏离，输入输出关系也逐步失去线性。传统的结构优化和力反馈方案能够在局部工况下抑制这类问题，但它们往往面向较窄的工作区间，或者需要附加反馈硬件与系统调试成本。对于目标是扩大动态范围、保持系统结构简洁并支持后续嵌入式部署的场景，前馈型非线性校准是一条更具工程吸引力的路径。

在当前项目语境下，本文不再把重点放在“提出一个全新的模型命名”上，而是把问题重新表述为：**如何在宽震级、宽频率测试协议下，构建一个既能抑制 MET 非线性频响漂移，又具备结构可解释性与部署潜力的前馈校准器。** 基于这一主旨，Wiener-KAN 采用“Wiener 线性前端 + KAN 静态非线性主体”的结构化写法：前端负责注入局部频率响应先验，后端负责在共享输入协议下拟合复杂非线性映射；训练阶段再通过 MAE 与 AFMAE 的组合损失，把时域拟合与幅频一致性同时纳入优化目标。

与只看单一时域误差的写法不同，本文把实验目标统一放到三个层面：

1. 物理性能：是否能同时压低固有频率漂移、灵敏度漂移和带内平均非线性误差；
2. 结构解释：哪些结构约束和前端先验是真正必要的，哪些只是可替换实现；
3. 工程可用性：在 QEMU 与 Keil 真实部署链路中，模型是否仍保持数值一致性，并具备可接受的速度与资源代价。

基于这些目标，本文的贡献可概括为三点：

1. 基于当前稳定实验协议，整理出一个统一的中文论文草稿框架，把数据集、损失、横评、消融和部署结果收口到同一套方法-结果链条中；
2. 通过 7 个代表性模型的横向对比、3 组损失消融、6 组结构消融和 6 个板端部署对象，给出 Wiener-KAN 在当前仓库中的真实强项、弱项与边界；
3. 明确指出 Wiener-KAN 当前最值得保留的主张是“在统一物理主指标上最优、在计算量上保持较低、在板端时延上最快，但 Flash 代价显著偏高”，从而为后续 MN 方向成稿提供更克制、更工程化的叙事基础。

## 2 系统设计与方法

### 2.1 实验数据与任务定义

当前实验协议围绕单器件、单环境、宽震级宽频率扫频测试构建。原始实验条件和后续训练窗口的稳定设置汇总如下。

**表 1  当前实验协议与数据窗口**

| Item | Value |
| --- | --- |
| Sensor sample | MTSS-1001 |
| Environment | 25 C |
| Frequency grid (saved evaluation) | 10-200 Hz, 14 points |
| Magnitude sweep | 0.24-6.0 m/s^2, 25 levels |
| Sequence duration | 4.0 s |
| Sampling rate | 2000 Hz |
| Window count | 8000 sequences |
| Reference sensitivity point | 100 Hz |
| Linearity band (official metric) | <=128 Hz, 12 points |

设输入序列为 $x_{1:T}$，目标线性响应为 $y_{1:T}$，补偿器输出为 $\hat{y}_{1:T}$，则统一监督学习形式写为：

$$
\hat{y}_{1:T} = f_{\theta}(x_{1:T}).
$$

其中 $f_{\theta}(\cdot)$ 对应不同 project 中的具体模型结构。需要强调的是，当前横评中的各 project 共用相同的数据协议，但 active loss 与学习率并不完全相同，因此本稿中的主横评是**共享协议下的代表性项目对比**，而不是严格的 architecture-only 单因子实验。

### 2.2 Wiener-KAN 结构

Wiener-KAN 的核心思想是把 MET 的非线性补偿写成“可解释线性前端 + 灵活静态非线性主体”的级联结构：

$$
\hat{y}_{1:T} = g_{\phi}\bigl(h_{\psi}(x_{1:T})\bigr),
$$

其中 $h_{\psi}(\cdot)$ 表示按局部频率响应初始化的 Wiener/IIR 线性前端，$g_{\phi}(\cdot)$ 表示 KAN 静态非线性映射。与纯黑盒时序模型不同，这种写法将“震级相关频响漂移”优先交给线性前端编码，再把剩余非线性误差交给 KAN 处理，从而使模型在物理解释上更接近 MET 的局部线性化校准过程。

在当前仓库里，结构对照主要围绕三个方向展开：

1. 前端替换：以 CNNKAN 或随机可训练 IIR 变体观察“保留 KAN 主体但改变前端”的影响；
2. 主体替换：以 FRIMLP 观察“保留同一 FRI/Wiener 前端，仅把 KAN 换成 MLP”的影响；
3. 约束移除：以 `nosym` 与 `nopositive` 观察对称性约束和正定约束的必要性。

这组设计使本稿能够回答：Wiener-KAN 当前的最优结果究竟来自哪里，是来自 KAN 主体、线性前端先验，还是来自二者与约束的协同作用。

### 2.3 损失函数与正式指标

#### 2.3.1 MAE、AFMAE 与组合损失

逐点 MAE 写为：

$$
\mathcal{L}_{\mathrm{MAE}} = \frac{1}{N}\sum_{i=1}^{N}\left|y_i-\hat{y}_i\right|.
$$

AFMAE 则首先对序列做分组能量聚合，再在对数域比较幅频相关误差：

$$
P_g(y)=\sum_{t=1}^{G}|y_{g,t}|,\qquad
P_g(\hat{y})=\sum_{t=1}^{G}|\hat{y}_{g,t}|,
$$

$$
\mathcal{L}_{\mathrm{AFMAE}}=
\frac{1}{M}\sum_{g=1}^{M}
\left|
\log(P_g(y)+\varepsilon)-\log(P_g(\hat{y})+\varepsilon)
\right|.
$$

当前标准 Wiener-KAN 基线采用的不是简单的“MAE + AFMAE 直接相加”，而是当前实现口径下的平衡组合损失：

$$
\mathcal{L}_{\mathrm{comb}} = k\,\mathcal{L}_{\mathrm{AFMAE}} + (1-k)\,\mathcal{L}_{\mathrm{MAE,bal}},\qquad k=0.2.
$$

其中 $\mathcal{L}_{\mathrm{MAE,bal}}$ 是按分组能量重新缩放后的 MAE 项。该写法的目的不是单纯追求更低的逐点误差，而是在训练阶段同时约束时域偏差和幅频响应漂移。

#### 2.3.2 正式物理指标

本文正式表格采用三类物理指标：

1. **Freq Drift (Hz)**：跨震级 compensated 固有频率的中位数中心漂移；
2. **Sens Drift (%)**：100 Hz 处插值灵敏度的中位数中心漂移；
3. **Linearity (%)**：`<=128 Hz` in-band 的平均非线性误差百分比，其实现形式为

$$
\mathrm{Linearity} = \frac{100}{K}\sum_{k=1}^{K}(1-R_k^2).
$$

其中 $K$ 只覆盖 `frequency_hz <= 128` 的频点，因此该指标越小越好，它不是“越大越好”的正向评分。

#### 2.3.3 复杂度与部署指标

当前仓库中的 `Compute Cost` 对应单样本、单时间步的加权语义操作估算：

$$
C = N_{\mathrm{add}} + N_{\mathrm{mul}} + 6N_{\mathrm{map}}.
$$

在部署子章节中，进一步引入：

- `QEMU-MAE`：C 推理相对 TensorFlow 参考路径的仿真一致性误差；
- `KEIL-MAE`：真机路径相对 TensorFlow 的一致性误差；
- `KEIL-SPEED (ms/point)`：单位点板端 wall-clock 时延；
- Flash / RAM：由 Keil `Program Size` 行换算得到的资源占用。

### 2.4 横评、消融与部署协议边界

本文实验分为三层：

1. **主横评**：覆盖 Wiener-KAN、TCN、1DCNN、LSTM、LSTMTransformer、RNN 和 GRU；
2. **损失消融**：固定 Wiener-KAN 主体，对比 MAE+AFMAE、MAE 和 AFMAE；
3. **结构消融**：围绕 CNN 前端替换、去对称、随机可训练 IIR、KAN→MLP 替换以及去正定 stress test 展开。

需要主动说明三条边界：

- 主横评中的 active loss 和学习率并不完全一致，因此这一节应理解为 canonical project comparison；
- 随机可训练 IIR 变体同时改变了初始化、可训练性和学习率，所以它更适合被解释为“前端替换代理”，而不是“仅去掉系统先验”的严格单因子实验；
- 当前部署子章节只纳入具有合法 `board_inference` 路径的模型。RNN 暂未进入部署表；CNN 家族在部署子章节中使用 board-ready 的 `1DCNN(board-ready)` 项，而不是主横评中的官方 1DCNN 行。

## 3 结果与讨论

### 3.1 物理漂移轨迹与主横向对比

图 1 给出了未补偿原始响应与若干代表性模型在震级维度上的物理轨迹。可以看到，原始 MET 的固有频率和 100 Hz 灵敏度都随震级持续抬升，且高震级区间的漂移更严重。相比之下，Wiener-KAN、TCN、1DCNN 和 LSTM 都能显著压平这两条曲线，但 Wiener-KAN 在整个震级范围内保持了更低、更平滑的固有频率漂移轨迹，并把 100 Hz 灵敏度约束在更窄的波动带内。

![Fig. 1](./fig_01_drift_trajectories.png)

进一步地，图 2(a) 将三类物理指标都转换为相对未补偿原始响应的抑制率。Wiener-KAN 对固有频率漂移、灵敏度漂移和带内平均非线性误差分别实现了 93.23%、89.25% 和 74.50% 的改善，在当前 7 个代表性项目的三项正式物理指标上都取得了最优结果。需要同时说明的是，TCN 与 LSTM 在线性度上已经非常接近 Wiener-KAN：TCN 的带内平均非线性误差为 0.517%，LSTM 为 0.512%，都与 Wiener-KAN 的 0.511% 处于同一数量级。图 2(b) 的归一化雷达图则说明：如果把物理指标、计算代价和参数规模一起考虑，Wiener-KAN 的整体轮廓最均衡，而 RNN 虽然参数最少、计算量最低，却在三个物理指标上明显落后。

![Fig. 2](./fig_02_horizontal_summary.png)

**表 2  主横向对比结果（共享协议下的 canonical projects）**

| Model | Freq Drift (Hz) | Sens Drift (%) | Linearity (in-band, %) | Compute Cost | Params |
| --- | --- | --- | --- | --- | --- |
| Wiener-KAN | 2.34 | 9.09 | 0.511 | 1710 | 2569 |
| TCN | 2.65 | 16.08 | 0.517 | 3904 | 1697 |
| 1DCNN | 3.72 | 16.24 | 0.920 | 1880 | 885 |
| LSTM | 4.50 | 13.80 | 0.512 | 3360 | 1441 |
| LSTMTransformer | 4.72 | 18.38 | 0.567 | 4488 | 1261 |
| RNN | 6.68 | 23.75 | 0.726 | 1280 | 577 |
| GRU | 7.40 | 27.15 | 0.726 | 2688 | 1201 |

从表 2 可以归纳出三个更具体的观察：

1. **Wiener-KAN 是当前三项正式物理指标最优的模型。** 它的固有频率漂移仅 2.34 Hz，比当前最强卷积基线 TCN 再低 11.72%，比 LSTM 再低 48.13%，同时把带内平均非线性误差压到 0.511%。
2. **TCN、LSTM 和 1DCNN 证明其他家族同样能有效压低漂移，但代价结构不同。** TCN 与 LSTM 在带内线性度上已经非常接近 Wiener-KAN，但分别要付出更高的计算量（3904、3360）；1DCNN 把参数压到 885，但两个漂移指标仍明显高于 Wiener-KAN。
3. **低计算量不自动意味着更好的物理校准。** RNN 的计算量最低（1280），但三个物理指标都没有进入第一梯队，说明在当前任务上，结构先验与损失设计比“极简模型”更关键。

### 3.2 损失函数消融：组合损失比纯 MAE 或纯 AFMAE 更稳

图 3 与表 3 展示了 Wiener-KAN 主干上的损失函数消融。这里的 pure AFMAE 行使用的是当前仓库中表现更稳定的 canonical 实例，而不是更早期的探索性 run，因此更接近本文拟采用的正式稿口径。

![Fig. 3](./fig_03_loss_ablation.png)

**表 3  损失函数消融结果**

| Variant | Active loss | Freq Drift (Hz) | Sens Drift (%) | Linearity (in-band, %) | Val MAE | Val AFMAE |
| --- | --- | --- | --- | --- | --- | --- |
| MAE+AFMAE | MAE+AFMAE | 2.34 | 9.09 | 0.511 | 0.0225 | 0.0379 |
| MAE | MAE | 13.00 | 20.28 | 0.793 | 0.0105 | 0.0903 |
| AFMAE | AFMAE | 4.00 | 21.06 | 1.068 | 0.0687 | 0.0350 |

结果非常清楚：

- 相对纯 MAE，组合损失把固有频率漂移再降低 82.03%，把灵敏度漂移再降低 55.17%，同时把带内平均非线性误差再降低 35.55%；
- 相对纯 AFMAE，组合损失把固有频率漂移再降低 41.64%，把灵敏度漂移再降低 56.82%，把带内平均非线性误差再降低 52.14%。

这说明两件事。第一，单纯优化逐点误差并不能自动带来更好的频响稳定性；纯 MAE 虽然有最低的验证 MAE，却在三类物理指标上都明显更差。第二，单纯优化 AFMAE 也不够，因为它会牺牲时域拟合并带来更差的整体平衡。对于当前 MET 任务，最有效的写法不是在 MAE 和 AFMAE 之间二选一，而是在实现口径下把两者组合起来，让频域稳定性和时域误差同时受控。

### 3.3 结构消融：KAN-IIR 协同有效，但前端价值仍需克制表述

图 4 和表 4 汇总了结构消融结果。这里有两个值得同时强调的事实：一方面，Wiener-KAN 的多个关键组件一旦被替换或移除，物理指标会明显恶化；另一方面，CNNKAN 与 Wiener-KAN 的差距并不大，这意味着当前证据更支持“Wiener-KAN 是最佳写法之一”，而不是“只有 Wiener/IIR 前端才可能工作”。

![Fig. 4](./fig_04_structure_ablation.png)

**表 4  结构消融结果**

| Variant | Freq Drift (Hz) | Sens Drift (%) | Linearity (in-band, %) | Compute Cost |
| --- | --- | --- | --- | --- |
| Wiener-KAN | 2.34 | 9.09 | 0.511 | 1710 |
| CNNKAN | 2.43 | 9.22 | 0.883 | 1718 |
| No symmetry | 10.91 | 10.61 | 2.109 | 1710 |
| Random trainable IIR | 5.34 | 21.38 | 3.216 | 1710 |
| FRIMLP | 5.82 | 27.48 | 0.782 | 5152 |
| No positive (stress) | 3.26e+05 | 50.87 | 38.586 | 1710 |

可以得到以下判断：

1. **对称性与正定约束不是装饰。** 去掉对称性后，固有频率漂移增大到 10.91 Hz，是基线的 4.67 倍；完全移除正定约束时，固有频率漂移直接爆炸到 3.26×10^5 Hz，约为基线的 1.39×10^5 倍。这说明约束在当前任务上确实编码了有效先验。
2. **随机化且可训练的 IIR 前端显著变差。** 该变体的灵敏度漂移达到 21.38%，比 Wiener-KAN 高 57.48%。需要注意，它并不是严格单因子实验，但至少说明“任意初始化 + 可训练”并不能替代当前的系统初始化前端。
3. **KAN 主体替换成 MLP 会同时损伤精度与复杂度。** FRIMLP 的固有频率漂移升到 5.82 Hz，灵敏度漂移升到 27.48%，带内平均非线性误差升到 0.78%，而计算量增至 5152，是 Wiener-KAN 的 3.01 倍。这说明当前最优结果并不是简单来自“任意非线性层”。
4. **CNN 前端替换保持竞争力。** CNNKAN 与 Wiener-KAN 的差距相对较小，因此当前证据更适合被解释为：Wiener/IIR 前端使模型达到最优工作点，而不是“没有这条前端就完全无效”。这一点对于后续论文措辞非常重要，应避免把前端贡献过度夸大成已经被完全证明的强结论。

### 3.4 部署一致性与资源代价：Wiener-KAN 最快，但 Flash 压力极高

当前部署子章节仅保留具备合法板端路径的模型，因此不包含 RNN。CNN 家族则使用 board-ready 的 `1DCNN(board-ready)` 项来代表卷积路径。图 5 与表 5 展示了部署子集在 QEMU / Keil 一致性、板端速度和资源占用上的权衡。

![Fig. 5](./fig_05_edge_tradeoff.png)

**表 5  MCU 部署子集结果**

| Model | QEMU-MAE | KEIL-MAE | KEIL speed (ms/point) | Flash (KB) | RAM (KB) |
| --- | --- | --- | --- | --- | --- |
| Wiener-KAN | 7.114e-03 | 7.114e-03 | 0.537 | 1022.0 | 21.2 |
| GRU | 2.451e-03 | 2.451e-03 | 0.602 | 12.7 | 58.7 |
| LSTMTransformer | 5.049e-03 | 5.049e-03 | 0.684 | 19.9 | 83.7 |
| LSTM | 5.070e-02 | 5.070e-02 | 0.701 | 13.8 | 58.7 |
| 1DCNN(board-ready) | 1.751e-06 | 1.751e-06 | 0.697 | 17.4 | 125.9 |
| TCN | 1.197e-06 | 1.197e-06 | 1.069 | 16.4 | 106.7 |

部署结果显示出非常鲜明的工程 trade-off：

- **Wiener-KAN 的板端时延最低。** 它在 STM32F405 上达到 0.537 ms/point，是当前部署子集里最快的模型；相较 TCN 提速约 1.99×，相较 LSTM 和 LSTMTransformer 也有明显优势。
- **Wiener-KAN 的 RAM 占用最低，但 Flash 占用极端偏大。** 当前导出实现约占 1022 KB Flash，只比 1 MB 级容量上限低约 2028 字节；这说明它虽然快，却几乎把程序空间用满，后续必须继续压缩 LUT 或权重存储。
- **TCN 和 1DCNN(board-ready) 的数值一致性最好。** 二者的 QEMU/Keil MAE 都达到 1e-6 量级，明显优于 Wiener-KAN。对本文而言，这意味着 Wiener-KAN 的当前 C 导出路径虽然可用，但仍存在查表/量化引入的近似误差，不应把“最快”直接偷换成“部署链最完美”。
- **与 LSTM 相比，Wiener-KAN 取得了更好的“速度 + 一致性 + RAM”平衡。** 这使它更适合作为当前项目继续推进的主部署对象；但若目标器件 Flash 更小，则当前导出实现会立即成为瓶颈。

综合来看，Wiener-KAN 当前最合理的部署结论不是“资源全面最优”，而是：**在当前导出路径中，它换取了最低时延和最低 RAM，但代价是极高的 Flash 占用，以及相比卷积基线更高的导出误差。**

### 3.5 适用范围与局限性

为了避免把当前结果写得超过证据边界，本文最后明确四项局限：

1. **数据边界仍然偏窄。** 当前实验基于单器件（MTSS-1001）和 25 C 单环境，不具备跨器件、跨温度泛化结论。
2. **横评不是严格等超参实验。** Wiener-KAN 使用 MAE+AFMAE，其余多数基线使用 MAE；学习率也并不完全一致，因此表 2 只能支撑“当前 canonical projects 的真实比较”，不能被夸大成完全正交的结构对照。
3. **前端先验故事还没有被完全钉死。** CNNKAN 的竞争力说明当前证据更适合支持“Wiener-KAN 是当前最好方案”而非“Wiener 前端是唯一可行方案”；如果后续要更强地证明这一点，仍需补更干净的前端 2×2 消融矩阵。
4. **部署链条尚未收口。** Wiener-KAN 的 Flash 代价过高，当前 MCU 版本更像“可以运行的研究原型”，而不是已经充分工程化的最终实现。

## 4 结论

本文整合了当前仓库中与 Wiener-KAN 相关的主横评、损失消融、结构消融和嵌入式部署结果，形成了一篇适合继续向 MN 方向收敛的中文论文草稿。基于当前正式数据，可以得出以下结论：

1. Wiener-KAN 在当前 7 个代表性 canonical projects 中取得了最低的固有频率漂移、最低的灵敏度漂移和最低的带内平均非线性误差，是当前统一物理主指标下最优的频响稳定化方案；
2. MAE+AFMAE 组合损失比纯 MAE 或纯 AFMAE 都更稳定，说明当前任务需要同时约束时域误差和幅频响应误差；
3. 对称性约束、正定约束以及 KAN 主体都对最终性能有实质贡献，但关于 Wiener/IIR 前端是否“绝对必要”，当前证据还应保持克制表述；
4. Wiener-KAN 在板端取得了最低时延和最低 RAM 占用，但其 Flash 占用几乎达到器件上限，后续工作应优先压缩 LUT / 权重存储并优化导出一致性。

因此，本文更建议把 Wiener-KAN 写成一种**以频率稳定化为主目标、兼顾结构化建模和板端部署潜力的 MET 前馈校准方案**，而不是再回到“模型创新点越多越好”的旧叙事。沿着这一方向，后续成稿应继续补齐更严格的协议验证、更加干净的前端先验消融，以及更加节制的部署优化结论。
