# 损失函数设计原则

## 功能概述

本项目里的损失函数设计，不是单纯追求训练期数值更低，而是为了让模型更稳定地逼近“幅度依赖频率响应”的目标现象。

长期可复用的判断标准是：损失函数必须服务于最终频响指标，而不是反过来让频响指标去迁就训练 loss。

## 设计目标

从 NOTEBOOK 的长期经验看，适合本项目的损失函数至少要满足三件事：

- 能在不同频率之间建立可比较的误差口径
- 能约束系统保持准线性特征，而不是只在局部时域片段拟合得好看
- 能对谐振峰值变化、中心频率漂移和幅度相关失真保持敏感

如果一个损失函数只能降低逐点波形误差，却无法改善上述现象，它对当前问题就不是合格的主判断信号。

## 为什么不能只看时域 loss

项目早期多次出现过同一种情况：

- 时域波形看起来还可以
- 训练 loss / MAE 也下降
- 但频响三指标依然明显失真

因此，时域损失只能说明“模型在当前采样窗口上更接近目标波形”，不能直接说明“频率响应补偿已经更好”。

这也是统一验收必须落到 [evaluation.md](evaluation.md) 和 [metrics.md](metrics.md) 的原因。

## 频域损失的角色

对当前问题而言，频域损失或幅频相关损失的主要作用是：

- 提高模型对频率相关误差的敏感性
- 降低模型只追逐局部时域细节的倾向
- 让训练更关注谐振区和幅度依赖行为

但频域项的作用也有边界：

- 它不能替代正确的数据预处理和频响测量口径
- 它不能弥补模型结构与问题定义不匹配
- 它也不能替代最终的统一指标验收

## MAE / MSE 与 AFMAE 的分工

### 时域损失

MAE / MSE 一类时域损失的价值在于：

- 训练稳定
- 易于优化
- 适合做基础波形拟合基线

但它们天然更关注逐点误差，不会自动把“频率漂移”和“灵敏度漂移”当作一等公民。

### AFMAE

AFMAE 在本项目中的价值，是把幅频层面的误差显式拉进训练关注范围。

它不是为了彻底替代时域损失，而是为了补足时域损失对频响行为不敏感的短板。

当前项目里关于具体对比任务、项目列表和输出格式，统一以 [mae_vs_afmae.md](mae_vs_afmae.md) 为准。

### 组合损失优先于极端单一目标

对当前问题，长期经验更支持“时域项 + 频域项”的组合，而不是简单地走向两个极端：

- 只看时域，容易频响失真
- 只看频域，可能时域行为或训练稳定性变差

具体权重可以实验，但设计原则不变：损失函数应帮助模型兼顾时域可训练性与频域目标一致性。

## 做损失函数消融时的约束

损失函数实验最容易被破坏可解释性的地方，不在实现，而在实验设计。

长期约束如下：

- 做 loss 消融时，优先固定数据和模型结构，只改 loss
- 不要一边改 loss，一边顺手改网络、batch size、数据范围或评估脚本
- 如果想看 loss 的效果，先在仿真系统上隔离验证，再上真实数据
- 如果用户要求“只调 loss”，就只调 loss，不要顺手把其他变量一起动掉

## 训练信号与验收信号分离

训练时可以观察：

- `loss`
- `val_loss`
- 训练期挂载的 MAE / AFMAE

但最终验收必须与这些训练信号分离。长期规则是：

- 训练期 loss 只用于看收敛和早停趋势
- 评估期统一重算的指标才用于项目对比
- `metrics.json` 才是最终横向比较口径

如果出现“loss 变好但频响指标变差”，应优先相信统一评估链路，而不是继续相信训练曲线。

## 常见误判

- 把更低的 `val_loss` 直接解释成更好的频响补偿
- 把一次实验中更好的训练曲线当成结构或 loss 的最终结论
- 在 loss 消融实验中同时改动模型或数据，导致结论失效
- 忽略缩放、反缩放或频响测量口径，把评估异常错判成 loss 设计失败

## 推荐工作流

1. 先用时域损失拿到可训练基线。
2. 再引入频域相关项或 AFMAE，观察频响指标是否实质改善。
3. 始终通过统一评估链路重算指标，不依赖训练时挂了哪些 metrics。
4. 如果结果混乱，优先回到仿真系统，用可控数据隔离 loss 的真实作用。
