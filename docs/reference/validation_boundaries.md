# 实验验证与外推边界

## 功能概述

本项目的实验验证不能只回答“当前指标是否变好”，还必须回答“这些结果到底能支撑到什么范围”。

这份文档用于收敛长期稳定的验证边界：

- 合成数据、实验台扫频数据和更真实场景数据分别能证明什么
- 当前实验结果可以支持哪些结论，不能直接外推出哪些结论
- 若要把论文或项目结论说得更强，还缺哪些补充验证

它不是论文写作指南，而是项目级的技术边界文档。

## 三类验证层级

### 1. 合成系统验证

合成系统验证更适合回答：

- 模型结构是否具备表达目标现象的能力
- 损失函数是否真的对频响相关误差敏感
- 统一测量链路是否稳定、可复现

这类验证的优势是：

- 变量可控
- 便于隔离结构、数据和 loss 的影响
- 容易定位问题是否来自实现或测量口径

但它不能直接证明：

- 模型已经适用于真实 MET 硬件
- 模型对温度、噪声、老化或装配差异具备鲁棒性
- 论文方法在真实地震激励下仍保持同样效果

### 2. 实验台扫频验证

当前项目最核心的真实验证层，是基于振动台、多频多震级扫频数据的实验台验证。

这类验证更适合回答：

- 模型是否能补偿幅度依赖频率响应
- 在统一频响口径下，指标是否实质改善
- 不同结构、损失函数和消融变体在当前问题上的相对优劣

这也是当前 `metrics.json`、频响对比图和大多数项目结论的直接依据。

但它仍有明确边界：

- 主要覆盖扫频正弦及其幅值扫描条件
- 更接近实验室控制环境，而非真实地震工况
- 对复杂随机输入、多分量激励和强环境扰动的覆盖不足

### 3. 复杂激励与工程场景验证

如果要把结论进一步外推到真实部署或更强的论文表述，仍需要第三层验证，例如：

- 真实地震波形
- 随机或多分量复合激励
- 噪声、温度、老化、硬件偏差等扰动条件
- 独立测试集、重复实验和统计显著性分析

没有这一层时，可以说“在当前扫频实验条件下有效”，但不应直接说“在真实地震场景下已经充分验证”。

## 当前结果可以支持的结论

在当前实验体系下，可以较稳妥地支持以下结论：

- FRIKAN 一类“线性先验 + 非线性主体”的结构在 MET 幅度依赖频响问题上是有价值的
- 不同模型、消融和损失函数在统一扫频实验口径下存在可比较的性能差异
- 频响相关指标相较基线有无改善，能通过统一评估链路判断
- 合成系统与实验台扫频之间，能够建立一条从机制验证到工程验证的中间桥梁

## 当前结果不应直接外推出的结论

- 模型已经对所有真实地震波形都有效
- 模型已经充分具备抗噪声、抗温漂、抗老化和跨设备泛化能力
- 在扫频正弦上的优势会自动迁移到任意随机输入场景
- 少量实验台数据或单次训练结果足以证明长期部署稳健性

## 审稿和对外表述时应保留的边界意识

从 NOTEBOOK 后期的多轮审视看，最容易被高标准审稿或技术复盘质疑的点主要有四类：

- 验证场景是否过于集中在扫频正弦
- 是否缺少真实或更复杂激励的补充测试
- 是否缺少噪声、温度、老化等扰动条件分析
- 是否缺少多次随机初始化、独立测试集和统计方差

因此，对外表述时应优先使用以下口径：

- 在当前多频多震级扫频实验条件下
- 在统一频响评估链路下
- 对实验台控制环境中的 MET 数据有效

而不应直接使用以下口径：

- 对真实地震场景普遍有效
- 已全面验证工程鲁棒性
- 已证明对所有复杂输入都成立

## 转投或重写主旨时的长期规则

当论文经历拒稿、转投或主旨重写时，长期上应把这件事理解为“重新收敛当前可被项目结果支撑的叙事边界”，而不是机械地逐条回应历史审稿意见。

长期规则是：

- 当前主旨以 `docs/IDEA.md` 为准，新的调研与文档扩写都应围绕这份主旨展开。
- `docs/FRIKAN_REJECT.md` 这类拒稿意见应被当作风险清单和边界提醒，而不是下一轮投稿必须逐条照抄的任务列表。
- 若主旨已切换，就应同步切换文献补充、理论支撑和任务编排方向，避免继续围绕旧主张堆材料。
- 对外表述时，优先收缩到当前证据真正覆盖的范围，而不是试图用更多文字把旧主张硬保下来。

这条规则的核心不是“回避审稿意见”，而是防止项目文档、调研任务和论文表述长期停留在已经失效的旧问题定义上。

## 如果要补强验证，优先级是什么

若后续要补强论文或工程说服力，长期建议按下面顺序推进：

1. 先补复杂激励验证：真实地震波形或更贴近地震学的复合波形。
2. 再补扰动条件分析：噪声、温度、老化、器件偏差。
3. 再补统计稳健性：多随机种子、独立测试集、均值与方差。
4. 最后再补部署侧量化：不同 LUT 精度、内存占用、实时吞吐。

这样可以优先解决“结论能不能外推”的核心疑问，而不是先堆更多边缘图表。
