# MET 非线性问题定义与建模原则

## 问题定义

本项目关注的核心对象不是“任意黑盒非线性”，而是电化学地震检波器（MET）这类具有幅度依赖频率响应的准线性系统。

长期稳定的现象描述应优先使用以下表述：

- 幅度依赖的频率响应
- 频率相关的幅度非线性
- 非线性动态补偿

当前仓库中的统一评估口径也围绕这一问题定义展开，重点观察：

- 谐振峰值随振幅变化
- 中心频率随振幅漂移
- 高振幅下的非线性失真与线性度恶化

## 建模判断

### 1. 先按“准线性系统”建模

MET 的主要困难不是完全无结构的强非线性，而是线性动态骨架上叠加了幅度相关的响应偏移。因此，模型设计应优先利用线性系统先验，而不是直接依赖纯黑盒时序网络。

这也是仓库后续从早期 LSTM 基线逐步转向 FRIKAN / FRIRNN / IIR 前端路线的原因。

从早期 LSTM 基线到 FRIKAN 系列的结构演化主线，以及“什么时候该换模型、什么时候不该先动模型”的长期判断，详见 [model_architecture_selection.md](model_architecture_selection.md)。

### 2. 带谐振峰的前端优先使用 IIR 或其等效表示

如果目标系统存在明显谐振峰、中心频率漂移和窄带动态特征，前端应优先使用 IIR、状态空间或其等效 RNN 表示。

低阶 FIR 难以经济地表达高 Q 谐振峰；若强行使用 FIR，通常需要显著更高阶数，既不利于解释，也不利于部署。

### 3. 非线性部分优先看块结构而不是先堆深网

从长期可复用的建模角度，以下结构最值得优先尝试：

- Wiener：适合“线性动态后接静态非线性”的近似场景
- Hammerstein-Wiener：适合非线性影响跨越输入侧与输出侧的场景
- 多个线性子系统加非线性加权的并联结构：适合描述随振幅切换或平滑过渡的动态模态
- IIR 前端加可训练非线性主体：适合把频率响应先验注入神经网络

仓库中的 FRIKAN、FRIMLP、FRIKAND 等模型，都应理解为这一思路下的不同主体替换，而不是完全脱离线性先验的独立模型族。

块结构仿真、非线性基选择和合成训练数据生成的长期规则，详见 [nonlinear_basis_simulation.md](nonlinear_basis_simulation.md)。

### 4. 时域拟合不能替代频域验收

NOTEBOOK 早期实验反复证明：

- 时域波形“看起来能跟上”，不代表频率响应已经正确
- loss 或 MAE 下降，不代表中心频率漂移和灵敏度漂移已经受控

因此，项目验收必须同时看时域与频域，最终以统一指标链路中的频响相关指标为准，而不是只看训练期 loss。

相关实现口径详见 [evaluation.md](evaluation.md) 与 [metrics.md](metrics.md)。

损失函数如何服务这些验收目标，详见 [loss_design.md](loss_design.md)。

## 仿真与频响测量约束

从时域仿真反推频率响应时，关键不是“能不能画出曲线”，而是测量口径是否统一、稳态是否充分建立。

Wiener / Duffing / FRIKAN 前端等所有依赖频响分析的实验，都应复用统一的时域数据与频响测量链路，详见 [timeseries_frequency_analysis.md](timeseries_frequency_analysis.md)。

## IIR 到 RNN 的实现原则

IIR 等效 RNN 的关键不是“能不能写出来”，而是时间步是否严格对齐。

长期保留的结论是：在等效实现中，输出 $y[n]$ 应视为依赖隐藏状态 $h[n+1]$，这比平移输入或平移输出序列更自然，也更适合作为仓库里的实现基线。

如果后续修改 IIR/RNN 前端，应先验证时间步对齐，再比较指标。

## 发展脉络概要

- 早期阶段先确认了时域数据可训练、频率响应可导出，但纯 LSTM 基线在频响上长期存在失真
- 中期阶段明确了问题属于“幅度依赖频率响应”的准线性建模，而非普通时序回归
- 随后逐步收敛到“线性动态先验 + 可训练非线性主体 + 频域验收”的路线，并发展出 FRIKAN 及其消融变体
