# 论文核心故事线与引言重构 (Paper Storyline & Introduction Refinement)

## 1. 背景与问题
在前期版本中，论文的标题和摘要过度聚焦于具体的算法名称和传统的控制论词汇（如“基于 Wiener-KAN 的前馈非线性补偿”），导致核心的工程贡献——在资源受限的微控制器（STM32）上实现实时补偿——被严重弱化。此外，引言部分在叙事上存在逻辑倒置（在尚未陈述现有深度学习面临的算力瓶颈前，直接抛出本文的板端实时框架），且过多强调了正文中未深入验证的“可解释性”。

## 2. 核心决策与定调
为了最大化论文在传感器和边缘计算领域的学术与工程价值，我们在以下几个维度进行了重构定调：

### 2.1 标题升级 (Title Refinement)
* **原标题:** `Feedforward Nonlinearity Compensation for Electrochemical Seismometer via Wiener-KAN`
* **新标题:** `On-Board Real-Time AI Compensation for Nonlinear Drift in Electrochemical Seismometers`
* **决策依据:**
  * **On-Board vs On-Sensor:** 考虑到 MET 传感器系统是“探头 + 采集板”的物理形态，算法实际运行在采集板的 MCU 上，使用 `On-Board` 最为严谨，避免了 `On-Sensor`（感算一体）可能带来的学术争议。
  * **AI Compensation vs Feedforward:** 去掉偏传统信号处理的 `Feedforward`，替换为 `Real-Time AI Compensation`，直接突出版端部署的深度学习能力，作为核心亮点前置。
  * **弱化具体算法名:** 在标题中隐去 `Wiener-KAN`，避免被审稿人误判为纯粹的 AI 方法学创新论文，牢牢树立 **“Sensor 为主，AI 为辅”** 的工程问题导向。

### 2.2 引言重构逻辑 (Introduction Funnel Structure)
严格遵循“漏斗式”论证逻辑，消灭逻辑倒置（剧透）：
1. **背景:** 软件补偿可以替代复杂的力反馈硬件。
2. **现有方法:** 深度学习补偿精度高，但长短期记忆网络等模型严重依赖密集的矩阵乘加运算。
3. **痛点 (GAP):** 高昂的计算代价导致现有模型难以部署在微控制器上；且纯数据驱动模型缺乏物理频率响应规律的约束。
4. **解决方案:** 顺理成章地引出 Wiener-KAN。
   * **Wiener:** 解决物理规律缺失（注入局部频率响应先验）。
   * **KAN:** 解决算力瓶颈（基于一维样条，可被高保真近似为一维查找表 LUT，将浮点运算转为内存寻址读表）。

## 3. 沉淀至 paper-spec 规范的经验
在本次打磨过程中，发现了多个典型的学术写作误区，现已提炼并沉淀至 `paper-spec` 技能规范中，用于约束后续的所有正文撰写：

1. **避免逻辑倒置与提前剧透：** 必须严守“背景 $\to$ GAP $\to$ 方案”的逻辑，禁止在点出痛点前就抛出最终架构。
2. **精准对应 GAP：** 提出的解决方案特性必须与前面的 GAP 严丝合缝地对应。对于正文没有专门验证的问题（如宽泛的“可解释性”），在引言中必须弱化或删减。
3. **禁用商业化和夸张词汇：** 严禁使用“壁垒”（改为局限/限制）、“护城河”、“填补空白”等陈词滥调或口语化词汇，保持严谨客观。
4. **规范括号使用：** 严禁在括号内放置解释性长句或定语从句，长说明必须融入句子主干，括号仅限用于缩写定义（如 MCU、KAN）。

## 4. 结论
通过本次重构，文章从一个“提出某种特定神经网络的算法报告”，成功蜕变为一篇“直击边缘算力限制、实现传感器板端高吞吐率实时补偿”的系统级工程导向论文，逻辑自洽且应用价值凸显。
