# 器件选型、SPICE 收敛与偏置排查

## 功能概述

本项目的模拟电路验证，不只依赖网络结构和权重映射，还高度依赖器件模型、仿真收敛条件和偏置误差来源。

这份文档用于收敛已经稳定的工程规则：

- 运放、二极管等关键器件应如何筛选
- `timestep too small` 一类 SPICE 报错应如何解释
- ReLU / 高通 / Dense 路线里，偏置更可能从哪里产生
- 调零电阻、测试点和跨仿真器复核应如何放置

## 当前代码与目录入口

与这条主线直接相关的入口如下：

- `src/spice_simulator/`：网表生成、器件模型与仿真工具
- `src/visualization/wnet5_circuit_validator.py`：分层频响与误差对比
- `src/inference/wavenet5_spice_backend.py`：WaveNet5 SPICE 推理后端
- `circuit_design/`：电路图、KiCad / LTSpice / Multisim 相关原理图资产

这份文档只沉淀长期规则，不替代具体器件库或某次仿真工程文件。

## 运放选型的长期规则

### 低偏置、低失调是第一道筛选，不是最终结论

对当前 Dense / ReLU / 精密整流路线，运放应优先关注：

- 输入偏置电压
- 输入偏置电流
- 低频漂移
- 器件模型在目标仿真器中的稳定性

长期上，低偏置和低失调只能作为第一道筛选条件，不能替代整层验证。器件手册更好看，不等于放进当前电路后误差就一定更小。

### 理想运放只能当下界，不是验收标准

理想运放的作用主要是：

- 验证电路连接是否合理
- 给出偏置与增益误差的理论下界

如果理想运放可以工作而真实模型失败，第一优先排查项应是：

- 输入极性是否接反
- 静态工作点是否被破坏
- 二极管或高阻网络是否让回路进入难收敛区间

不要把“理想模型能跑”直接当作“真实器件一定没问题”。

## 二极管选型的长期规则

### 精密整流优先看漏电，不优先看正向压降

在运放反馈式精密整流结构里，正向压降会被闭环增益显著压缩；真正更容易决定低频小信号误差的，是反向漏电流和温漂。

因此，长期规则是：

- 精密整流场景优先比较漏电流等级
- 再看温漂与离散性
- 正向压降只在离开闭环精密整流假设后才重新变得主导

### BAS116 比 1N4148 更适合作为低频精密整流默认候选

对当前低频、高精度整流需求，长期判断是：

- `BAS116` 一类低漏电二极管比 `1N4148` 更适合作为默认候选
- `1N4148` 更适合一般开关或非精密场景，不适合作为精密整流的默认器件

这条判断的根因不是品牌偏好，而是低频精密整流里漏电误差会直接进入零偏误差预算。

## SPICE 收敛问题的解释原则

### `timestep too small` 应视为非收敛症状，不是根因

当前长期经验是：当仿真报出 `timestep too small` 时，应优先把它理解为“某个子电路进入了难收敛状态”，而不是把错误文字本身当作根因解释。

更常见的根因包括：

- 回路增益过高或工作点不稳定
- 二极管与运放组合导致刚性增强
- 偏置路径或高阻路径使数值条件恶化
- 器件模型对当前时域步进非常敏感

### Gear 与容差放宽是诊断手段，不是黄金配置

当真实器件模型难收敛时，当前有用的诊断方向包括：

- `method=gear`
- 增大 `reltol`
- 调整 `chgtol`
- 调整 `trtol`

这些设置的长期定位是：

- 帮助判断问题是数值收敛，还是电路本身错误
- 缩小问题范围
- 为真实器件仿真提供更稳定的起点

但它们不是“一次配置，永久通用”的黄金参数模板。若放宽容差后结果才勉强稳定，仍需回查电路拓扑和工作点。

## 跨仿真器复核原则

### 先核对连接，再比较仿真器

当 LTSpice、ngspice、Multisim 等结果不一致时，长期顺序应为：

1. 先核对原理图与网表连接。
2. 再核对器件模型是否等价。
3. 最后才讨论哪个仿真器更可信。

不要把跨仿真器差异直接归因于“某软件不准”。

### 当前 Dense 增益验证优先用 LTSpice 复核

从现有笔记收敛出的长期实践是：当 Dense 层放大倍数或静态偏置存在争议时，优先用 LTSpice 做一次独立复核，尤其是在原理图已从 KiCad 明确导入的前提下。

这不是把 LTSpice 绝对化，而是把它作为当前更稳定的二次确认工具。

## 偏置来源的长期归因

### ReLU 路线的偏置累计带有结构性

对当前模拟 ReLU / nReLU 路线，长期判断是：

- 偏置不能只当成偶然仿真噪声
- 它往往是二极管非理想、运放偏置和层间级联共同作用的结果
- 随着层数增加，小偏置会逐渐转化成非预期非线性

因此，ReLU 路线不应以“后面总能调零掉”为默认前提。

### 二极管非理想性是首要偏置来源之一

当前经验已经表明：

- 理想二极管与非理想二极管之间的差异，会显著改变中间层静态误差
- 若理想二极管结果接近、真实二极管结果明显恶化，应优先从器件非理想出发分析

这也是为什么精密整流器件应优先关注漏电与温漂。

### 被动高通和超大阻值网络容易把响应拉向 0

对高通偏置或调零路径，长期规则是：

- 超大阻值高通或分压网络，容易在负载作用下削弱静态控制能力
- 一旦看到整层增益明显往 0 收敛，应先检查阻值等级与负载耦合
- 无源高通如果已经表现出强负载敏感性，不应继续当作默认可用方案

## 调零与测试点的长期放置原则

### 调零优先放在 Dense 级

当前长期结论是：调零电阻优先放在 Dense 级，而不是权重板或更前面的纯线性部分。

原因是：

- 权重板可以保持更极简
- Dense 级更接近误差真正汇聚的位置
- 运放与激活级联后的偏置更容易在这一层统一修正

### ReLU 前测试点应保留

如果输出链路包含 ReLU 或整流级，长期上应优先保留 ReLU 前的测试点。这样可以把：

- 线性加权误差
- 激活引入的偏置误差

拆开看，而不是全部混在最终输出里。

## 当偏置难以压下去时的结构选择

如果偏置已经成为主导误差源，当前更值得优先验证的路线是：

- 无显式 bias 的 tanh 路线
- 中心对称非线性
- 更低阻抗、更可控的调零与高通结构

长期原因是，这些方案通常比偏置驱动型 ReLU 更不容易把微小失调放大为结构性误差。

## 推荐排查顺序

1. 先用理想器件确认连接和功能。
2. 再换真实器件模型，看偏置和收敛是否恶化。
3. 出现 `timestep too small` 时，先按非收敛问题排查，不直接改结论。
4. 对精密整流优先换低漏电二极管，而不是先纠结正向压降。
5. 对调零优先在 Dense 级处理，并保留 ReLU 前测试点。
