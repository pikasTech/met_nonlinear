# 偏置误差分析功能测试计划

## 测试概述

本测试计划旨在全面验证 `cli.py -a` 命令的偏置误差分析增强功能。测试采用渐进式策略，从单元测试到系统集成测试，确保功能的正确性、稳定性和性能。

## 测试环境

- **Python版本**: 3.9 (tf26 conda环境)
- **依赖框架**: TensorFlow 2.6, NumPy 1.19.5
- **测试框架**: pytest
- **测试平台**: Linux WSL2

## 测试策略

采用分层测试策略，按以下顺序执行：
1. 单元测试 → 2. 组件测试 → 3. 集成测试 → 4. 系统测试 → 5. 性能测试 → 6. 异常测试

## 1. 单元测试 (Unit Tests)

### 1.1 核心算法测试

#### TEST-U-001: 稳态提取算法
- **目的**: 验证 `SteadyStateBiasAnalyzer` 正确计算偏置
- **输入**: 纯DC信号 (DC=2.5V, 1000样本)
- **预期**: 计算偏置误差 < 0.001V
- **验证点**: 
  - 稳态段识别正确
  - 偏置计算精度符合要求

#### TEST-U-002: 频域滤波算法
- **目的**: 验证 `FrequencyDomainBiasAnalyzer` 的DC分量提取
- **输入**: DC(1.5V) + 50Hz正弦波(0.5V) + 150Hz谐波(0.2V)
- **预期**: 提取DC分量误差 < 0.000001V
- **验证点**:
  - FFT正确执行
  - DC分量准确分离
  - 窗函数应用正确

#### TEST-U-003: 自动方法选择
- **目的**: 验证 `AutoBiasAnalyzer` 的智能选择逻辑
- **输入**: 
  - 瞬态信号（指数衰减）
  - 多频率信号（3个以上频率分量）
- **预期**: 
  - 瞬态信号 → 选择稳态方法
  - 多频率信号 → 选择频域方法
- **验证点**: 信号特征检测准确

#### TEST-U-004: 通道偏置分析器
- **目的**: 验证 `ChannelBiasAnalyzer` 多通道处理
- **输入**: 10通道信号，每通道不同DC偏置(0.5V递增)
- **预期**: 各通道偏置计算误差 < 0.001V
- **验证点**: 通道独立性保持

### 1.2 数据结构测试

#### TEST-U-005: 偏置误差矩阵生成
- **目的**: 验证多层偏置矩阵正确生成
- **输入**: 5层×10通道的模拟数据
- **预期**: 生成shape=(5,10)的偏置矩阵
- **验证点**:
  - 矩阵维度正确
  - 数据类型为float
  - 无NaN或Inf值

#### TEST-U-006: 统计信息计算
- **目的**: 验证统计指标计算正确性
- **输入**: 已知偏置的测试矩阵
- **预期**: 
  - 均值、标准差计算正确
  - 最差情况定位准确
- **验证点**: 统计公式实现正确

## 风险和缓解措施

1. **风险**: 大数据处理可能导致内存问题
   - **缓解**: 实现分批处理机制

2. **风险**: 浮点精度可能影响偏置计算
   - **缓解**: 使用适当的容差范围

3. **风险**: 并行处理可能导致竞态条件
   - **缓解**: 确保线程安全
