# WaveNet5 极端偏置调整验证测试计划

## 背景

基于前一轮测试发现小偏置值（0.001-0.002级别）对输出范围影响微小的情况，本测试计划采用极端偏置值（0.1-1.0级别）来明确验证偏置调整功能的实际效果。

## 测试目标

1. **验证偏置调整实际生效**: 使用极端偏置值确保功能真正影响推理结果
2. **量化偏置调整影响**: 通过对比输出范围的数值差异来量化影响程度
3. **确认功能健壮性**: 验证极端值下系统的稳定性和数值正确性
4. **提供明确证据**: 为偏置调整功能的有效性提供不可辩驳的数值证据

## 测试策略

### 核心策略
- **极端值测试**: 使用0.1-1.0范围的偏置值，确保影响可见
- **直接对比**: 通过输出范围的直接对比来判断效果
- **避免分析器**: 不使用 `-a` 分析命令，避免兼容性问题
- **聚焦范围差异**: 重点关注最小值、最大值、范围跨度的变化

### 偏置值设计原则
1. **足够大**: 偏置值应大到足以影响输出范围
2. **方向性**: 使用正负混合值测试不同方向的影响
3. **层级差异**: 不同层使用不同幅度的偏置值
4. **对称性**: 设计便于观察和验证的对称模式

## 测试用例

### TC-E1: 极端偏置启用测试
**目标**: 验证极端偏置值下的推理结果
**配置**: `bias_compensation.enabled = true`
**执行命令**: `conda run -n tf26 python cli.py -i -f WNET5q1h2u6l3 --layers 2`

**关键观察点**:
- 神经网络第1层输出范围
- 神经网络第2层输出范围 
- SPICE第1层输出范围
- SPICE第2层输出范围
- NumPy第1层输出范围
- NumPy第2层输出范围

**期望结果**:
- 输出范围应明显偏移，体现偏置调整的影响
- 各通道应表现出与偏置值相对应的变化方向
- 系统应保持数值稳定，无异常值或发散

### TC-E2: 偏置禁用对照测试
**目标**: 获取无偏置调整的基准结果
**配置**: `bias_compensation.enabled = false`
**执行命令**: `conda run -n tf26 python cli.py -i -f WNET5q1h2u6l3 --layers 2`

**关键观察点**:
- 与TC-E1相同的输出范围数据
- 确保为纯净的基准结果

**期望结果**:
- 应与之前小偏置值测试的结果一致
- 为差异对比提供可靠基准

### TC-E3: 差异量化分析
**目标**: 量化极端偏置的具体影响
**方法**: 手动计算和对比TC-E1与TC-E2的结果

**分析指标**:
1. **绝对差异**: |启用值 - 禁用值|
2. **相对差异**: |(启用值 - 禁用值) / 禁用值| × 100%
3. **范围变化**: 最大值和最小值的分别变化
4. **中心偏移**: (最大值+最小值)/2 的变化

**期望结果**:
- 绝对差异应在0.1-1.0量级，与偏置值相当
- 相对差异应为明显的百分比变化（>5%）
- 能清晰观察到偏置方向对应的数值变化

## 成功标准

### 主要成功标准
1. **可观察差异**: 启用和禁用偏置调整的输出范围存在明显差异（>0.05）
2. **方向一致性**: 偏置值的正负与输出变化方向一致
3. **幅度合理性**: 输出变化幅度与偏置值大小成比例关系
4. **系统稳定性**: 极端偏置值下系统无错误、崩溃或数值异常

### 定量标准
- **最小可检测差异**: 输出范围变化 ≥ 0.1
- **相对变化率**: ≥ 10% 的相对差异
- **一致性**: 多个通道的变化方向与偏置符号一致
- **收敛性**: 推理过程正常完成，无数值发散

### 失败标准
- 启用和禁用偏置调整的结果完全相同
- 系统出现错误、崩溃或数值异常
- 偏置影响方向与设定值相反
- 变化幅度远超偏置值设定范围（数值不稳定）

## 风险评估

### 极端值风险
1. **数值溢出**: 大偏置值可能导致激活函数饱和或数值溢出
2. **收敛问题**: 可能影响推理过程的数值稳定性
3. **物理意义**: 极端偏置可能超出模型的有效工作范围

### 缓解措施
1. **渐进测试**: 如果1.0级别偏置过大，逐步降低到0.5、0.3级别
2. **错误监控**: 密切监控推理过程是否出现错误或警告
3. **范围检查**: 确保输出值在物理和数学上合理
