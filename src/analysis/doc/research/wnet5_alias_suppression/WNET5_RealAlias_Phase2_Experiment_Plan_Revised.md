# WNET5_RealAlias Phase2 实验计划（修订版）

## 背景与目标

Phase1实验确认**E05（宽层Dense网络）**为最优配置，实现了90.3%的抑制率，远超基线的54.2%。Phase2以**E05作为新基线**，继续采用控制变量的对照试验方法，探索进一步优化的可能性。

## 新基线配置分析（E05最优）

### E05配置回顾

### Phase1关键洞察
1. **宽层优于深层**：16单元×3层 > 12单元×6层
2. **频率扩展有效**：E01的频率扩展带来显著改善
3. **激活函数敏感**：relu远优于tanh
4. **学习率适中**：过于保守的学习率改善有限

## Phase2 实验设计（6个实验）

基于控制变量原则，每个实验仅改变E05基线的一个关键参数：

### E07 - 组合频率优势
**实验目的**：将E01的频率扩展优势与E05的宽层设计结合
**变更参数**：
- `init_center_freqs`: [3, 8, 25, 50, 85, 120, 180, 250] (E01配置)
- `init_quality_factors`: [1.5, 2.0, 2.5, 3.0, 4.0, 5.0, 6.0, 7.0] (E01配置)
- `kernal_units`: 8 (对应频率数量)

**预期情况分析**：
- **情况A (协同效应)**: 抑制率达到95%+ → 结论：频率扩展与宽层设计产生协同效应
- **情况B (相加效应)**: 抑制率92-95% → 结论：两种优势简单叠加
- **情况C (无额外收益)**: 抑制率90-92% → 结论：E05已达到频率相关的性能上限
- **情况D (性能下降)**: 抑制率<90% → 结论：过多频率点引入干扰

### E08 - 极致宽层网络
**实验目的**：探索宽层设计的性能上限
**变更参数**：
- `post_dense_units`: 24 (从16增加)

**预期情况分析**：
- **情况A (继续提升)**: 抑制率93%+ → 结论：网络容量仍是瓶颈，可继续扩展
- **情况B (收益递减)**: 抑制率90-93% → 结论：接近宽层设计的最优点
- **情况C (无明显改善)**: 抑制率87-90% → 结论：16单元已是最优宽度
- **情况D (过拟合)**: 抑制率<87% → 结论：过宽网络导致过拟合

### E09 - 超浅层设计
**实验目的**：验证是否可以用更浅的网络达到相似效果
**变更参数**：
- `post_dense_layers`: 2 (从3减少)
- `post_dense_units`: 20 (适当增加补偿深度减少)

**预期情况分析**：
- **情况A (效率提升)**: 抑制率88%+ → 结论：超浅层+更宽是更优设计
- **情况B (性能相当)**: 抑制率85-88% → 结论：深度和宽度可以互换
- **情况C (性能下降)**: 抑制率<85% → 结论：3层是必要的最小深度

### E10 - 精进学习策略
**实验目的**：基于E05优化学习率和训练策略
**变更参数**：
- `learning_rate`: 0.015 (从0.02微调)
- `auto_lr_decay_steps`: 1500 (延缓衰减)
- `epoch_train`: 40000 (增加训练时间)

**预期情况分析**：
- **情况A (精细优化)**: 抑制率92%+ → 结论：E05受限于训练策略而非网络容量
- **情况B (边际改善)**: 抑制率90-92% → 结论：训练策略有一定作用但非主要因素
- **情况C (无明显差异)**: 抑制率88-90% → 结论：当前学习率策略已经合适

### E11 - 滤波器数量优化
**实验目的**：验证增加滤波器数量对性能的影响
**变更参数**：
- `kernal_units`: 8 (从6增加)
- `init_center_freqs`: [5, 15, 30, 50, 75, 100, 130, 170] (重新分布)
- `init_quality_factors`: [1.0, 1.5, 2.0, 2.5, 3.0, 4.0, 5.0, 6.0] (重新分布)

**预期情况分析**：
- **情况A (频率精度提升)**: 抑制率93%+ → 结论：更多滤波器能更精确捕获aliasing
- **情况B (有限改善)**: 抑制率90-93% → 结论：6个滤波器接近最优
- **情况C (复杂度增加无收益)**: 抑制率<90% → 结论：过多滤波器引入噪声

### E12 - 质量因子精调
**实验目的**：基于E05精细调节质量因子配置
**变更参数**：
- `init_quality_factors`: [1.2, 1.8, 2.2, 2.8, 3.5, 4.5] (微调数值)

**预期情况分析**：
- **情况A (频率选择性优化)**: 抑制率92%+ → 结论：质量因子的精确配置很重要
- **情况B (微小改善)**: 抑制率90-92% → 结论：当前Q值配置已接近最优
- **情况C (无明显差异)**: 抑制率88-90% → 结论：网络对Q值变化不敏感

## 实验优先级与执行策略

### 高优先级（立即执行）
1. **E07** - 组合频率优势（最有希望突破95%）
2. **E08** - 极致宽层网络（探索性能上限）

### 中优先级（根据前两个结果决定）
3. **E10** - 精进学习策略（如果E07-E08未达预期）
4. **E11** - 滤波器数量优化（如果频率相关策略有效）

### 低优先级（可选执行）
5. **E09** - 超浅层设计（效率优化）
6. **E12** - 质量因子精调（微调优化）

## 成功标准与决策树

### 性能标准（基于E05：90.3%）
- **突破性成功**: 抑制率 > 95%，综合评分 > 95
- **显著改善**: 抑制率 92-95%，综合评分 92-95
- **边际改善**: 抑制率 90-92%，综合评分 90-92
- **无改善**: 抑制率 < 90%

### 决策逻辑

#### 如果E07达到95%+
→ **结论**：频率扩展+宽层设计是最优组合
→ **行动**：项目成功完成，进入应用阶段
→ **后续**：可选择性执行E08验证性能上限

#### 如果E07达到92-95%
→ **结论**：组合策略有效但未达最优
→ **行动**：执行E08和E10寻求进一步突破
→ **后续**：分析最佳组合

#### 如果E07无显著改善（<92%）
→ **结论**：E05可能已接近架构性能上限
→ **行动**：重点执行E08-E11探索其他优化方向
→ **后续**：总结当前架构的性能边界
